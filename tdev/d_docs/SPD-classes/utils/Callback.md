## 2. 文件职责说明

### 核心职责
定义通用回调接口，提供无参数、无返回值的函数调用契约，用于事件处理、异步操作完成通知和命令模式实现。

### 系统定位
作为游戏引擎中的基础回调机制，为各种事件驱动场景提供标准化的回调接口。

### 不负责什么
- 不负责回调的执行时机管理
- 不处理异常传播
- 不提供参数传递机制

## 11. 使用示例

### 基本用法
```java
// 使用匿名内部类
Callback callback = new Callback() {
    @Override
    public void call() {
        System.out.println("Callback executed!");
    }
};
callback.call();

// 使用Lambda表达式（Java 8+）
Callback lambdaCallback = () -> System.out.println("Lambda callback!");
lambdaCallback.call();
```

### 游戏场景示例
```java
// 游戏中的延迟执行
Game.runIn(1.0f, new Callback() {
    @Override
    public void call() {
        hero.heal(10);
        GLog.i("Hero healed!");
    }
});

// 事件监听
button.onClick(new Callback() {
    @Override
    public void call() {
        startNewGame();
    }
});
```

## 12. 开发注意事项

### 状态依赖
- 回调实现可能依赖外部状态
- 需要注意闭包变量的生命周期

### 生命周期耦合
- 回调对象的生命周期应与调用者的生命周期协调
- 避免内存泄漏（如持有Activity引用）

### 常见陷阱
- 在回调中修改UI时忘记检查线程安全
- 忘记处理回调中的异常
- 创建循环引用导致内存泄漏
- 在错误的时机调用回调（如对象已销毁后）

## 13. 修改建议与扩展点

### 适合扩展的位置
- 可以创建带参数的回调接口（Callback<T>）
- 可以添加异常处理机制

### 不建议修改的位置
- 接口本身（简单性是其优势）

### 重构建议
- 考虑使用Java 8的FunctionalInterface注解
- 对于复杂场景，考虑使用更具体的回调接口
