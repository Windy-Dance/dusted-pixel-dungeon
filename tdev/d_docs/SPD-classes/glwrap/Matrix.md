## 2. 文件职责说明

### 核心职责
Matrix类提供4x4矩阵操作的实用方法，用于OpenGL变换计算，包括平移、旋转、缩放、倾斜和矩阵乘法等操作。

### 系统定位
作为OpenGL变换矩阵的计算工具，为上层渲染系统提供高效的矩阵运算支持，属于图形数学基础设施层。

### 不负责什么
- 不负责矩阵数据的存储管理（仅操作传入的数组）
- 不负责OpenGL状态的直接操作
- 不负责向量或点的变换（仅提供矩阵操作）

## 3. 结构总览

### 主要成员概览
- `G2RAD` (static final float): 角度到弧度的转换系数（π/180）
- `identity` (private static float[]): 单位矩阵常量

### 主要逻辑块概览
- 矩阵复制：clone()和copy()方法
- 矩阵初始化：setIdentity()方法
- 变换操作：rotate()、skewX()、skewY()、scale()、translate()方法
- 矩阵乘法：multiply()方法

### 生命周期/调用时机
在需要计算变换矩阵时调用，通常在场景对象的位置/旋转/缩放更新时使用。

## 7. 方法详解

### clone(float[] m)

**可见性**：public static

**是否覆写**：否

**方法职责**：创建矩阵数组的深拷贝

**参数**：
- `m` (float[])：源矩阵数组

**返回值**：float[]，新创建的矩阵数组副本

**前置条件**：m不能为null，长度必须为16（4x4矩阵）

**副作用**：无

**核心实现逻辑**：
使用倒序循环复制数组元素，从最后一个元素开始向前复制

**边界情况**：如果m为null会抛出NullPointerException

### copy(float[] src, float[] dst)

**可见性**：public static

**是否覆写**：否

**方法职责**：将源矩阵数组复制到目标数组

**参数**：
- `src` (float[])：源矩阵数组
- `dst` (float[])：目标矩阵数组

**返回值**：void

**前置条件**：src和dst都不能为null，长度都必须为16

**副作用**：修改dst数组的内容

**核心实现逻辑**：
使用倒序循环将src数组元素复制到dst数组

**边界情况**：如果数组为null或长度不匹配会抛出异常

### setIdentity(float[] m)

**可见性**：public static

**是否覆写**：否

**方法职责**：将指定矩阵设置为单位矩阵

**参数**：
- `m` (float[])：要设置的矩阵数组

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容

**核心实现逻辑**：
使用System.arraycopy将预定义的identity数组复制到m

**边界情况**：如果m为null或长度不足会抛出异常

### rotate(float[] m, float a)

**可见性**：public static

**是否覆写**：否

**方法职责**：对矩阵应用绕Z轴的旋转变换

**参数**：
- `m` (float[])：要变换的矩阵数组
- `a` (float)：旋转角度（度）

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容（前两列）

**核心实现逻辑**：
1. 将角度转换为弧度（a *= G2RAD）
2. 计算sin和cos值
3. 对矩阵的前两列应用2D旋转变换公式

**边界情况**：大角度值会被三角函数自动处理（周期性）

### skewX(float[] m, float a)

**可见性**：public static

**是否覆写**：否

**方法职责**：对矩阵应用X轴倾斜变换

**参数**：
- `m` (float[])：要变换的矩阵数组
- `a` (float)：倾斜角度（度）

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容（第2列）

**核心实现逻辑**：
1. 计算倾斜系数t = tan(a * G2RAD)
2. 将第1列乘以-t后加到第2列

**边界情况**：当a接近±90度时，tan值会很大，可能导致数值不稳定

### skewY(float[] m, float a)

**可见性**：public static

**是否覆写**：否

**方法职责**：对矩阵应用Y轴倾斜变换

**参数**：
- `m` (float[])：要变换的矩阵数组
- `a` (float)：倾斜角度（度）

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容（第1列）

**核心实现逻辑**：
1. 计算倾斜系数t = tan(a * G2RAD)
2. 将第2列乘以t后加到第1列

**边界情况**：当a接近±90度时，tan值会很大，可能导致数值不稳定

### scale(float[] m, float x, float y)

**可见性**：public static

**是否覆写**：否

**方法职责**：对矩阵应用XY平面缩放变换

**参数**：
- `m` (float[])：要变换的矩阵数组
- `x` (float)：X轴缩放因子
- `y` (float)：Y轴缩放因子

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容（前两列）

**核心实现逻辑**：
将矩阵的第1列乘以x，第2列乘以y

**边界情况**：
- 负缩放因子会产生镜像效果
- 零缩放因子会使对应维度坍缩

### translate(float[] m, float x, float y)

**可见性**：public static

**是否覆写**：否

**方法职责**：对矩阵应用XY平面平移变换

**参数**：
- `m` (float[])：要变换的矩阵数组
- `x` (float)：X轴平移距离
- `y` (float)：Y轴平移距离

**返回值**：void

**前置条件**：m不能为null，长度必须为16

**副作用**：修改m数组的内容（第4列的前两个元素）

**核心实现逻辑**：
使用齐次坐标变换公式：m[12] += m[0]*x + m[4]*y; m[13] += m[1]*x + m[5]*y;

**边界情况**：无特殊边界情况

### multiply(float[] left, float[] right, float[] result)

**可见性**：public static

**是否覆写**：否

**方法职责**：计算两个4x4矩阵的乘积

**参数**：
- `left` (float[])：左矩阵（第一个操作数）
- `right` (float[])：右矩阵（第二个操作数）
- `result` (float[])：存储结果的矩阵数组

**返回值**：void

**前置条件**：所有参数都不能为null，长度都必须为16

**副作用**：修改result数组的内容

**核心实现逻辑**：
实现标准的4x4矩阵乘法，按行主序存储，逐元素计算结果

**边界情况**：无特殊边界情况，但要注意矩阵乘法不满足交换律

## 11. 使用示例

### 基本用法
```java
// 创建变换矩阵
float[] transform = new float[16];
Matrix.setIdentity(transform);

// 应用复合变换：先平移，再旋转，最后缩放
Matrix.translate(transform, 100f, 200f);
Matrix.rotate(transform, 45f); // 45度旋转
Matrix.scale(transform, 2f, 2f); // 2倍缩放

// 或者分别创建矩阵然后相乘
float[] translateMat = new float[16];
Matrix.setIdentity(translateMat);
Matrix.translate(translateMat, 100f, 200f);

float[] rotateMat = new float[16];
Matrix.setIdentity(rotateMat);
Matrix.rotate(rotateMat, 45f);

float[] finalTransform = new float[16];
Matrix.multiply(translateMat, rotateMat, finalTransform);
```

## 12. 开发注意事项

### 状态依赖
- 所有操作都直接修改传入的数组，需要注意引用共享问题
- 矩阵必须是16元素的float数组（4x4矩阵）

### 生命周期耦合
- 矩阵数组的生命周期由调用者管理
- 多次变换操作会累积效果，注意变换顺序的重要性

### 常见陷阱
- **变换顺序很重要**：translate→rotate→scale 与 scale→rotate→translate 结果完全不同
- **角度单位**：所有角度参数都是度数，内部自动转换为弧度
- **内存效率**：避免频繁创建新数组，重用现有数组更高效
- **数值精度**：多次变换可能导致浮点精度累积误差
