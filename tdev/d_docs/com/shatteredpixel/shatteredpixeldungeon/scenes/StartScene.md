# StartScene 类文档

## 概述
`StartScene` 是 Shattered Pixel Dungeon 的游戏选择场景类，继承自 `PixelScene`。它显示所有已保存的游戏进度，并允许玩家选择继续游戏或开始新游戏。

## 类结构
```java
public class StartScene extends PixelScene
```

## 主要功能

### 1. 游戏存档管理
- **存档槽位显示**: 显示所有已存在的游戏存档和空闲的存档槽位
- **最大槽位限制**: 支持最多 `GamesInProgress.MAX_SLOTS` 个存档槽位
- **动态布局**: 根据存档数量自动调整按钮间距和位置

### 2. 存档信息展示
每个存档槽位显示丰富的游戏信息：

**基础信息**:
- 英雄职业图标和头像
- 角色名称（转职后的子职业名或基础职业名）
- 最后游玩时间（智能时间格式化：分钟/小时/天/月）

**游戏进度**:
- 当前关卡深度（楼梯图标 + 数字）
- 英雄等级（职业图标 + 等级数字）
- 特殊模式标识（挑战模式、每日挑战、自定义种子等）

**视觉反馈**:
- 挑战模式：所有文字和图标显示为金色
- 每日挑战：楼梯图标显示为青色（普通）或紫色（重播）
- 自定义种子：楼梯图标显示为黄色

### 3. 排序功能
- **排序选项**: 支持按关卡深度或最后游玩时间排序
- **排序切换**: 点击排序按钮在两种模式间切换
- **无缝刷新**: 切换排序后自动重新加载场景

### 4. 用户交互
- **新游戏**: 点击空存档槽位跳转到英雄选择场景
- **继续游戏**: 点击已有存档显示详细信息窗口 (`WndGameInProgress`)
- **返回标题**: 按返回键返回主菜单

### 5. 响应式设计
- **自适应布局**: 根据屏幕高度和存档数量调整按钮间距
- **安全区域处理**: 自动适配刘海屏和圆角屏
- **设备兼容性**: 在小屏幕上优化显示效果

## 内部类

### SaveSlotButton (存档槽位按钮)
继承 `Button` 的专用按钮类，负责显示单个存档槽位：

**UI组件**:
- `bg`: 背景九宫格图片 (`Chrome.Type.TOAST_TR`)
- `hero`: 英雄头像精灵（12x15像素）
- `name`: 角色名称文本
- `lastPlayed`: 最后游玩时间文本
- `steps`: 楼梯图标（表示关卡深度）
- `depth`: 关卡深度数字
- `classIcon`: 职业图标
- `level`: 英雄等级数字

**主要方法**:
- `set(int slot)`: 设置存档槽位数据
- `layout()`: 精确布局所有UI元素
- `onClick()`: 处理点击事件（新游戏或继续游戏）

## 生命周期方法

### create()
场景初始化的核心方法：
1. 加载全局徽章和日志数据
2. 创建标题背景
3. 添加退出按钮
4. 创建标题文本（"选择游戏"）
5. 获取所有存档信息并创建对应的按钮
6. 添加新游戏按钮（如果有空闲槽位）
7. 创建排序按钮（当存档数≥2时）
8. 执行淡入动画

### onBackPressed()
处理返回键事件，返回到主菜单场景

## 数据处理

### 时间格式化
智能处理最后游玩时间的显示：
- <1分钟：显示"刚刚"
- <2小时：显示"X分钟前"
- <2天：显示"X小时前"  
- <2个月：显示"X天前"
- >99个月：不显示文本（避免过长）

### 角色名称处理
- 有转职：显示子职业名称（首字母大写）
- 无转职：显示基础职业名称（首字母大写）

### 特殊模式检测
- `info.challenges > 0`: 挑战模式（金色标识）
- `info.daily`: 每日挑战模式（彩色楼梯图标）
- `!info.customSeed.isEmpty()`: 自定义种子模式（黄色楼梯图标）

## 性能优化
- **延迟加载**: 只在需要时创建UI组件
- **内存管理**: 正确清理不再需要的图像资源
- **布局缓存**: 避免重复计算布局位置

## 用户体验设计
- **直观分类**: 新游戏和已有游戏清晰区分
- **信息密度**: 在有限空间内显示最多有用信息
- **视觉层次**: 重要信息（角色名、关卡）更加突出
- **无障碍设计**: 足够大的触摸目标区域

## 特殊注意事项
1. **数据同步**: 确保在显示前加载完成所有全局数据
2. **内存泄漏**: 正确管理动态创建的UI组件生命周期
3. **本地化**: 所有文本通过消息系统获取，支持多语言
4. **性能考虑**: 避免在布局计算中执行昂贵操作
