# WelcomeScene 类文档

## 概述
`WelcomeScene` 是 Shattered Pixel Dungeon 的欢迎场景类，继承自 `PixelScene`。它在游戏启动时显示，处理版本检查、数据迁移、新功能提示以及首次运行的引导流程。

## 类结构
```java
public class WelcomeScene extends PixelScene
```

## 主要功能

### 1. 版本管理
- **版本检测**: 检测游戏是否为新安装或版本升级
- **数据迁移**: 处理旧版本数据到新版本的自动迁移
- **兼容性检查**: 确保存档和设置与当前版本兼容
- **清理操作**: 清理过时的临时文件和缓存

### 2. 首次运行引导
- **新手教程**: 为新玩家提供基础操作指导
- **设置初始化**: 设置默认的游戏选项和偏好
- **语言选择**: 引导玩家选择游戏语言
- **控制配置**: 帮助玩家配置输入控制方式

### 3. 更新通知系统
- **重大更新提示**: 显示重要功能变更的通知
- **强制更新检查**: 检测是否需要强制更新
- **新功能介绍**: 介绍新版本的主要功能和改进
- **已知问题警告**: 提醒玩家注意已知的问题

### 4. 数据完整性检查
- **存档验证**: 验证现有存档的完整性和有效性
- **资源检查**: 确保所有必要资源文件存在且完整
- **错误恢复**: 处理损坏数据的自动恢复机制
- **备份创建**: 在重要操作前自动创建数据备份

## 视觉设计

### 启动界面
- **标题显示**: 显示游戏标题和版本信息
- **进度指示**: 显示加载和初始化的进度
- **品牌元素**: 展示游戏的品牌标识和视觉风格
- **动画效果**: 简单的启动动画增加专业感

### 通知系统
- **硬通知窗口**: 使用 `WndHardNotification` 显示重要消息
- **错误对话框**: 使用 `WndError` 显示严重错误
- **确认对话框**: 获取用户对重要操作的确认
- **进度条**: 显示长时间操作的进度

### 用户引导
- **步骤指示**: 清晰的多步骤引导流程
- **视觉提示**: 使用图标和颜色突出重要信息
- **交互反馈**: 提供明确的操作反馈
- **跳过选项**: 允许有经验的玩家跳过引导

## 技术实现

### 文件系统集成
- **FileUtils**: 使用文件工具类进行文件操作
- **数据序列化**: 处理复杂数据结构的序列化和反序列化
- **版本标记**: 在数据文件中标记版本信息
- **原子操作**: 确保数据操作的原子性和一致性

### 错误处理
- **异常捕获**: 全面的异常处理和错误恢复
- **日志记录**: 详细的错误日志记录
- **用户友好**: 将技术错误转换为用户友好的消息
- **安全降级**: 在错误情况下提供安全的降级方案

### 资源管理
- **Assets加载**: 确保所有必要资源正确加载
- **内存优化**: 避免启动过程中的内存峰值
- **异步加载**: 在可能的情况下使用异步资源加载
- **缓存策略**: 实现有效的资源缓存策略

## 生命周期方法

### create()
场景初始化的核心逻辑：
1. 执行版本检测和数据迁移
2. 验证数据完整性和兼容性
3. 处理首次运行的特殊逻辑
4. 显示必要的通知和引导
5. 根据结果决定下一步（进入主菜单或显示错误）

### update()
每帧更新：
- 处理长时间操作的进度更新
- 管理通知窗口的显示和交互
- 处理用户输入和确认操作

### destroy()
清理启动过程中的临时资源

## 数据处理

### 版本跟踪
- **当前版本**: 从 `Game.version` 获取当前版本
- **最后版本**: 从设置中读取上次运行的版本
- **迁移标志**: 标记已完成的数据迁移
- **首次运行**: 检测是否为首次安装

### 设置管理
- **默认设置**: 初始化合理的默认设置值
- **用户偏好**: 保留用户的个性化设置
- **平台适配**: 根据平台调整默认设置
- **向后兼容**: 确保旧设置在新版本中有效

### 存档处理
- **格式验证**: 验证存档文件的格式正确性
- **版本兼容**: 处理不同版本存档的兼容性
- **损坏检测**: 检测和修复损坏的存档文件
- **备份策略**: 实施智能的存档备份策略

- **最小干扰**: 只在必要时显示通知和引导

- **明确选择**: 提供清晰的操作选项
- **进度透明**: 显示操作进度和预期时间

- **无障碍考虑**: 考虑不同能力用户的需求

- **开发者选项**: 在调试模式下显示额外选项

## 性能优化
