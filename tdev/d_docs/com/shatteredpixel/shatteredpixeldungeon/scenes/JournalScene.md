# JournalScene 类文档

## 概述
`JournalScene` 是 Shattered Pixel Dungeon 的游戏日志场景类，继承自 `PixelScene`。它提供玩家查看已收集的游戏信息、怪物图鉴、物品识别记录等知识库内容的界面。

## 类结构
```java
public class JournalScene extends PixelScene
```

## 主要功能

### 1. 知识库分类系统
**四个主要标签页**:
- **怪物图鉴**: 显示已遇到的所有怪物信息（生命值、伤害、特性等）
- **地形特征**: 显示各种陷阱、植物、地形元素的信息
- **物品识别**: 显示已识别的药水、卷轴、戒指等消耗品效果
- **任务笔记**: 显示当前进行的任务和目标信息

### 2. 内容展示
- **图标识别**: 使用对应物品或怪物的图标进行视觉识别
- **详细描述**: 每个条目都有详细的文字说明
- **进度追踪**: 显示知识库的收集进度（已发现/总数）
- **分类排序**: 条目按类型和名称自动排序

### 3. 动态内容更新
- **实时同步**: 游戏过程中获得的新信息会立即在日志中显示
- **未解锁隐藏**: 未发现的条目不会显示（保持探索乐趣）
- **成就关联**: 某些知识收集与徽章系统关联

### 4. 用户交互
- **条目详情**: 点击任意条目显示完整详情窗口 (`WndJournal`)
- **标签切换**: 快速在不同知识分类间切换
- **返回导航**: 退出按钮返回主菜单

## 技术实现

### 数据源集成
- **MonsterCatalog**: 怪物图鉴数据源
- **AlchemizeTab**: 炼金转化配方（如果已解锁）
- **TerrainFeaturesTilemap**: 地形特征信息
- **Potion/Scroll/Ring**: 消耗品识别信息

### 视觉系统
- **ItemSprite**: 使用标准物品精灵系统
- **TerrainFeaturesTilemap**: 地形特征渲染
- **Chrome.Type**: 统一的UI组件样式

## 生命周期方法

### create()
场景初始化：
1. 停止当前音乐并可能播放背景音乐
2. 创建标题背景和基础UI元素
3. 加载所有知识库数据
4. 创建四个标签页按钮
5. 执行淡入动画

### destroy()
场景销毁时清理资源

## 用户体验设计

### 信息架构
- **层级清晰**: 四个主要分类覆盖所有游戏知识
- **渐进披露**: 只显示已发现的内容，保持神秘感
- **视觉一致**: 使用游戏内相同的图标和样式

### 可用性考虑
- **快速访问**: 直接从主菜单进入
- **离线可用**: 所有信息本地存储，无需网络
- **搜索友好**: 条目按逻辑分组，便于查找

### 教育价值
- **学习工具**: 帮助新玩家理解游戏机制
- **策略参考**: 资深玩家可以查阅详细数据
- **收集驱动**: 鼓励玩家探索和实验

## 特殊功能

### 图鉴完成度
- 显示每个分类的完成百分比
- 激励玩家收集所有信息
- 与某些成就相关联

### 动态更新
- 游戏过程中实时更新
- 支持跨游戏会话的数据持久化
- 自动处理新版本添加的内容

## 性能优化
- **按需加载**: 详情窗口按需创建
- **缓存机制**: 图标和数据使用缓存
- **内存管理**: 正确清理临时资源
