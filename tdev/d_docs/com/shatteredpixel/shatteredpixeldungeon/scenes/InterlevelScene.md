# InterlevelScene 类文档

## 概述
`InterlevelScene` 是 Shattered Pixel Dungeon 的关卡间过渡场景类，继承自 `PixelScene`。它处理玩家在不同关卡之间移动、死亡复活、游戏开始/结束等各种过渡状态。

## 类结构
```java
public class InterlevelScene extends PixelScene
```

## 主要功能

### 1. 过渡模式管理
定义多种游戏过渡模式：

**关卡切换**:
- `DESCEND`: 向下进入新关卡
- `ASCEND`: 向上返回上一关卡  
- `FALL`: 掉入深渊后的关卡切换
- `RETURN`: 返回到之前访问过的关卡

**特殊状态**:
- `RESURRECT`: 死亡后使用安克符文复活
- `CONTINUE`: 继续已保存的游戏
- `RESTART`: 重新开始游戏
- `NONE`: 无特殊过渡（正常情况）

### 2. 关卡加载和初始化
- **关卡生成**: 根据过渡模式生成或加载目标关卡
- **英雄位置**: 设置英雄在新关卡中的起始位置
- **物品处理**: 处理跨关卡携带的物品和背包
- **状态同步**: 同步英雄的各种状态和buff

### 3. 特殊事件处理
**死亡处理**:
- 英雄死亡时的特殊逻辑
- 安克符文复活机制
- 背包丢失和物品处理
- 死亡记录保存

**胜利处理**:
- 击败最终Boss后的胜利逻辑
- 胜利成就验证
- 游戏完成记录保存

**异常恢复**:
- 游戏崩溃后的恢复机制
- 存档损坏的错误处理
- 网络问题的优雅降级

### 4. 视觉过渡效果
- **淡入淡出**: 平滑的场景切换动画
- **传送特效**: 使用传送卷轴或符文时的视觉效果
- **屏幕震动**: 特殊事件时的屏幕震动效果
- **粒子效果**: 各种过渡相关的粒子系统

## 技术架构

### 状态机设计
使用静态变量管理当前过渡状态：
- `mode`: 当前过渡模式
- `lastRegion`: 上一个区域标识（用于内存优化）
- `returnDepth`: 返回目标深度（用于RETURN模式）

### 关卡管理
- **Level类集成**: 与关卡生成系统深度集成
- **地形特征**: 处理楼梯、深渊、传送门等地形特征
- **房间系统**: 支持特殊房间的初始化

### 物品系统
- **LostBackpack**: 死亡时的背包丢失处理
- **Item管理**: 跨关卡物品的持久化
- **Artifact处理**: 特殊物品（如骷髅钥匙）的状态同步

## 生命周期方法

### create()
场景创建和过渡逻辑的核心：
1. 根据当前模式执行相应的过渡逻辑
2. 加载或生成目标关卡
3. 初始化英雄和环境
4. 处理特殊事件（死亡、胜利等）
5. 切换到GameScene或相应的目标场景

### update()
每帧更新过渡动画和状态检查

### destroy()
清理过渡相关的临时资源
