# TitleScene 类文档

## 概述
`TitleScene` 是 Shattered Pixel Dungeon 的主菜单场景类，继承自 `PixelScene`。它提供游戏的入口界面，包含所有主要功能按钮、版本信息以及动态内容更新提示（新闻、更新等）。

## 类结构
```java
public class TitleScene extends PixelScene
```

## 主要功能

### 1. 视觉元素
- **标题横幅**: 根据设备方向自动选择横屏或竖屏版本的标题图片
- **火焰效果**: 在标题两侧显示动态火焰特效（`Fireball`）
- **发光效果**: 标题周围的动态发光动画
- **背景**: 使用 `TitleBackground` 提供适应性的背景渲染

### 2. 功能按钮系统
采用响应式布局，根据设备方向自动调整按钮排列：

**主要按钮**:
- `btnPlay`: 进入游戏（无存档时跳转英雄选择，有存档时跳转开始场景）
- `btnSupport`: 支持者场景（金色主题，鼓励用户支持开发者）
- `btnRankings`: 排行榜场景（查看历史记录和成就）
- `btnJournal`: 游戏日志（查看收集到的信息、怪物图鉴等）
- `btnNews`: 新闻场景（显示开发者博客文章）
- `btnChanges`: 更新日志（显示版本变更信息）
- `btnSettings`: 设置菜单（配置游戏选项）
- `btnAbout`: 关于页面（显示游戏信息和许可证）

### 3. 动态内容集成
**新闻系统集成**:
- 自动检查新文章
- 显示未读文章数量（最多显示9篇）
- 未读文章时按钮文字变为动态彩色效果

**更新系统集成**:
- 自动检查游戏更新
- 有可用更新时按钮文字变更为"更新"
- 点击时显示更新详情对话框，提供更新或查看变更日志选项

**语言状态提示**:
- 当使用未完成翻译的语言时，设置按钮显示特殊图标
- 按钮文字变为动态红色警告效果

### 4. 特殊功能
**UI淡出功能**:
- 右下角提供淡出按钮，可以临时隐藏UI元素（用于截图）
- 点击屏幕任意位置恢复UI显示
- 使用补间动画实现平滑过渡效果

**胜利庆祝**:
- 首次检测到玩家获得胜利成就时，自动显示祝贺窗口
- 防止重复显示（通过 `victoryNagged` 设置标记）

**调试支持**:
- 调试模式下长按"进入游戏"按钮可快速开始新游戏

### 5. 设备适配
- **桌面平台**: 显示退出按钮
- **移动平台**: 优化触摸区域和按钮大小
- **横竖屏切换**: 完全不同的布局方案
- **安全区域**: 自动处理刘海屏和圆角屏

## 内部类

### NewsButton (新闻按钮)
继承 `StyledButton` 的专用按钮类：
- 在构造时自动检查新闻
- 动态显示未读计数
- 实现彩色脉冲动画效果

### ChangesButton (变更按钮)  
继承 `StyledButton` 的专用按钮类：
- 在构造时自动检查更新
- 有更新时显示特殊文本和动画
- 点击时显示智能更新对话框

### SettingsButton (设置按钮)
继承 `StyledButton` 的专用按钮类：
- 根据语言翻译状态显示不同图标
- 未完成翻译时显示警告动画
- 自动跳转到语言设置页面

### SupportButton (支持按钮)
继承 `StyledButton` 的专用按钮类：
- 使用金色图标和特殊文本颜色
- 直接跳转到支持者场景

## 生命周期方法

### create()
场景创建时的主要初始化工作：
1. 播放主题音乐（循环播放两首主题曲）
2. 创建标题背景和视觉元素
3. 初始化所有功能按钮并设置点击事件
4. 计算响应式布局（横竖屏不同算法）
5. 添加版本号显示
6. 设置UI淡出功能
7. 添加退出按钮（仅桌面平台）
8. 检查胜利成就并显示祝贺
9. 执行淡入动画

### updateFade()
UI淡出效果的核心方法：
- 控制所有UI元素的透明度
- 启用/禁用按钮交互
- 确保视觉一致性

### placeTorch()
辅助方法，用于在指定位置放置火焰特效

## 音频管理
- **主题音乐**: 自动播放并循环 `Assets.Music.THEME_1` 和 `Assets.Music.THEME_2`
- **音量控制**: 使用统一的音量设置
- **内存管理**: 场景销毁时自动停止音乐

## 性能优化
- **延迟加载**: 新闻和更新检查在按钮创建时进行，避免阻塞主线程
- **内存清理**: 继承自 `PixelScene` 的纹理缓存清理机制
- **动画优化**: 使用高效的补间动画系统

## 用户体验设计
- **直观导航**: 清晰的功能分类和图标标识
- **反馈机制**: 动态颜色变化提供状态反馈
- **无障碍设计**: 足够大的触摸目标区域
- **国际化**: 所有文本通过消息系统获取，支持多语言

## 特殊注意事项
1. **音乐管理**: 确保在场景切换时正确停止和播放音乐
2. **成就同步**: 胜利成就检查需要全局徽章数据加载完成
3. **网络依赖**: 新闻和更新功能依赖网络连接
4. **本地化**: 未完成翻译的语言会触发特殊UI状态
