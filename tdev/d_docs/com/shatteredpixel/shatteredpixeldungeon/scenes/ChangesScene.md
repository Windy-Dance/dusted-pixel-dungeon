# ChangesScene 类文档

## 概述
`ChangesScene` 是 Shattered Pixel Dungeon 的更新日志场景类，继承自 `PixelScene`。它显示游戏的所有版本变更历史，按主要版本号分类展示详细的更新内容。

## 类结构
```java
public class ChangesScene extends PixelScene
```

## 主要功能

### 1. 版本历史展示
- **版本分类**: 按主要版本号分组（v0.1.x, v0.2.x, ..., v3.x）
- **时间倒序**: 最新版本显示在列表顶部
- **详细变更**: 每个版本包含完整的新增功能、修复和改进列表
- **图标标识**: 使用不同图标区分变更类型（新增、修复、改进等）

### 2. 变更类型分类
使用专门的变更信息类处理不同类型的变更：
- **新内容**: 新增物品、怪物、关卡等
- **平衡调整**: 数值和机制的平衡性修改  
- **Bug修复**: 各种问题的修复记录
- **界面改进**: UI/UX相关的优化
- **性能优化**: 游戏性能和稳定性的提升

### 3. 动态内容加载
- **模块化设计**: 每个版本的变更记录在独立的类中定义
- **按需创建**: 变更详情窗口只在用户点击时创建
- **内存优化**: 避免一次性加载所有版本数据

### 4. 用户交互
- **版本选择**: 点击版本按钮显示该版本的完整变更日志
- **标签式界面**: 支持标签页式的变更浏览体验 (`WndChangesTabbed`)
- **返回导航**: 退出按钮返回主菜单

## 技术架构

### 变更列表系统
- **ChangeInfo**: 变更信息的基础数据结构
- **版本专用类**: 每个主要版本有对应的变更类 (`v0_1_X_Changes`, `v0_2_X_Changes`, 等)
- **动态实例化**: 使用反射或工厂模式创建变更列表

### UI组件集成
- **ScrollPane**: 支持长列表的滚动查看
- **StyledButton**: 统一的版本选择按钮样式
- **Icons**: 使用标准图标系统

## 生命周期方法

### create()
场景初始化：
1. 创建标题背景和基础UI元素
2. 动态创建所有版本的变更按钮
3. 根据静态变量 `changesSelected` 高亮选中版本
4. 添加退出按钮和导航控件
5. 执行淡入动画

### update()
每帧更新逻辑：
- 处理未读变更的视觉效果（如果有）
- 更新各种动态UI元素状态

## 数据管理

### 版本组织
- **主要版本**: v0.x, v1.x, v2.x, v3.x 等大版本
- **次要版本**: 在主要版本内包含所有小版本变更
- **最新版本**: 通常作为默认选中项

### 变更格式
每个变更条目包含：
- **变更类型**: 使用不同颜色和图标标识
- **变更描述**: 详细的文字说明
- **影响范围**: 如果适用，说明影响的游戏系统

### 本地化支持
- 所有变更文本通过消息系统本地化
- 支持多语言的变更日志显示
- 英文作为默认后备语言

## 用户体验设计

### 信息密度
- **概览友好**: 列表视图提供快速概览
- **详情深入**: 点击后提供完整详细信息
- **层次清晰**: 不同类型的变更清晰区分

### 导航便利
- **快速跳转**: 直接跳转到特定版本
- **连续阅读**: 支持从新到旧的连续浏览
- **返回容易**: 简单的返回操作

### 视觉反馈
- **选中状态**: 当前选中的版本高亮显示
- **新内容标识**: 新增内容使用特殊颜色
- **修复标识**: Bug修复使用不同的视觉样式

## 特殊功能

### 更新集成
- **自动标记**: 与更新检查系统集成
- **新版本提示**: 有新更新时在主菜单显示提示
- **变更预览**: 更新前可预览变更内容

### 调试支持
- 开发者可以快速访问特定版本的变更日志
- 支持测试不同版本的变更显示效果

## 性能考虑
- **延迟加载**: 变更详情按需加载
- **内存管理**: 避免同时保持多个大型变更列表
- **渲染优化**: 使用高效的列表渲染技术
