# Stylus.java - 刻写笔

## 概述
`Stylus` 类代表刻写笔道具，用于将普通护甲升级为带有特殊符文（Glyph）的强化护甲。这个可堆叠物品允许玩家在未被诅咒的护甲上刻写符文，提供额外的防护和特殊能力。

## 核心功能

### 护甲刻写系统
- **inscribe(Armor armor)**: 主要刻写方法，执行完整的刻写流程
- **条件检查**: 
  - 护甲必须已识别（cursedKnown = true）
  - 护甲不能被诅咒或带有诅咒符文
- **符文生成**: 调用armor.inscribe()生成随机符文

### 使用机制
- **目标选择**: 必须选择背包中的护甲进行刻写
- **资源消耗**: 每次使用消耗1支刻写笔
- **时间成本**: 需要2回合完成刻写过程

### 视觉和音效反馈
- **粒子效果**: 紫色爆发粒子效果（PurpleParticle.BURST）
- **操作动画**: 显示英雄刻写护甲的动作
- **符文显示**: Enchanting.show()显示符文生成效果
- **音效**: 燃烧音效（Assets.Sounds.BURNING）

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.STYLUS` 图像
- **stackable = true**: 支持堆叠多支刻写笔
- **defaultAction = AC_INSCRIBE**: 默认动作为刻写
- **bones = true**: 可以出现在英雄遗骸中
- **isUpgradable()**: 返回false（不可升级）
- **isIdentified()**: 返回true（始终已识别）

### 经济价值
- **value()**: 每单位价值30金币
- **统计追踪**: Catalog系统记录使用次数

## 交互逻辑

### 动作系统
- **AC_INSCRIBE**: 刻写动作，打开物品选择器
- **itemSelector**: 护甲专用选择器
  - 首选背包（Backpack）
  - 只允许选择护甲物品

### 错误处理
- **未识别护甲**: 显示"identify"提示，要求先识别护甲
- **被诅咒护甲**: 显示"cursed"提示，无法对被诅咒护甲进行刻写
- **成功提示**: 显示"inscribed"提示表示刻写成功

### 安全机制
- **诅咒检查**: 严格防止对被诅咒物品进行操作
- **状态验证**: 确保只有合适的护甲才能被刻写
- **资源管理**: 自动从背包中扣除使用的刻写笔

## 使用场景

### 战略应用
- **护甲强化**: 在关键战斗前强化护甲
- **符文实验**: 尝试不同的符文组合寻找最佳效果
- **资源投资**: 将刻写笔作为长期护甲投资

### 游戏进度
- **中期获取**: 通常在游戏中期获得，用于强化现有装备
- **后期优化**: 在游戏后期优化最终护甲配置
- **风险规避**: 避免在重要护甲上冒险使用未知卷轴

## 注意事项
1. 刻写后的符文是永久性的，无法移除或更改
2. 符文效果完全随机，可能产生积极或消极效果
3. 刻写过程需要2回合，在战斗中使用需谨慎考虑时机
4. 被诅咒的护甲无法被刻写，需要先移除诅咒
5. 视觉效果强调了这是一个魔法强化过程
6. 与护甲系统的符文机制深度集成，影响整体装备策略
