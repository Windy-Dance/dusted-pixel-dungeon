# LiquidMetal.java - 液态金属

## 概述
`LiquidMetal` 类代表液态金属道具，用于修复和强化投掷武器（MissileWeapon）。这个可堆叠物品可以通过制作系统从不需要的投掷武器转化而来，并用于恢复投掷武器的耐久度或增加数量。

## 核心功能

### 投掷武器修复系统
- **耐久度修复**: 计算投掷武器当前损失的耐久度并进行修复
- **数量补充**: 当投掷武器数量未达到默认值时，可以增加武器数量
- **智能计算**: 
  - 基础用量：5 × (武器等级 + 1)
  - 等级加成：每级增加35%用量效率
  - 耐久度转换：100点耐久度 = maxToUse单位液态金属

### 使用机制
- **目标选择**: 只能选择投掷武器（除飞镖Dart外）
- **用量计算**: 
  - 计算需要修复的耐久度百分比
  - 向上取整确定实际使用量
  - 自动从背包中扣除使用的液态金属
- **特殊情况处理**:
  - 武器已满耐久：显示"already_fixed"提示
  - 武器数量已满：显示"already_fixed"提示

### 制作系统集成
- **Recipe内部类**: 提供从投掷武器制作液态金属的配方
- **制作条件**: 
  - 输入：单个未被诅咒的投掷武器
  - 成本：3点能量水晶
- **产量计算**: 
  - 基础产量：5 × (武器等级 + 1)
  - 数量修正：根据武器默认数量调整
  - 等级加成：最多5级加成，每级35%
  - 耐久度加成：额外0.0075 × 当前耐久度

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.LIQUID_METAL` 图像
- **stackable = true**: 支持堆叠
- **defaultAction = AC_APPLY**: 默认动作为应用
- **bones = true**: 可以出现在英雄遗骸中
- **isUpgradable()**: 返回false（不可升级）
- **isIdentified()**: 返回true（始终已识别）

### 经济价值
- **value()**: 每单位价值1金币（等于数量）
- **统计追踪**: Catalog系统记录使用次数

## 交互逻辑

### 动作系统
- **AC_APPLY**: 应用动作，打开物品选择器
- **itemSelector**: 投掷武器专用选择器
  - 首选魔法枪套（MagicalHolster）
  - 排除飞镖（Dart）类型

### 视觉和音效反馈
- **操作动画**: 显示英雄使用液态金属的动作
- **粒子效果**: 光粒子效果（Speck.LIGHT），产生10个光粒子
- **音效**: 喝水音效（Assets.Sounds.DRINK）
- **成功提示**: 显示使用数量的提示信息

### 投掷行为
- **特殊地形**: 投掷到井（WELL）或深坑（pit）中时正常掉落
- **普通地面**: 在普通地面破碎，播放破碎音效和银色飞溅效果（0xBFBFBF）

## 技术实现细节

### 耐久度计算
- **durabilityPerMetal**: 每单位液态金属提供的耐久度
- **percentDurabilityLost**: 计算已损失的耐久度百分比
- **向上取整**: 确保修复效果足够显著

### 数量管理
- **defaultQuantity检查**: 确保不会超过武器的默认数量
- **耐久度阈值**: 修复后确保武器能达到最大使用次数

### 安全机制
- **舍入误差处理**: 减去微小值避免浮点计算误差
- **边界检查**: 确保不会出现负数或超限情况

## 使用场景

### 资源管理
- **武器维护**: 在长时间战斗后修复投掷武器
- **制作转化**: 将不需要的投掷武器转化为液态金属
- **战术准备**: 在重要战斗前确保投掷武器处于最佳状态

### 经济策略
- **成本效益**: 3点能量水晶换取大量液态金属
- **资源循环**: 建立投掷武器→液态金属→修复/补充的循环

## 注意事项
1. 液态金属不能用于飞镖（Dart），因为飞镖有特殊的丢失机制
2. 制作时会自动标记该武器集为已升级（通过UpgradedSetTracker）
3. 耐久度修复和数量补充是互斥的，优先处理数量不足的情况
4. 视觉效果使用光粒子强调了这是一个修复和强化过程
5. 与MissileWeapon系统的深度集成确保了耐久度计算的准确性
6. 经济价值极低，鼓励玩家大量使用而不是囤积
