# BrokenSeal.java - 破碎封印

## 概述
`BrokenSeal` 类代表破碎封印道具，是战士职业的专属物品，可以附着在护甲上提供特殊能力。封印可以携带符文（Glyph），并为英雄提供战斗中的护盾保护。

## 核心功能

### 符文系统
- **glyph**: 存储附着的护甲符文（Armor.Glyph）
- **canTransferGlyph()**: 检查是否可以转移符文
  - 需要Runic Transference天赋支持
  - 天赋等级1：只能转移常见和不常见符文
  - 天赋等级2：可以转移所有符文
- **setGlyph()/getGlyph()**: 设置和获取符文

### 护盾机制
- **WarriorShield**: 内部类，实现战士护盾Buff
- **maxShield()**: 计算最大护盾值（3 + 2×护甲等级 + Iron Will天赋点数）
- **activate()**: 激活护盾并开始冷却计时
- **智能护盾管理**: 
  - 无敌人时自动消失护盾并返还部分冷却时间
  - 战斗中保持护盾激活

### 附着系统
- **affixToArmor()**: 将封印附着到指定护甲上
- **兼容性检查**: 
  - 护甲必须已识别（cursedKnown = true）
  - 被诅咒的护甲只能附着诅咒符文
- **符文选择**: 当目标护甲和封印都有符文时，显示选择界面

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.SEAL` 图像
- **unique = true**: 作为唯一物品，死亡后保留
- **bones = false**: 不会出现在英雄遗骸中
- **cursedKnown = levelKnown = true**: 始终已识别和已知等级
- **isUpgradable()**: 仅在等级为0时可升级（模拟卷轴升级效果）

### 视觉表现
- **glowing()**: 显示符文的发光效果
- **name()**: 如果有符文，显示符文名称+封印名称
- **info()**: 显示符文描述信息

## 交互逻辑

### 动作系统
- **AC_AFFIX**: 附着动作，选择目标护甲
- **AC_INFO**: 信息窗口动作（主要用于快捷栏教程）
- **armorSelector**: 护甲选择器，只允许选择护甲物品

### UI集成
- **BuffIndicator**: 显示护盾状态和冷却时间
  - 活跃护盾：显示护盾数值
  - 冷却中：显示倒计时
  - 负冷却：显示特殊状态
- **图标着色**: 
  - 冷却中：降低亮度
  - 负冷却：反色显示

## WarriorShield Buff详解

### 状态管理
- **shieldUsePriority = 2**: 护盾使用优先级
- **detachesAtZero = false**: 护盾归零时不自动移除
- **冷却机制**: 
  - 基础冷却：150回合
  - 无战斗时返还最多50%冷却时间
  - 受Iron Will天赋影响

### 天赋集成
- **IRON_WILL**: 影响护盾最大值和Metamorphed状态下的基础值
- **RUNIC_TRANSFERENCE**: 控制符文转移能力
- **HOLD_FAST**: 影响护盾衰减因子

### 数据持久化
- **storeInBundle()/restoreFromBundle()**: 保存和恢复护盾状态
- **向后兼容**: 处理3.1版本前的护盾数据

## 使用场景
- **战士职业核心**: 战士职业的标志性能力
- **护甲强化**: 为任意护甲添加额外保护
- **符文携带**: 在不同护甲间转移符文效果
- **战术选择**: 根据战斗情况选择合适的符文组合

## 注意事项
1. 破碎封印是战士职业专属，其他职业无法有效使用
2. 符文转移受天赋等级限制，需要合理规划天赋点数
3. 护盾机制鼓励积极战斗，避免长时间脱离战斗
4. 负冷却状态提供特殊战术选择（如快速连续使用）
5. 与护甲系统深度集成，需要理解护甲符文机制才能发挥最大效果
