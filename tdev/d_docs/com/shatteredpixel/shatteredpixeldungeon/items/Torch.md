# Torch.java - 火把

## 概述
`Torch` 类代表火把道具，用于为英雄提供临时的光源效果。这个可堆叠物品可以激活Light Buff，在指定持续时间内照亮周围的黑暗区域，帮助玩家在黑暗环境中进行探索和战斗。

## 核心功能

### 光源系统
- **Light Buff**: 应用Light Buff到英雄身上，持续时间为Light.DURATION
- **视野扩展**: 在黑暗区域中提供额外的可视范围
- **临时效果**: 效果持续时间固定，无法叠加或延长

### 使用机制
- **即时激活**: 使用后立即生效，无需目标选择
- **资源消耗**: 每次使用消耗1支火把
- **时间成本**: 需要1回合完成点燃过程

### 视觉和音效反馈
- **粒子效果**: 火焰粒子效果（FlameParticle.FACTORY），产生3个火焰粒子
- **操作动画**: 显示英雄点燃火把的动作
- **音效**: 燃烧音效（Assets.Sounds.BURNING）
- **中心发射器**: 粒子从英雄中心位置发射

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.TORCH` 图像
- **stackable = true**: 支持堆叠多个火把
- **defaultAction = AC_LIGHT**: 默认动作为点火
- **isUpgradable()**: 返回false（不可升级）
- **isIdentified()**: 返回true（始终已识别）

### 经济价值
- **value()**: 每单位价值8金币
- **统计追踪**: Catalog系统记录使用次数

## 交互逻辑

### 动作系统
- **AC_LIGHT**: 点火动作，直接应用光源效果
- **actions()**: 只显示LIGHT动作（无丢弃、投掷等其他选项，但继承自Item的默认动作）

### 使用流程
1. **消耗资源**: 从背包中移除1支火把
2. **应用效果**: 为英雄添加Light Buff
3. **播放反馈**: 播放音效和粒子效果
4. **时间消耗**: 英雄花费1回合完成点燃过程

### 安全机制
- **自动管理**: Buff系统自动处理效果的开始和结束
- **资源验证**: 自动检查背包中是否有足够的火把数量

## 使用场景

### 探索辅助
- **黑暗区域**: 在没有自然光源的区域提供照明
- **陷阱探测**: 帮助发现隐藏的陷阱和危险
- **敌人定位**: 在黑暗中定位敌人的位置

### 战斗支援
- **视野优势**: 在战斗中保持对敌人的视野
- **战术照明**: 临时照亮关键区域进行战术决策
- **紧急情况**: 在意外进入黑暗区域时的应急措施

## 注意事项
1. Light Buff的持续时间是固定的，无法通过天赋或其他方式延长
2. 火把效果与环境光源叠加，不会覆盖现有的照明效果
3. 在已经有充足光源的区域使用火把可能造成资源浪费
4. 粒子效果强调了这是一个火焰相关的魔法/物理效果
5. 与Buff系统的集成确保了效果的正确管理和清理
6. 经济价值较低，鼓励玩家在需要时大量使用
