# Honeypot.java - 蜂蜜罐

## 概述
`Honeypot` 类代表蜂蜜罐道具，可以释放一只蜜蜂（Bee）作为临时盟友协助战斗。蜂蜜罐使用后会破碎成ShatteredPot（破碎罐），用于追踪和管理召唤出的蜜蜂。

## 核心功能

### 蜜蜂召唤系统
- **shatter(Char owner, int pos)**: 主要召唤方法
  - 在指定位置生成一只蜜蜂
  - 蜜蜂会继承当前地牢深度的强度缩放
  - 设置蜜蜂的"家"位置和拥有者信息
- **智能位置选择**:  
  - 如果目标位置有角色，会尝试在相邻空位生成蜜蜂
  - 如果没有合适位置，则不释放蜜蜂并保持物品完整

### 双重使用方式
- **投掷使用**: 将蜂蜜罐投掷到目标位置，自动破碎并召唤蜜蜂
- **直接破碎**: 使用SHATTER动作在当前位置破碎蜂蜜罐

### 破碎罐系统 (ShatteredPot)
- **状态追踪**: 破碎罐记录蜜蜂的"家"位置和拥有者
- **动态更新**: 当破碎罐被拾取、丢弃、投掷或销毁时，自动更新关联蜜蜂的状态
- **多蜜蜂支持**: 支持堆叠的破碎罐管理多个蜜蜂

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.HONEYPOT` 图像
- **stackable = true**: 支持堆叠多个蜂蜜罐
- **defaultAction = AC_THROW**: 默认动作为投掷
- **usesTargeting = true**: 投掷时需要目标选择
- **isUpgradable()**: 返回false（不可升级）
- **isIdentified()**: 返回true（始终已识别）

### 经济价值
- **value()**: 每单位价值30金币
- **统计追踪**: Catalog系统记录使用次数

## 交互逻辑

### 动作系统
- **AC_SHATTER**: 直接破碎动作，在当前位置召唤蜜蜂
- **AC_THROW**: 投掷动作，将蜂蜜罐投掷到目标位置

### 视觉和音效反馈
- **破碎动画**: 
  - 播放破碎音效（Assets.Sounds.SHATTER）
  - 显示黄色飞溅效果（0xffd500）
- **蜜蜂生成**:
  - 播放蜜蜂音效（Assets.Sounds.BEE）
  - 渐显动画（alpha从0到1）
  - 推动动画（如果需要移动位置）

## ShatteredPot详解

### 生命周期管理
- **pickupPot()**: 拾取时通知蜜蜂新的拥有者
- **dropPot()**: 丢弃时更新蜜蜂的"家"位置
- **movePot()**: 移动时更新蜜蜂的"家"位置  
- **destroyPot()**: 销毁时移除蜜蜂的"家"信息

### 蜜蜂查找机制
- **findBees(int potPos)**: 查找与指定位置关联的蜜蜂
- **findBees(Char potHolder)**: 查找与指定拥有者关联的蜜蜂
- **数量限制**: 最多返回与破碎罐数量相等的蜜蜂数量

### 安全机制
- **敌人限制**: 只更新敌对蜜蜂的状态（bee.alignment == Char.Alignment.ENEMY）
- **状态验证**: 确保只处理有效的蜜蜂实例

## 使用场景
- **战斗支援**: 释放蜜蜂攻击敌人，提供额外伤害输出
- **探索辅助**: 蜜蜂可以触发陷阱或探测危险区域
- **战术配合**: 与其他召唤物或技能配合使用
- **资源利用**: 将多余的蜂蜜罐转换为战斗优势

## 注意事项
1. 蜜蜂的强度会根据当前地牢深度进行适当缩放
2. 蜜蜂的生命周期独立于蜂蜜罐，即使破碎罐被销毁，蜜蜂仍会继续存在
3. 破碎罐的堆叠机制允许同时管理多个蜜蜂
4. 蜜蜂的"家"概念主要用于AI行为，影响其移动和攻击模式
5. 投掷到深坑（pit）中时，蜂蜜罐不会破碎而是正常掉落
