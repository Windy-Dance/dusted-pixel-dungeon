# LostBackpack.java - 失落背包

## 概述
`LostBackpack` 类代表失落背包道具，用于在英雄失去所有物品后恢复背包功能。当英雄受到特定诅咒效果（如被剥夺装备）时，这个唯一物品会出现在地图上，拾取后可以重新激活所有保留物品的功能。

## 核心功能

### 背包恢复机制
- **移除丢失状态**: 拾取时自动移除 `LostInventory` Buff
- **物品重新激活**: 遍历英雄当前拥有的所有物品，重新激活其功能
- **状态清理**: 清理物品的临时保留标志（keptThoughLostInvent）

### 特殊物品处理
- **可装备物品**: 重新调用activate()方法激活装备效果
- **CloakOfShadows**: 如果英雄拥有Light Cloak天赋，重新激活披风
- **HolyTome**: 如果英雄拥有Light Reading天赋，重新激活圣典  
- **魔杖系统**: 
  - 在MagicalHolster中的魔杖使用特殊充能倍率
  - 普通魔杖使用标准充能机制
- **法师法杖**: 重新应用魔杖充能Buff

### 英雄状态更新
- **生命值更新**: 调用updateHT(false)更新英雄最大生命值
- **视觉刷新**: 更新快捷栏显示和英雄护甲外观
- **音效反馈**: 播放露珠音效表示成功恢复

## 物品属性

### 基础设置
- 使用 `ItemSpriteSheet.BACKPACK` 图像
- **unique = true**: 作为唯一物品，确保不会重复生成
- **isUpgradable()**: 继承自Item，默认返回true（但实际不可升级）
- **isIdentified()**: 继承自Item，默认需要识别（但通常已知）

### 日志集成
- **Notes系统**: 拾取后自动移除LOST_PACK地标记录
- **Journal追踪**: 作为重要游戏事件被记录

## 使用场景

### 触发条件
- **Curse Infusion**: 被诅咒融合效果影响后失去所有物品
- **Death's Door**: 死亡之门挑战失败后的惩罚
- **其他丢失机制**: 任何导致英雄失去背包的特殊情况

### 战略意义
- **游戏延续**: 允许玩家在极端情况下继续游戏
- **物品保留**: 确保重要的保留物品（如神器、祝福物品）不丢失功能
- **进度保护**: 保护玩家的游戏进度和投入

## 注意事项
1. 失落背包只会在英雄实际失去物品的情况下出现
2. 拾取后会立即恢复所有保留物品的完整功能
3. 与MagicalHolster等特殊容器的集成确保了魔杖系统的正确性
4. 天赋效果（如Light Cloak、Light Reading）会被正确重新应用
5. 视觉和音效反馈帮助玩家理解背包已成功恢复
6. 日志系统的集成帮助玩家追踪这个重要事件

## 技术实现细节
- **遍历机制**: 遍历hero.belongings获取所有当前物品
- **条件判断**: 通过keptThroughLostInventory()区分临时保留和永久保留物品
- **安全激活**: 只对支持activate()方法的物品调用该方法
- **性能优化**: 单次遍历完成所有激活操作，避免重复计算
