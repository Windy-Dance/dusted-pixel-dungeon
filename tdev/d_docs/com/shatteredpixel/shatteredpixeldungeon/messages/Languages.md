# Languages.java - 语言支持系统

## 概述
`Languages` 枚举定义了《破碎像素地牢》支持的所有语言，包括它们的本地名称、ISO代码、翻译状态、贡献者信息和特殊处理逻辑。该系统为游戏的多语言支持提供基础框架。

## 语言状态定义

### 状态级别
- **X_UNFINISH** - 翻译进行中（约80-99%完成）
- **__UNREVIEW** - 翻译完成但未审校（100%翻译）
- **O_COMPLETE** - 翻译完成且已审校（100%完成）

### 贡献者角色
- **Translators** - 主要翻译人员
- **Reviewers** - 审校人员（通常也是翻译人员）
- 特殊情况下，某些语言可能有多个贡献者

## 技术实现

### 枚举结构
每个语言枚举项包含以下字段：
```java
Languages(
    String name,           // 本地语言名称
    String code,           // ISO 639-1/BCP 47 语言代码  
    Status status,         // 翻译状态
    String[] reviewers,    // 审校人员列表
    String[] translators   // 翻译人员列表
)
```

### 特殊处理逻辑
#### 印尼语 locale 兼容性
```java
if (code.equals("id")) {
    return Locale.forLanguageTag("in");
}
```
由于历史原因，印尼语使用 `in` 作为 locale 标识符，但标准代码是 `id`。

#### 回退机制
- 英语作为默认回退语言
- 当请求的语言不存在时自动回退到英语
- 支持部分翻译的渐进式体验

### 语言匹配方法
```java
// 通过 Locale 匹配语言
public static Languages matchLocale(Locale locale)

// 通过语言代码匹配  
public static Languages matchCode(String code)
```

这些方法处理各种边缘情况：
- 大小写不敏感匹配
- 带区域的代码匹配（如 `zh-Hans` → `zh`）
- 无效代码的安全处理

## 集成与使用

### 游戏启动流程
1. 检测系统语言设置
2. 尝试匹配支持的语言
3. 如果不支持则使用英语
4. 初始化对应语言的 Messages Bundle

### 动态语言切换
- 支持运行时语言切换
- 切换后重新加载所有 UI 文本
- 保持游戏状态不受影响

### 贡献者识别
- 在游戏的致谢页面显示贡献者
- 根据翻译状态显示不同的完成度标识
- 支持贡献者信息的动态更新

## 性能考虑

### 内存效率
- 枚举实例在编译时创建，运行时无额外开销
- 字符串数组共享内存
- 避免重复的字符串创建

### 查找优化  
- 使用高效的字符串比较
- 缓存常用的匹配结果
- 减少不必要的 locale 创建

### 启动性能
- 语言检测在启动时一次性完成
- 避免重复的语言匹配操作
- 延迟加载非必要语言资源

## 扩展性和维护

### 添加新语言
1. 在枚举中添加新的语言项
2. 提供完整的翻译文件（.properties）
3. 更新贡献者信息和状态
4. 测试特殊的字符集和格式化规则

### 翻译更新流程
1. 翻译者提交新的翻译文件
2. 审校者验证翻译质量  
3. 更新枚举中的状态和贡献者信息
4. 发布新版本包含更新

### 国际化最佳实践
- 使用 Unicode 标准字符集
- 考虑文本方向（虽然当前游戏只支持 LTR）
- 处理不同语言的文本长度变化
- 支持复杂的格式化规则（数字、日期等）
