## 2. 文件职责说明

### 核心职责
GrassyGraveRoom 类负责创建一个长满草的坟墓类型的房间，该房间内部填充草地(GRASS)地形，并在其中随机生成多个墓冢(TOMB)堆，其中一个包含随机物品，其余包含金币。

### 系统定位
作为标准房间的一种具体实现，GrassyGraveRoom 在地牢关卡生成过程中被用于创建具有墓地特征的房间，为玩家提供探索和收集资源的机会。

### 不负责什么
- 不负责关卡的整体布局规划（由 Level 和 Room 相关类处理）
- 不负责地形绘制的具体实现（由 Painter 类处理）
- 不负责具体的怪物生成逻辑

## 7. 方法详解

### merge()

**可见性**：public

**是否覆写**：是，覆写自 Room

**方法职责**：处理与其他房间的合并逻辑

**参数**：
- `l` (Level)：当前关卡
- `other` (Room)：要合并的另一个房间
- `merge` (Rect)：合并区域
- `mergeTerrain` (int)：合并后的地形类型

**返回值**：void

**前置条件**：无

**副作用**：
- 修改 level.map 数组中的地形数据

**核心实现逻辑**：
如果合并地形是空地(Terrain.EMPTY)，并且另一个房间是 GrassyGraveRoom 或 PlantsRoom 类型，则将合并区域设置为草地(Terrain.GRASS)；否则使用父类的默认合并逻辑。

**边界情况**：当与其他类型房间合并时，使用标准合并逻辑

### paint()

**可见性**：public

**是否覆写**：是，覆写自 StandardRoom

**方法职责**：将房间实际绘制到关卡地图上

**参数**：
- `level` (Level)：要绘制到的关卡

**返回值**：void

**前置条件**：房间的边界已经确定

**副作用**：
- 修改 level.map 数组中的地形数据
- 在关卡中创建物品堆(Heap)
- 设置连接门的类型

**核心实现逻辑**：
1. 使用 Painter.fill() 方法填充外层墙壁(Terrain.WALL)
2. 将所有连接门设置为常规门类型(Door.Type.REGULAR)
3. 使用 Painter.fill() 方法填充内层草地(Terrain.GRASS)
4. 计算墓冢数量：取房间宽度和高度的最大值除以2
5. 随机选择一个墓冢位置作为特殊物品位置
6. 根据房间形状（宽大于高或高大于宽）确定墓冢的排列方式：
   - 如果宽度大于高度：墓冢沿垂直方向排列，水平位置随机
   - 如果高度大于宽度：墓冢沿水平方向排列，垂直位置随机
7. 为每个墓冢位置创建物品堆：
   - 特殊位置：放置随机物品(Generator.random())
   - 其他位置：放置随机数量的金币(new Gold().random())
8. 所有物品堆的类型都设置为墓冢(Heap.Type.TOMB)

**边界情况**：
- 当房间尺寸很小时，墓冢数量可能很少
- 墓冢排列会根据房间的长宽比例自动调整方向

## 12. 开发注意事项

### 状态依赖
- 依赖于房间边界已正确设置
- 依赖于关卡的 map 数组已初始化
- 依赖于物品生成系统的正常工作

### 生命周期耦合
- 必须在关卡生成流程的正确阶段调用 paint() 方法
- merge() 方法必须在房间合并时正确调用

### 常见陷阱
- 如果房间尺寸过小，可能导致墓冢数量过少或排列不合理
- 与特定房间类型（GrassyGraveRoom, PlantsRoom）合并时会产生特殊的草地连接效果
- 随机物品生成依赖于 Generator 类的状态

## 13. 修改建议与扩展点

### 适合扩展的位置
- 墓冢数量的计算公式可以根据游戏平衡需求调整
- 特殊物品的类型可以修改以适应不同的游戏难度

### 不建议修改的位置
- merge() 方法的特殊合并逻辑不应移除，这是设计特色
- 墓冢类型应保持为 TOMB，以确保正确的视觉效果和游戏机制

### 重构建议
当前实现简洁明了，无需重构。
