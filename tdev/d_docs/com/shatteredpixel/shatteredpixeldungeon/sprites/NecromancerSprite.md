# NecromancerSprite - 死灵法师精灵

## 概述
`NecromancerSprite` 类是死灵法师（Necromancer）敌人的视觉表现类，继承自 `MobSprite`。该精灵负责处理死灵法师的所有动画效果，包括空闲、奔跑、攻击（施法）、充电和死亡状态。

## 动画帧
死灵法师精灵使用 16x16 像素的纹理贴图，包含以下动画序列：

- **空闲动画 (idle)**: 帧序列 [0, 0, 0, 1, 0, 0, 0, 0, 1]，循环播放
- **奔跑动画 (run)**: 帧序列 [0, 0, 0, 2, 3, 4]，8 FPS，循环播放  
- **施法动画 (zap)**: 帧序列 [5, 6, 7, 8]，10 FPS，非循环
- **充电动画 (charging)**: 帧序列 [7, 8]，5 FPS，循环播放
- **死亡动画 (die)**: 帧序列 [9, 10, 11, 12]，10 FPS，非循环
- **攻击动画 (attack)**: 复制施法动画

## 特殊功能

### 召唤系统
死灵法师精灵具有特殊的召唤骨骼效果：

- **召唤粒子效果**: 当死灵法师进行召唤时，在目标位置创建骨骼粒子发射器 (`summoningBones`)
- **视觉可见性**: 粒子效果仅在玩家视野范围内可见
- **音效**: 完成召唤时播放骨骼音效 (`Assets.Sounds.BONES`)
- **取消机制**: 提供 `cancelSummoning()` 方法来取消正在进行的召唤

### 动画状态管理
- **充电状态**: 通过 `charge()` 方法切换到充电动画
- **召唤完成**: `finishSummoning()` 方法处理召唤完成后的清理工作
- **动画回调**: 重写 `onComplete()` 方法来处理施法动画完成后的逻辑

## 主要方法

### `link(Char ch)`
- 链接角色实体
- 如果角色正在召唤，则自动开始施法动画

### `update()`
- 更新精灵状态
- 管理召唤粒子发射器的可见性

### `die()` 和 `kill()`
- 处理死亡逻辑
- 清理召唤粒子效果

### `zap(int cell)`
- 执行施法动画
- 初始化召唤粒子效果
- 播放充电音效

### `onComplete(Animation anim)`
- 处理动画完成回调
- 根据死灵法师的召唤状态决定后续动作（继续充电或完成施法）

## 技术细节
- **纹理资源**: 使用 `Assets.Sprites.NECRO` 纹理
- **特效系统**: 利用 `CellEmitter` 和 `Speck` 创建粒子效果
- **音频系统**: 通过 `Sample.INSTANCE.play()` 播放音效
- **视野系统**: 与 `Dungeon.level.heroFOV` 集成以控制特效可见性

## 使用场景
此精灵专门用于死灵法师敌人，处理其独特的召唤行为和相应的视觉反馈，为玩家提供清晰的战斗状态指示。
