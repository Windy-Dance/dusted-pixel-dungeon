# TenguSprite - 天狗精灵

## 概述
`TenguSprite` 类是天狗（Tengu）Boss敌人的视觉表现类，继承自 `MobSprite`。该精灵具有复杂的移动系统、远程攻击能力和独特的同步机制，专门用于表现游戏中的重要Boss角色。

## 动画帧
天狗精灵使用天狗纹理贴图 (`Assets.Sprites.TENGU`)，使用 14x16 像素的帧大小：

- **空闲动画 (idle)**: 帧序列 [0, 0, 0, 1]，2 FPS，循环播放
- **奔跑动画 (run)**: 帧序列 [2, 3, 4, 5, 0]，15 FPS，非循环  
- **攻击动画 (attack)**: 帧序列 [6, 7, 7, 0]，15 FPS，非循环
- **施法动画 (zap)**: 复制攻击动画
- **死亡动画 (die)**: 帧序列 [8, 9, 10, 10, 10, 10, 10, 10]，8 FPS，非循环

## 特殊功能

### 移动系统
- **自定义移动**: 重写 `move()` 方法实现特殊的移动逻辑
- **水波纹效果**: 在水中移动时创建水波纹特效 (`GameScene.ripple()`)
- **方向控制**: 使用 `turnTo()` 确保正确的面向方向
- **移动状态**: 通过 `isMoving` 标志跟踪移动状态

### 远程攻击
- **手里剑投掷**: 非近战攻击时投掷天狗手里剑 (`TenguShuriken`)
- **导弹精灵**: 使用 `MissileSprite` 创建飞行的投掷物效果
- **回调机制**: 通过回调函数通知攻击完成

### 同步机制
- **线程同步**: 使用 `synchronized` 块和 `notifyAll()` 实现状态同步
- **移动等待**: 其他操作需要等待移动完成
- **状态管理**: 精确控制 `isMoving` 标志的状态转换

### 动画控制
- **暂停更新**: 在暂停状态下仍更新非循环动画
- **初始状态**: 构造函数中直接播放奔跑动画并设置移动状态

## 主要方法

### `move(int from, int to)`
- 处理自定义移动逻辑
- 触发水波纹特效
- 管理移动状态标志

### `attack(int cell)`
- 判断攻击距离
- 近战攻击使用标准攻击动画
- 远程攻击投掷手里剑

### `play(Animation anim)`
- 监控移动状态变化
- 触发同步通知

### `onComplete(Animation anim)`
- 处理奔跑动画完成后的状态转换
- 重置为空闲状态并触发同步

### `update()`
- 在暂停状态下继续更新非循环动画

## 技术细节
- **纹理资源**: 使用 `Assets.Sprites.TENGU` 纹理贴图
- **嵌套类**: 包含 `TenguShuriken` 内部类用于投掷物
- **物品系统**: 投掷物使用 `ItemSpriteSheet.SHURIKEN` 图像
- **场景交互**: 与 `GameScene` 和 `Dungeon.level` 系统深度集成
- **线程安全**: 使用同步机制确保状态一致性

## 使用场景
此精灵专门用于天狗Boss，通过复杂的移动系统、远程手里剑攻击、水波纹特效和精确的状态同步，准确表现了作为重要Boss的高机动性和多样的攻击方式，为玩家提供具有挑战性的战斗体验。
