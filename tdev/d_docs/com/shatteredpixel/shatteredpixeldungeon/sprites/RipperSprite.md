# RipperSprite - 撕裂者精灵

## 概述
`RipperSprite` 类是撕裂者（Ripper）敌人的视觉表现类，继承自 `MobSprite`。该精灵具有复杂的攻击模式切换、跳跃动画和预备攻击状态，专门用于表现敏捷且具有多种攻击方式的敌人。

## 动画帧
撕裂者精灵使用撕裂者纹理贴图 (`Assets.Sprites.RIPPER`)，使用 15x14 像素的帧大小：

- **空闲动画 (idle)**: 帧序列 [1, 0, 1, 2]，4 FPS，循环播放
- **奔跑动画 (run)**: 帧序列 [3, 4, 5, 6, 7, 8]，15 FPS，循环播放  
- **攻击动画 (attack)**: 帧序列 [0, 9, 10, 9]，12 FPS，非循环
- **刺击动画 (stab)**: 帧序列 [0, 9, 11, 9]，12 FPS，非循环
- **预备动画 (prep)**: 单帧 [9]，1 FPS，静态显示
- **跳跃动画 (leap)**: 单帧 [12]，1 FPS，静态显示
- **死亡动画 (die)**: 帧序列 [1, 13, 14, 15, 16]，15 FPS，非循环

## 特殊功能

### 攻击模式切换
- **交替攻击**: 使用 `alt` 标志在标准攻击和刺击之间交替
- **随机初始化**: 构造函数中随机决定初始攻击模式 (`Random.Int(2) == 0`)
- **流畅过渡**: 通过 `onComplete()` 方法确保刺击完成后正确返回到攻击完成逻辑

### 跳跃系统
- **预备状态**: 提供 `leapPrep()` 方法设置跳跃预备姿态
- **跳跃动画**: 重写 `jump()` 方法在跳跃时播放专用的跳跃动画
- **方向控制**: 预备状态下自动转向目标位置

### 动作设计
- **复杂空闲动画**: 包含4帧的循环序列，展现撕裂者的警觉状态
- **多阶段攻击**: 标准攻击和刺击两种不同的攻击模式，刺击使用不同的帧（帧11 vs 帧10）
- **详细死亡过程**: 5帧的死亡动画展现完整的倒下过程

## 主要方法

### `leapPrep(int cell)`
- 设置跳跃预备姿态
- 自动转向目标位置
- 播放预备动画

### `jump(int from, int to, float height, float duration, Callback callback)`
- 执行跳跃逻辑
- 播放专用的跳跃动画
- 调用父类跳跃实现

### `attack(int cell)`
- 执行攻击逻辑
- 根据 `alt` 标志决定使用标准攻击或刺击
- 切换攻击模式标志

### `onComplete(Animation anim)`
- 处理刺击动画完成后的状态转换
- 确保正确的回调链路

## 技术细节
- **纹理资源**: 使用 `Assets.Sprites.RIPPER` 纹理贴图
- **随机系统**: 集成 `Random.Int()` 实现攻击模式的随机初始化
- **动画控制**: 通过条件判断和标志位实现攻击模式切换
- **动作协调**: 预备、跳跃、攻击三种不同状态的协调管理

## 使用场景
此精灵专门用于撕裂者敌人，通过攻击模式切换、跳跃系统和多阶段攻击设计，准确表现了敏捷且具有多种攻击方式的敌人的特性，为玩家提供不可预测的战斗体验和战术挑战。
