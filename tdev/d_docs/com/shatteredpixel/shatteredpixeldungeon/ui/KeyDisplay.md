# KeyDisplay 类

## 概述
`KeyDisplay` 是 Shattered Pixel Dungeon 中用于显示钥匙计数的自定义视觉组件。它直接操作 OpenGL 顶点缓冲区，高效地渲染多个小钥匙图标。

## 功能特性
- **高效渲染**：使用 Vertexbuffer 直接操作 GPU，避免过多的 Image 对象
- **自适应布局**：根据可用空间自动调整钥匙排列和数量
- **类型排序**：按特定顺序显示不同类型的钥匙（旧钥匙、水晶钥匙、金钥匙、铁钥匙）
- **缺失标记**：用黑色钥匙表示错过的楼层钥匙
- **动态更新**：根据当前深度和日记记录实时更新显示

## 核心机制

### 钥匙类型映射
通过静态 `keyMap` 定义钥匙类型的显示顺序和槽位：
- **0**: 黑色钥匙（错过的楼层）
- **1**: 旧钥匙 (`WornKey.class`)
- **2**: 水晶钥匙 (`CrystalKey.class`)  
- **3**: 金色钥匙 (`GoldenKey.class`)
- **4**: 铁钥匙 (`IronKey.class`)

### 布局算法
- **最大行数**: `(height + 1) / 5`
- **每行最大数量**: `(width + 1) / 4`
- **短钥匙模式**: 当总高度不足以显示长钥匙时启用
- **自动截断**: 空间不足时优先保留数量更多的钥匙类型

### 顶点管理
- 使用 `FloatBuffer` 和 `Vertexbuffer` 直接操作图形数据
- 每个钥匙占用 16 个 float 值（4个顶点 × 4个属性）
- 纹理坐标从 `Assets.Interfaces.MENU_BTN` 精确定位

## 核心方法

### 状态管理
- **updateKeys()**: 
  - 从日记记录 (`Notes.KeyRecord`) 获取当前钥匙信息
  - 根据当前深度计算应显示的钥匙
  - 触发脏标记以重新生成顶点数据

- **keyCount()**: 返回当前显示的钥匙总数

### 渲染控制
- **draw()**: 
  - 检查脏标记并更新顶点数据
  - 绑定纹理并绘制所有钥匙
  - 应用颜色和透明度变换

- **updateVertices()**: 
  - 计算布局参数（行数、列数、短钥匙模式）
  - 截断超出空间限制的钥匙
  - 生成每个钥匙的屏幕坐标和纹理坐标

### 资源管理
- **destroy()**: 释放 Vertexbuffer 资源

## 视觉设计

### 尺寸规格
- **短钥匙**: 3x4 像素（高度 4）
- **长钥匙**: 3x7 像素（高度 7）
- **间距**: 钥匙之间有 1 像素间隔

### 纹理定位
- 所有钥匙纹理位于 `Assets.Interfaces.MENU_BTN` 的特定区域
- X 坐标: `43 + 3*keyIdx`
- Y 坐标: 短钥匙为 8，长钥匙为 0

### 颜色编码
- **黑色钥匙**: 表示错过的楼层（深度小于当前深度）
- **其他颜色**: 根据钥匙纹理本身的色彩

## 使用示例
```java
// 创建钥匙显示组件
KeyDisplay keyDisplay = new KeyDisplay();

// 设置尺寸
keyDisplay.width = 20;
keyDisplay.height = 10;

// 更新钥匙状态
keyDisplay.updateKeys();

// 添加到父容器
parent.add(keyDisplay);
```

## 注意事项
- 组件继承自 `Visual` 而不是 `Component`，需要手动管理生命周期
- 必须在渲染前调用 `updateKeys()` 以确保显示正确
- 尺寸必须足够大以容纳至少一个钥匙（建议最小 4x5 像素）
- 自动截断逻辑确保不会超出指定的宽高限制
- 键盘纹理支持长短两种模式以适应不同空间
- 性能优化：只有在状态改变时才重新生成顶点数据
