# Archs 类

## 概述

`Archs` 类是 Shattered Pixel Dungeon 游戏中的拱形背景装饰组件。它继承自 `Component` 类，通过双层滚动的拱形图案和渐变暗化效果，为游戏场景提供动态的视觉背景装饰。

该组件的主要特性包括：
- **双层滚动效果**：背景层和前景层以不同速度滚动，创造视差深度感
- **动态暗化渐变**：顶部到底部的黑色渐变增强地牢氛围
- **方向控制**：支持正向和反向滚动
- **性能优化**：针对无透明度背景进行渲染优化
- **自动适配**：根据容器尺寸自动调整显示区域

## 继承关系

- `com.watabou.noosa.ui.Component`
  - `com.shatteredpixel.shatteredpixeldungeon.ui.Archs`

## 常量定义

- **`SCROLL_SPEED = 20f`**：背景滚动的基础速度（像素/秒）

## 字段

### UI元素

- **`arcsBg: SkinnedBlock`**：拱形背景层（较慢的滚动速度）
- **`arcsFg: SkinnedBlock`**：拱形前景层（较快的滚动速度，速度是背景层的2倍）
- **`darkness: Image`**：暗化渐变遮罩，从透明到深黑的垂直渐变

### 状态管理

- **`offsB: float`** *(静态)*：背景层的垂直偏移量（用于状态保存）
- **`offsF: float`** *(静态)*：前景层的垂直偏移量（用于状态保存）
- **`reversed: boolean`**：滚动方向标记，`true` 时反向滚动

## 方法

### 生命周期方法

- **`createChildren(): void`**  
  创建所有子UI元素：
  - 背景层 `arcsBg`：使用 `ARCS_BG` 纹理，禁用混合模式以提升性能
  - 前景层 `arcsFg`：使用 `ARCS_FG` 纹理，启用正常混合模式
  - 暗化遮罩 `darkness`：创建垂直渐变纹理（透明 → 34%黑 → 85%黑 → 153%黑 → 238%黑）

- **`layout(): void`**  
  布局计算：
  - 设置两层拱形图案的尺寸为容器大小
  - 水平居中对齐：`offsetX = textureWidth/4 - (width % textureWidth)/2`
  - 配置暗化遮罩：旋转90度并设置适当的缩放比例

- **`update(): void`**  
  每帧更新滚动动画：
  - 计算基于帧时间的位移量：`shift = Game.elapsed * SCROLL_SPEED`
  - 根据 `reversed` 标记决定滚动方向
  - 背景层移动 `shift` 像素，前景层移动 `shift * 2` 像素
  - 保存当前偏移量到静态字段，用于组件重建时的状态恢复

## 技术特点

1. **视差滚动**：前景层滚动速度是背景层的2倍，创造深度感
2. **渲染优化**：背景层禁用Alpha混合（`Blending.disable()`），因为其纹理无透明度通道
3. **无光照渲染**：使用 `NoosaScriptNoLighting` 提升性能
4. **渐变遮罩**：通过程序化生成的渐变纹理实现自然的暗化效果
5. **状态持久化**：静态偏移量确保在组件重建时保持连续的滚动状态

## 使用场景

1. **主菜单背景**：为游戏主界面提供动态装饰
2. **地牢场景装饰**：在游戏场景边缘添加拱形装饰元素
3. **过场动画**：作为转场动画的背景元素
4. **UI装饰**：在各种窗口和面板的背景中使用
5. **氛围营造**：通过暗化渐变增强地牢的阴森氛围

## 相关类

- **`SkinnedBlock`**：可重复贴图的块状渲染组件
- **`TextureCache`**：纹理缓存管理器，用于创建渐变纹理
- **`Blending`**：OpenGL混合模式控制
- **`NoosaScriptNoLighting`**：无光照渲染脚本
- **`Assets.Interfaces`**：界面资源管理，包含 `ARCS_BG` 和 `ARCS_FG` 纹理

## 注意事项

- 暗化渐变的实际颜色值超过了标准ARGB范围（如0xEE000000），这是为了实现更强烈的暗化效果
- 水平居中算法考虑了纹理的重复特性，确保无缝拼接
- 组件设计为全屏或大区域使用，小尺寸下可能无法展现最佳效果
- 反向滚动功能主要用于特殊场景或动画效果
- 性能优化仅适用于背景层，前景层仍使用标准Alpha混合
