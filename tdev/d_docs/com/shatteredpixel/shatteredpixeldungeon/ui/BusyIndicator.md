# BusyIndicator 类

## 概述

`BusyIndicator` 类是 Shattered Pixel Dungeon 游戏中的忙碌指示器组件。它继承自 `Image` 类，显示一个旋转的动画图标，用于指示游戏英雄角色当前处于忙碌状态（即无法执行新操作）。

该组件会自动根据英雄的状态显示或隐藏：
- 当英雄存活且未就绪时（`!Dungeon.hero.ready`），指示器可见
- 当英雄死亡或已就绪时，指示器隐藏

## 继承关系

- `com.watabou.noosa.Image`
  - `com.shatteredpixel.shatteredpixeldungeon.ui.BusyIndicator`

## 构造函数

- **`BusyIndicator()`**  
  初始化忙碌指示器：
  - 从 `Icons.BUSY` 获取图标资源
  - 设置旋转中心点为图像中心
  - 设置角速度为 720 度/秒（每秒旋转两圈）

## 方法

### 更新方法

- **`update(): void`**  
  每帧调用的更新方法，控制指示器的可见性：
  ```java
  visible = Dungeon.hero.isAlive() && !Dungeon.hero.ready;
  ```
  
  这意味着指示器仅在以下条件同时满足时显示：
  - 英雄角色仍然存活
  - 英雄角色当前未准备好执行新操作（正在执行某个动作，如移动、攻击、使用物品等）

## 使用场景

忙碌指示器通常用于以下情况：
- 英雄正在执行回合制操作的冷却时间
- 角色被某些效果（如眩晕、冻结）暂时禁用
- 正在处理复杂的交互逻辑

## 技术特点

1. **自动状态管理**：无需手动控制显示/隐藏，完全基于游戏状态自动更新
2. **流畅动画**：通过设置角速度实现平滑的旋转动画效果
3. **资源高效**：直接复用现有的图标资源，避免重复加载
4. **实时响应**：每帧检查英雄状态，确保指示器状态与游戏逻辑同步

## 相关类

- **`Icons`**：图标资源管理类，提供 `BUSY` 图标
- **`Dungeon`**：游戏核心类，包含当前英雄对象
- **`Hero`**：英雄角色类，包含 `isAlive()` 和 `ready` 状态属性
- **`Image`**：基础图像组件类，提供旋转和动画功能
