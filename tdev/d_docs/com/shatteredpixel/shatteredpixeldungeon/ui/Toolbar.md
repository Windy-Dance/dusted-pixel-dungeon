# Toolbar 类

## 概述
`Toolbar` 是 Shattered Pixel Dungeon 游戏界面底部的工具栏组件，包含玩家主要的操作按钮。它管理多个子工具按钮，包括等待、搜索、背包和快捷物品槽等。

## 功能特性
- **多按钮管理**：包含等待、搜索、背包和最多 6 个快捷物品槽
- **动态布局**：支持多种布局模式（拆分、组合、居中）和翻转选项
- **交互状态管理**：根据英雄状态自动启用/禁用按钮
- **物品拾取动画**：显示物品拾取时的上移动画效果
- **检查模式**：支持进入物品检查模式（examining）

## 内部类

### Mode 枚举
定义工具栏布局模式：
- `SPLIT`: 拆分布局
- `GROUP`: 组合布局  
- `CENTER`: 居中布局

### Tool 类
基础工具按钮类，继承自 `Button`，提供统一的视觉样式和交互反馈。

### QuickslotTool 类  
专门用于快捷物品槽的按钮，包装 `QuickSlotButton` 组件。

### SlotSwapTool 类
用于在快捷物品槽之间切换的特殊工具按钮，显示当前槽位的物品图标。

### PickedUpItem 类
用于显示物品拾取动画的精灵组件，从地面位置平滑移动到背包按钮位置。

## 核心方法

### 初始化和销毁
- `Toolbar()` - 构造函数，初始化所有子按钮
- `destroy()` - 销毁工具栏并清理静态实例引用

### 布局管理
- `layout()` - 根据当前设置重新布局所有按钮
- `updateLayout()` - 静态方法，更新现有工具栏实例的布局

### 状态控制
- `alpha(float value)` - 设置整个工具栏的透明度
- `update()` - 更新按钮的启用状态（基于英雄是否就绪）

### 物品拾取
- `pickup(Item item, int cell)` - 触发物品拾取动画

### 快捷槽管理
- `swappedQuickslots` - 静态变量，跟踪当前是否交换了快捷槽
- `SWAP_INSTANCE` - 静态引用，指向当前的交换工具实例

## 子按钮说明

### 主要操作按钮
- `btnWait`: 等待/休息按钮，长按可深度休息
- `btnSearch`: 搜索/检查按钮，长按可主动搜索
- `btnInventory`: 背包按钮，长按可快速背包

### 快捷物品槽
- `btnQuick[]`: 数组包含最多 6 个快捷物品槽
- `btnSwap`: 槽位交换按钮，用于在两组快捷槽之间切换

### 隐藏按钮
工具栏还包含多个隐藏的按钮用于处理特定的键盘绑定：
- 快捷槽选择器
- 休息快捷键
- 等待/拾取快捷键
- 背包选择器

## 使用示例
```java
// 创建工具栏
Toolbar toolbar = new Toolbar();

// 更新布局（例如屏幕方向改变后）
Toolbar.updateLayout();

// 显示物品拾取动画
toolbar.pickup(myItem, heroPosition);

// 控制透明度
toolbar.alpha(0.8f);
```

## 注意事项
- 工具栏使用静态实例 `instance` 来确保全局唯一性
- 快捷槽的数量会根据屏幕宽度自动调整（4-6个）
- 布局模式可通过 `SPDSettings.toolbarMode()` 配置
- 工具栏会自动根据英雄状态启用/禁用相关按钮
- 物品拾取动画使用独立的相机进行坐标转换
