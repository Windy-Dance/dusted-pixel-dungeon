# GameLog 类

## 概述
`GameLog` 是 Shattered Pixel Dungeon 中的游戏日志UI组件，继承自 `Component` 并实现了 `Signal.Listener<String>` 接口。它负责显示游戏中的各种消息和事件通知，支持多种颜色类型的消息（正面、负面、警告、高亮等），并具有智能的文本合并和行数限制功能。

## 功能特性
- **多色消息支持**：支持默认、正面（绿色）、负面（红色）、警告（橙色）、高亮（中性色）五种消息类型
- **智能文本合并**：相同颜色的连续消息会自动合并到同一行，避免碎片化
- **动态行数限制**：根据界面大小设置自动调整显示行数（3-5行）
- **异步安全**：使用同步机制确保线程安全，避免并发问题
- **状态持久化**：支持清除日志和重新创建已保存的条目
- **信号监听**：通过 GLog.update 信号接收新消息

## 核心方法

### 构造函数
- `GameLog()` - 创建游戏日志组件并注册到 GLog 更新信号

### 重写方法
- `update()` - 处理新消息的添加、合并和显示逻辑
- `layout()` - 布局日志条目的垂直位置
- `onSignal(String text)` - 处理接收到的新消息

### 静态方法
- `wipe()` - 清除所有日志条目和待处理消息

### 实例方法
- `newLine()` - 强制创建新行，不合并到上一条消息

### 私有方法
- `recreateLines()` - 重新创建所有日志条目

## 内部组件
- `lastEntry` - 最后一个日志条目，用于消息合并判断
- `lastColor` - 最后一个条目的颜色，用于同色合并
- `entries` - 静态列表，存储所有日志条目信息
- `textsToAdd` - 同步列表，存储待处理的新消息
- `MAX_LINES` - 最大行数常量（3行）
- `PUNCTUATION` - 标点符号正则表达式模式
- `Entry` - 内部类，封装日志条目的文本和颜色信息

## 使用示例
```java
// 创建游戏日志
GameLog gameLog = new GameLog();

// 添加到界面
add(gameLog);

// 通过GLog系统发送消息（自动处理）
GLog.i("这是一条普通消息");
GLog.p("这是一条正面消息");
GLog.n("这是一条负面消息");
GLog.w("这是一条警告消息");

// 清除所有日志
GameLog.wipe();
```

## 注意事项
- 消息前缀通过 GLog.POSITIVE、GLog.NEGATIVE 等常量标识
- 日志条目使用 RenderedTextBlock 组件渲染6号字体
- 行间距为2像素，确保良好的可读性
- 当条目数量超过限制时，会自动移除最早的条目
- 使用 synchronized 关键字确保多线程安全
- 支持换行符（GLog.NEW_LINE）来强制分隔消息
