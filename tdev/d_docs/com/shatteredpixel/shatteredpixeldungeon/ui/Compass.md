# Compass 类

## 概述
`Compass` 是 Shattered Pixel Dungeon 中的指南针UI组件，继承自 `com.watabou.noosa.Image`。它显示一个指向特定地图格子的箭头，用于指示重要位置（如目标、宝藏或危险区域）的方向，会根据玩家视角的移动自动旋转以保持正确的指向。

## 功能特性
- **方向指示**：始终指向指定的地图格子，无论玩家如何移动或滚动屏幕
- **智能可见性**：仅在玩家已访问过或已探索过目标格子时才显示
- **自动旋转**：根据相机位置和目标格子的相对位置实时计算并更新角度
- **世界坐标转换**：将地图格子坐标转换为世界坐标进行精确计算
- **性能优化**：仅在相机滚动发生变化时重新计算角度，避免不必要的计算

## 核心方法

### 构造函数
- `Compass(int cell)` - 创建指向指定地图格子的指南针，初始状态为不可见

### 重写方法
- `update()` - 更新指南针的可见性和旋转角度

## 内部组件
- `cell` - 目标地图格子的索引
- `cellCenter` - 目标格子在世界坐标系中的中心位置
- `lastScroll` - 上一次相机滚动位置，用于检测是否需要重新计算角度
- `RAD_2_G` - 弧度到角度的转换常量（180/π）
- `RADIUS` - 指南针半径常量，固定为12像素

## 使用示例
```java
// 创建指向第50个格子的指南针
Compass compass = new Compass(50);

// 添加到场景
scene.add(compass);

// 指南针会自动根据相机位置更新方向和可见性
```

## 注意事项
- 指南针原点设置在图像中心底部（width/2, RADIUS），确保旋转时箭头正确指向
- 只有当目标格子在Dungeon.level.visited或Dungeon.level.mapped数组中标记为true时才可见
- 角度计算使用atan2函数，考虑了Y轴反转（center.y - cellCenter.y）
- 当目标格子无效（小于0或超出关卡范围）时，指南针会隐藏
- 指南针图标来自Icons.COMPASS枚举值
