# WndChangesTabbed 类

## 概述

`WndChangesTabbed` 类是 Shattered Pixel Dungeon 游戏中的分页式更新日志窗口组件。它继承自 `WndTabbed` 类，专门用于以标签页的形式展示游戏的版本更新内容，每个标签页对应一个更新版本或类别。

该组件的主要特性包括：
- **分页显示**：使用罗马数字（I, II, III...）作为标签页标识
- **自适应宽度**：根据内容长度和屏幕方向自动调整窗口宽度
- **点击穿透**：背景区域点击可关闭窗口
- **动态布局**：自动计算最佳显示尺寸以适应不同设备
- **多消息支持**：可同时显示多个版本的更新内容

## 继承关系

- `com.shatteredpixel.shatteredpixeldungeon.windows.Window`
  - `com.shatteredpixel.shatteredpixeldungeon.windows.WndTabbed`
    - `com.shatteredpixel.shatteredpixeldungeon.ui.changelist.WndChangesTabbed`

## 常量定义

- **`WIDTH_MIN = 120`**：窗口最小宽度（像素）
- **`WIDTH_MAX = 220`**：窗口最大宽度（像素）
- **`GAP = 2`**：元素间的间距（像素）

## 字段

- **`texts: ArrayList<RenderedTextBlock>`**：存储所有更新日志文本块的列表

## 构造函数

- **`WndChangesTabbed(Image icon, String title, String... messages)`**  
  创建分页更新日志窗口：
  - `icon`：窗口标题图标
  - `title`：窗口标题文本
  - `messages`：可变参数，每个字符串对应一个标签页的更新内容
  
  **初始化流程**：
  1. 创建背景点击区域（点击可关闭窗口）
  2. 创建带图标的标题栏
  3. 为每个消息创建对应的文本块和标签页
  4. 自动调整窗口宽度以适应内容
  5. 默认选择第一个标签页

## 方法

### 核心方法

- **`numToNumeral(int num): String`**  
  将数字转换为罗马数字字符串：
  - 支持1-10的罗马数字（I, II, III, IV, V, VI, VII, VIII, IX, X）
  - 超过10的数字直接返回阿拉伯数字字符串

### 内部类：LabeledTab

匿名内部类继承自 `WndTabbed.LabeledTab`，重写 `select()` 方法：
- 当标签页被选中时，显示对应的文本块
- 当标签页未被选中时，隐藏对应的文本块

## 技术特点

1. **响应式设计**：在横屏模式下自动扩展宽度以充分利用屏幕空间
2. **内存优化**：所有文本块都预先创建并存储在列表中，避免重复渲染
3. **用户体验**：背景点击关闭提供便捷的操作方式
4. **视觉一致性**：使用罗马数字作为标签页标识，符合游戏的整体美术风格
5. **布局智能**：自动计算文本高度并设置合适的窗口尺寸

## 使用场景

1. **版本更新日志**：显示游戏新版本的更新内容
2. **补丁说明**：展示热修复或小版本更新的详细信息  
3. **功能介绍**：分页展示新功能的详细说明
4. **平衡性调整**：按类别展示游戏平衡性改动

## 相关类

- **`WndTabbed`**：分页窗口基类，提供标签页管理功能
- **`RenderedTextBlock`**：文本渲染组件，支持多行文本显示
- **`IconTitle`**：带图标的标题组件
- **`PixelScene`**：场景工具类，提供UI相机和布局辅助方法
- **`PointerArea`**：指针事件处理区域

## 注意事项

- 窗口宽度会在120-220像素范围内自动调整
- 在横屏模式下，如果内容高度超过屏幕限制，会优先增加宽度而非高度
- 标签页数量与传入的消息数组长度一致
- 所有文本块都使用6号字体大小
- 背景点击区域覆盖整个UI相机区域，确保任何空白区域点击都能关闭窗口
