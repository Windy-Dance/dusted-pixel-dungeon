# ChangeInfo 类

## 概述

`ChangeInfo` 类是 Shattered Pixel Dungeon 游戏中的更新日志信息项组件。它继承自 `Component` 类，用于在更新日志窗口中显示单个更新条目，支持标题、描述文本和关联按钮的功能组合。

该组件的主要特性包括：
- **分级标题**：支持主要标题（大字体）和次要标题（小字体）两种模式
- **文本描述**：可选的详细描述文本，支持多行显示
- **按钮集成**：支持添加多个关联按钮（如链接、详情等）
- **自动布局**：智能排列标题、文本和按钮的位置
- **颜色编码**：支持整体颜色调整以区分不同类型的更新

## 继承关系

- `com.watabou.noosa.ui.Component`
  - `com.shatteredpixel.shatteredpixeldungeon.ui.changelist.ChangeInfo`

## 字段

### UI元素

- **`line: ColorBlock`**：分隔线或边框装饰
- **`title: RenderedTextBlock`**：标题文本块
- **`text: RenderedTextBlock`**：描述文本块（可为空）
- **`buttons: ArrayList<ChangeButton>`**：关联按钮列表

### 状态标志

- **`major: boolean`**：标记是否为主要标题（影响字体大小和分隔线样式）

## 构造函数

- **`ChangeInfo(String title, boolean majorTitle, String text)`**  
  创建更新信息项：
  - `title`：标题文本
  - `majorTitle`：是否为主要标题（`true` = 大字体9pt，`false` = 小字体6pt）
  - `text`：描述文本（可为 `null` 或空字符串）
  
  **初始化逻辑**：
  - 根据 `majorTitle` 选择字体大小和分隔线颜色
  - 创建对应的文本块组件
  - 设置主要/次要标题标志

## 方法

### 样式方法

- **`hardlight(int color): void`**  
  对所有文本元素应用硬光滤镜效果，用于高亮显示特定内容

### 按钮管理

- **`addButton(ChangeButton button): void`**  
  添加关联按钮到信息项中：
  - 将按钮添加到内部列表
  - 设置按钮尺寸为16x16像素
  - 触发重新布局

### 交互方法

- **`onClick(float x, float y): boolean`**  
  处理点击事件：
  - 遍历所有关联按钮
  - 如果点击位置在某个按钮内，触发该按钮的点击事件
  - 返回 `true` 表示事件已被处理

### 布局方法

- **`layout(): void`**  
  实现复杂的自动布局算法：
  
  **布局流程**：
  1. **标题定位**：水平居中，根据是否为主要标题调整垂直位置
  2. **文本定位**：位于标题下方，支持多行文本自动换行
  3. **按钮排列**：
     - 自动计算最佳按钮行数
     - 支持多行按钮布局
     - 水平居中对齐
     - 考虑按钮高度差异进行垂直对齐
  
  **特殊处理**：
  - 主要标题：顶部显示水平分隔线
  - 次要标题：左侧或右侧显示垂直分隔线（根据位置决定）

## 技术特点

1. **智能布局**：自动适应不同数量和尺寸的按钮
2. **内存优化**：使用ArrayList高效管理动态按钮列表
3. **视觉层次**：通过字体大小和分隔线样式区分重要性
4. **交互友好**：按钮点击区域精确定位，避免误触
5. **扩展性强**：易于添加新的UI元素类型

## 使用场景

1. **版本更新日志**：显示每个版本的主要变更和详细说明
2. **功能介绍**：展示新功能的标题、描述和相关操作按钮
3. **分类展示**：按类别组织更新内容（如平衡性调整、新内容、修复等）
4. **交互式文档**：提供链接到详细说明或相关功能的按钮

## 相关类

- **`ChangeButton`**：更新日志专用按钮类
- **`RenderedTextBlock`**：文本渲染组件
- **`ColorBlock`**：纯色块组件，用于分隔线
- **`PixelScene`**：场景工具类，提供文本渲染和对齐功能
- **`Component`**：UI组件基类

## 注意事项

- 按钮尺寸被强制设置为16x16像素，确保视觉一致性
- 文本块的最大宽度受容器宽度限制，自动换行
- 按钮布局算法会优先尝试单行排列，超出宽度时自动换行
- 主要标题（major=true）会额外增加顶部间距和更粗的分隔线
- 所有文本内容都支持本地化，实际显示文本来自消息系统
