# VaultSentry（金库哨兵）

## 角色概述
VaultSentry 是 Shattered Pixel Dungeon 中一种位于金库区域的防御性 NPC，表现为固定的扫描装置。它会按照预设的锥形范围和方向进行周期性扫描，能够检测到隐形状态的英雄并发出警报。

## 核心机制
### 扫描系统
- **锥形扫描**：使用 `ConeAOE` 类实现锥形范围的扫描效果
- **多方向支持**：通过 `scanDirs` 二维数组支持复杂的多方向扫描模式
- **参数配置**：可配置扫描宽度（`scanWidth`）和长度（`scanLength`）

### 冷却机制
- **扫描冷却**：`curCooldown` 控制当前扫描冷却时间
- **扫描后冷却**：`afterScanCooldown` 定义扫描完成后的基础冷却时间
- **连续扫描**：`scansAfterCooldown` 允许在一次冷却后连续进行多次扫描

### 警告系统
- **视觉警告**：在下次扫描前一回合显示红色警告标记
- **可配置**：`giveWarning` 标志控制是否启用警告功能（默认关闭）
- **智能显示**：仅在玩家视野范围内时显示警告

## 技术特性
### 扫描逻辑
- **弹道计算**：使用 `Ballistica` 类计算每个扫描方向的中心线
- **锥形扩展**：基于中心线和配置参数生成完整的锥形扫描区域
- **实体阻挡**：扫描会在遇到固体地形或目标时停止

### 音效管理
- **集体音效**：所有扫描哨兵共享音效播放限制（最多每 80ms 播放一次）
- **检测音效**：发现英雄时播放特殊的检测音效
- **实时同步**：使用 `ShatteredPixelDungeon.realTime` 确保音效同步

### 视觉效果
- **检测效果**：使用 `CheckedCell` 显示扫描检测到的格子
- **目标标记**：使用 `TargetedCell` 显示警告位置
- **精灵链接**：通过 `WardSprite.linkVisuals()` 正确显示外观

## 免疫特性
- **完全免疫**：对所有效果类型完全免疫
- **无敌状态**：无法被任何方式伤害或影响
- **被动存在**：纯粹作为环境威胁存在

## 数据持久化
### 存档数据
- **扫描参数**：保存扫描宽度、长度和方向数组
- **冷却时间**：保存当前和扫描后的冷却时间
- **扫描配置**：保存连续扫描次数和警告设置

### 兼容性处理
- **版本兼容**：包含对 3.3.X 版本存档的兼容处理
- **Bundle系统**：正确集成到游戏的存档和加载机制中

## 游戏内应用
### 地图生成
- **金库房间**：主要出现在金库相关的特殊房间中
- **固定位置**：作为不可移动的防御装置存在
- **隐身克制**：专门用于对抗英雄的隐形能力

### 战术考量
- **路径规划**：玩家需要观察扫描模式并规划安全路径
- **隐身风险**：即使处于隐形状态也会被扫描发现
- **时机把握**：利用冷却时间间隙快速通过危险区域

## 行为特性
### 检测逻辑
- **隐形检测**：能够检测到 `invisible == 0` 的英雄（即非完全隐形状态）
- **状态警告**：发现英雄时会在其上方显示"!!!"警告文本
- **仇恨生成**：为被发现的英雄添加对哨兵的仇恨

### 行动逻辑
- **持续行动**：每个回合都会尝试行动
- **物品投掷**：继承 NPC 的 `throwItems()` 方法，但通常不适用
- **优先级**：遵循标准的 NPC 行动优先级

## 交互机制
- **无交互**：`interact()` 方法直接返回 true，不提供任何交互选项
- **被动威胁**：纯粹作为环境威胁存在，不响应玩家互动

## 技术实现细节
### 精灵系统
- **WardSprite**：使用守卫精灵类作为基础
- **IMMOVABLE 属性**：确保不会被移动或位移
- **视觉链接**：通过 `linkVisuals()` 方法正确显示外观

### 状态管理
- **扫描计数**：`scansMade` 跟踪当前连续扫描次数
- **方向循环**：自动循环回到第一个扫描模式
- **冷却重置**：正确处理连续扫描和冷却重置逻辑
