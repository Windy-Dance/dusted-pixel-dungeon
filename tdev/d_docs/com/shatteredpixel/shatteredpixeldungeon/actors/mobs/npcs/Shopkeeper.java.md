# Shopkeeper（商人）

## 角色概述
Shopkeeper 是 Shattered Pixel Dungeon 中的核心 NPC，作为地下城各层商店的经营者，为玩家提供商品买卖服务。商人出现在地下城第 6、11、16 和 20 层的固定商店位置，是游戏中最重要的经济系统组成部分。

## 核心特性
### 外观与行为
- **精灵类**：使用 `ShopkeeperSprite` 作为视觉表现
- **不可移动**：拥有 `IMMOVABLE` 属性，固定在商店位置
- **地图标记**：在地图上显示为"SHOP"地标

### 安全机制
- **伤害响应**：对任何伤害或负面状态都会触发警告机制
- **逃跑机制**：在受到持续伤害后会逃跑并清空商店物品
- **气体净化**：受到伤害时会自动清除商店区域内的所有有害气体

## 交易系统
### 商品买卖
- **出售功能**：玩家可以向商人出售符合条件的物品
- **购买功能**：玩家可以从商店购买商品
- **回购系统**：支持最多 3 个最近售出物品的回购

### 价格计算
- **售价公式**：`item.value() * 5 * (Dungeon.depth / 5 + 1)`
- **等级影响**：价格随地下城深度增加而提高
- **公平交易**：确保玩家能够通过合理的价格进行交易

### 可售物品条件
物品必须满足以下所有条件才能被出售：
- 物品价值大于 0
- 物品不是唯一的且不可堆叠（如神器）
- 如果是防具，不能带有封印
- 如果已装备且被诅咒，不能出售

## 交互机制
### 对话系统
- **多选项界面**：提供"出售"、"对话"和可回购物品的选项
- **分层对话**：根据地下城深度显示不同的对话文本
  - 第 6 层：监狱介绍和职业特定对话
  - 第 11 层：洞穴相关对话
  - 第 16 层：城市相关对话  
  - 第 20 层：大厅相关对话
- **Ascension模式**：在 Ascension 挑战中显示特殊对话

### 界面设计
- **金币显示**：交互时会显示当前金币数量
- **图标支持**：回购物品会显示对应的物品图标
- **启用状态**：只有足够金币时才能购买物品

## 保护机制
### 伤害处理
- **首次警告**：第一次受到伤害时会发出警告并开始计时
- **缓冲机制**：有 1 回合的缓冲时间防止连锁效果导致立即逃跑
- **逃跑条件**：缓冲时间结束后再次受到伤害会立即逃跑

### 逃跑后果
- **商店清空**：逃跑时会摧毁所有待售商品
- **粒子效果**：逃跑时播放特殊的粒子效果
- **地图更新**：移除商店地标

## 数据持久化
### 存档数据
- **buybackItems**：保存可回购的物品列表
- **turnsSinceHarmed**：保存自上次受伤害以来的回合数
- **Bundle系统**：正确集成到游戏的存档和加载机制中

### 状态管理
- **MAX_BUYBACK_HISTORY**：限制回购历史最多 3 个物品
- **实时更新**：物品状态会在交易过程中实时更新

## 技术实现
### 选择器系统
- **itemSelector**：WandBag.ItemSelector 实现物品选择逻辑
- **条件检查**：`canSell()` 方法验证物品是否可售
- **交易窗口**：`WndTradeItem` 提供完整的交易界面

### 免疫能力
- **状态免疫**：对所有负面状态效果免疫
- **伤害免疫**：虽然会响应伤害但不会实际受损
- **被动存在**：主要作为功能性 NPC 存在

## 游戏内影响
### 经济系统
- **核心枢纽**：作为游戏经济系统的核心组成部分
- **资源循环**：允许玩家通过出售物品获得金币
- **战略价值**：商店位置成为玩家的重要战略点

### 统计数据
- **金币收集**：回购物品时会从统计数据中扣除相应金币
- **升级追踪**：投掷武器的升级阈值会被正确追踪和保存

## 扩展应用
- **ImpShopkeeper**：小恶魔商人的基类
- **模块化设计**：所有功能都可通过继承进行扩展
- **一致性保证**：确保所有商店体验的一致性
