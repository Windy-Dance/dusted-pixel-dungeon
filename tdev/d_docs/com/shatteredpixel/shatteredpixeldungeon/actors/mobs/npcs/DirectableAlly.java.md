# DirectableAlly（可指挥盟友）

## 角色概述
DirectableAlly 是 Shattered Pixel Dungeon 中的一个基础 NPC 类，用于创建可以被玩家直接控制的盟友单位。这个类本身不直接在游戏中出现，而是作为其他可指挥盟友（如召唤物）的基类。

## 核心行为机制
### 移动状态
DirectableAlly 拥有两种主要状态：
- **WANDERING（游荡状态）**：默认状态下会跟随玩家或移动到指定位置
- **HUNTING（狩猎状态）**：当发现敌人时自动进入攻击状态

### 行为优先级
- 行为优先级设置为 `MOB_PRIO + 1`，确保在其他怪物之前行动
- 对齐方式设置为 `ALLY`（盟友），不会攻击玩家

## 控制指令系统
### 防御位置控制
```java
public void defendPos(int cell)
```
- 命令盟友移动到指定位置并进行防御
- 盟友会持续尝试到达该位置，即使路径被阻挡
- 到达位置后会停止移动并专注于该区域的战斗

### 跟随英雄
```java
public void followHero()
```
- 取消所有特殊指令，恢复默认跟随玩家的行为
- 盟友会自动跟随玩家移动

### 目标锁定
```java
public void targetChar(Char ch)
```
- 命令盟友专门攻击指定的角色
- 适用于敌对目标，会立即进入狩猎状态

### 单元格指向
```java
public void directTocell(int cell)
```
- 智能指令系统，根据单元格内容自动选择合适的行为：
  - 如果单元格是空的或不可见：命令防御该位置
  - 如果单元格是玩家：命令跟随玩家
  - 如果单元格是敌人：命令攻击该敌人

## 自动战斗机制
### 自动攻击
- 默认启用自动攻击（`attacksAutomatically = true`）
- 当敌人在视野范围内且不在防御移动状态时，会自动切换到狩猎状态
- 攻击行为遵循标准的怪物 AI 逻辑

### 防御行为
- 在 WANDERING 状态下会始终朝向目标移动
- 如果无法移动到防御位置，会将当前位置设为新的防御点
- 防御状态下仍会响应敌人的威胁

## 数据持久化
### 存档机制
- **DEFEND_POS**：保存当前的防御位置
- **MOVING_TO_DEFEND**：标记是否正在移动到防御位置
- 这些数据会在游戏存档和加载时正确保存和恢复

## 技术特性
### 寻路系统
- 使用标准的 `getCloser()` 方法进行寻路
- 支持复杂的地形和障碍物处理
- 移动速度受角色自身速度属性影响

### 状态转换
- 状态转换逻辑处理了各种边界情况
- 确保在不同指令之间平滑过渡
- 避免状态冲突和逻辑错误

## 游戏内应用
DirectableAlly 类主要用于以下场景：
- **召唤物系统**：如某些法术或技能召唤的临时盟友
- **任务相关 NPC**：需要玩家指挥完成特定目标的 NPC
- **扩展功能**：作为其他可控制单位的基础实现

## 与其他系统的交互
- **Actor 系统**：正确集成到游戏的 Actor 管理系统中
- **视野系统**：尊重玩家的视野范围限制
- **战斗系统**：完全兼容标准的战斗和伤害计算机制
