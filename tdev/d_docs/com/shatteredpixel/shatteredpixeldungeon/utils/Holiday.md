# Holiday.java - 节日系统

## 概述
`Holiday` 枚举实现了《破碎像素地牢》的节日事件系统，根据当前日期自动检测并激活相应的节日内容。该系统为游戏增加季节性变化和特殊活动，提升玩家的游戏体验。

## 支持的节日

### 主要节日列表
- **LUNAR_NEW_YEAR** (农历新年) - 1月底至2月底的某时（7天）
- **APRIL_FOOLS** (愚人节) - 4月1日（1天，可覆盖复活节）
- **EASTER** (复活节) - 3月底至4月底的某时（6-7天）  
- **PRIDE** (骄傲月) - 6月24日至30日（7天）
- **SHATTEREDPD_BIRTHDAY** (破碎像素地牢生日) - 8月1日至7日（7天）
- **HALLOWEEN** (万圣节) - 10月24日至31日（7天）
- **PD_BIRTHDAY** (像素地牢生日) - 12月1日至7日（7天）
- **WINTER_HOLIDAYS** (冬季节日) - 12月15日至26日（12天）
- **NEW_YEARS** (新年) - 12月27日至1月2日（7天）

### 节日统计
- 每年总计 **61-62个** 节庆日
- 主要集中在 **10月下旬至2月上旬**
- 大部分节日持续 **7天**，提供足够的参与时间

## 技术实现

### 日期检测逻辑
#### 农历新年计算
- 使用硬编码的农历新年日期表（2020-2100年）
- 预先计算并存储每年的农历新年日期
- 提供高效的日期查找算法

```java
public static boolean isLunarNewYear(int year, int dayOfYear)
```

#### 复活节计算
- 实现复杂的复活节日期算法
- 考虑闰年对日期计算的影响
- 支持格里高利历的精确计算

```java
public static boolean isEaster(int year, int dayOfYear, boolean leapYear)
```

#### 公历节日检测
- 使用标准的 Calendar API 进行日期比较
- 简单的月份和日期匹配逻辑
- 特殊处理愚人节的优先级（可覆盖复活节）

### 缓存机制
```java
// 缓存当前节日以避免重复计算
private static Holiday cached;

public static void clearCachedHoliday()
public static Holiday getCurrentHoliday()
```

- **性能优化**：避免每帧重新计算节日
- **状态保持**：确保游戏过程中节日逻辑一致
- **自动清理**：游戏启动和场景切换时清空缓存

### 游戏机制调整
- 节日限定的掉落率调整

## 性能考虑

### 计算效率
- 日期检测只在必要时执行
- 使用预计算的查找表减少实时计算
- 缓存结果避免重复工作

### 内存使用
- 枚举实例内存占用极小
- 日期表使用紧凑的数据结构
- 避免不必要的对象创建

### 启动性能
- 节日检测在游戏启动时一次性完成
- 不影响游戏运行时性能
- 延迟初始化非必要资源

## 扩展性设计

### 添加新节日
1. 在枚举中添加新的节日项
2. 实现相应的日期检测逻辑
3. 创建节日内容和视觉资源
4. 集成到现有的节日系统中

### 节日优先级管理
- 愚人节具有最高优先级（可覆盖其他节日）
- 其他节日按日期顺序处理
- 支持复杂的重叠逻辑

### 区域特定节日
- 当前系统基于全球通用节日
- 可扩展支持区域性节日
- 需要考虑不同地区的文化差异

## 使用模式

### 获取当前节日
```java
Holiday current = Holiday.getCurrentHoliday();
if (current == Holiday.HALLOWEEN) {
    // 应用万圣节主题
}
```

### 清除节日缓存
```java
// 在场景切换时

```

### 日期特定检测
```java
GregorianCalendar cal = new GregorianCalendar(2026, Calendar.OCTOBER, 25);
Holiday holiday = Holiday.getHolidayForDate(cal);
```
