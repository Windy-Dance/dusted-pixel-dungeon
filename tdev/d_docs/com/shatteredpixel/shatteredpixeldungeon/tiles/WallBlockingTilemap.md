# WallBlockingTilemap.java - 墙壁阻挡视觉系统

## 概述
`WallBlockingTilemap` 管理墙壁阻挡瓦片，用于战争迷雾效果中遮挡墙壁后的可见性。该系统创建更真实的视线遮挡效果，确保玩家无法看到被墙壁完全遮挡的区域。

## 核心功能

### 阻挡状态管理
系统定义六种阻挡状态常量：
- **CLEARED (0)** - 清除状态，无阻挡效果
- **BLOCK_NONE (1)** - 无阻挡，完全可见
- **BLOCK_RIGHT (2)** - 右侧阻挡
- **BLOCK_LEFT (3)** - 左侧阻挡  
- **BLOCK_ALL (4)** - 完全阻挡
- **BLOCK_BELOW (5)** - 下方阻挡

### 复杂阻挡逻辑
`updateMapCell(int cell)` 方法实现复杂的阻挡状态计算，基于以下因素：
- **墙壁可见性**：当前单元格是否为墙壁且可见
- **邻居关系**：相邻单元格的地形类型和可见性
- **迷雾隐藏状态**：单元格是否被战争迷雾隐藏
- **特殊关卡处理**：特定关卡（如Boss关、矿洞）的特殊规则

### 区域更新支持
提供多种更新方法以支持高效渲染：
```java
public void updateArea(int cell, int radius)
public void updateArea(int x, int y, int w, int h)  
public void updateMap()
public void updateMapCell(int cell)
```

## 技术实现

### 特殊关卡处理
#### HallsBossLevel 支持
- Boss关卡有特殊的墙壁阻挡逻辑
- 考虑Boss房间的特殊布局需求
- 处理Boss战期间的动态阻挡变化

#### MiningLevel 支持  
- 矿洞关卡的特殊阻挡规则
- 处理岩石和水晶的阻挡特性
- 支持挖掘过程中的动态阻挡更新

### 迷雾集成
- 与 `FogOfWar` 系统紧密集成
- 使用 `fogHidden(int cell)` 方法检测迷雾状态
- 确保阻挡效果与可见性状态一致

### 性能优化
- **增量更新**：只更新受影响的单元格
- **缓存友好**：重用计算结果避免重复工作
- **早期退出**：在简单情况下快速返回结果

## 使用模式

### 基本集成
```java
// 在游戏场景中创建墙壁阻挡图层
WallBlockingTilemap wallBlocking = new WallBlockingTilemap();
wallBlocking.map(Dungeon.level.map, Dungeon.level.width());
scene.add(wallBlocking);
```

### 动态更新
```java
// 当地形发生变化时更新阻挡
wallBlocking.updateMapCell(changedCell);

// 批量更新受影响区域
wallBlocking.updateArea(affectedRect);
```

### 状态查询
虽然主要用于渲染，但阻挡状态也可用于游戏逻辑：
```java
// 检查某位置是否被阻挡
boolean isBlocked = wallBlocking.getTileVisual(pos, tile, flat) != WallBlockingTilemap.CLEARED;
```

## 性能特性

### 内存效率
- 使用原始整数数组存储状态
- 避免不必要的对象创建
- 与底层地图数据共享内存

### 渲染优化
- 与 `DungeonTilemap` 渲染管线集成
- 支持视锥剔除跳过屏幕外单元格
- 批处理渲染提高GPU效率

### 计算复杂度
- 单元格更新：O(1) 平均时间复杂度
- 区域更新：O(area) 时间复杂度
- 全图更新：O(n) 时间复杂度，n为地图大小
