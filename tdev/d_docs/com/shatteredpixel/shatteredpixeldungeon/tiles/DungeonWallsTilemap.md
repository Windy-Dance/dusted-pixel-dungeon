# DungeonWallsTilemap.java - 墙壁悬垂渲染系统

## 概述
`DungeonWallsTilemap` 负责渲染墙壁、门框和其他结构的悬垂和衔接效果，创建立体感和深度错觉。该系统通过在基础地形之上添加悬垂瓦片，使墙壁和门看起来具有三维结构，显著增强地牢环境的视觉质量。

## 核心功能

### 悬垂效果类型
系统支持多种类型的悬垂和衔接效果：

#### 墙壁悬垂 (Wall Overhangs)
- **标准墙壁**：`WALL_OVERHANG` 瓦片覆盖在墙壁上
- **特殊墙壁**：不同区域的墙壁有独特的悬垂样式  
- **动态适应**：根据邻接关系调整悬垂形状

#### 门框效果 (Door Frames)
- **普通门**：`DOOR_OVERHANG` 提供门框视觉效果  
- **开启状态**：`OPEN_DOOR_OVERHANG` 表示门已打开
- **锁定状态**：`LOCKED_DOOR_OVERHANG` 显示锁定门的特殊样式
- **水晶门**：`CRYSTAL_DOOR_OVERHANG` 用于特殊的水晶材质门

#### 其他结构悬垂
- **雕像基座**：`STATUE_OVERHANG` 为雕像提供底座效果
- **出口标记**：`EXIT_OVERHANG` 高亮显示楼层出口
- **草地覆盖**：高草和犁沟草上的悬垂效果
- **炼金锅装饰**：`ALCHEMY_OVERHANG` 为炼金设施添加装饰

### 衔接处理
- **内部墙壁衔接**：使用 `DungeonTileSheet.stitchInternalWallTile()` 处理复杂墙壁连接
- **角落检测**：自动识别并正确渲染墙壁角落
- **边缘平滑**：确保悬垂效果与邻接瓦片无缝衔接

## 技术实现

### 跳过单元格机制
```java
private HashSet<Integer> skipCells = new HashSet<>();
```

- **性能优化**：某些单元格不需要悬垂效果（如完全被遮挡的墙壁）
- **动态管理**：根据地形变化动态更新跳过列表
- **内存效率**：只存储需要跳过的单元格位置

### 视觉生成逻辑
`getTileVisual()` 方法实现复杂的悬垂选择逻辑：

1. **跳过检查**：首先检查单元格是否在跳过列表中
2. **墙壁检测**：识别墙壁类型并应用内部衔接逻辑  
3. **门类处理**：为各种门类应用适当的悬垂效果
4. **其他结构**：处理雕像、出口等特殊结构的悬垂

### 交互支持
```java
@Override
public boolean overlapsPoint(float x, float y) {
    return true;
}

@Override
public boolean overlapsScreenPoint(int x, int y) {
    return true;
}
```

- **点击穿透**：始终返回true以支持与底层元素的交互
- **事件传递**：确保玩家可以正常与门、陷阱等互动
- **用户体验**：悬垂效果不影响游戏操作

## 渲染特性

### 视觉层次
- **渲染顺序**：位于基础地形之上，其他特征之下
- **深度感知**：通过悬垂效果创建逼真的三维错觉
- **阴影处理**：适当的阴影增强立体感和真实感

### 变体支持
- **区域特定**：不同地牢区域有独特的墙壁悬垂样式  
- **随机变体**：利用 `DungeonTileSheet` 的变体系统增加视觉多样性
- **节日主题**：支持节日特定的悬垂视觉效果

## 性能优化

### 增量更新
- **智能跳过**：避免对不需要悬垂的单元格进行计算
- **局部更新**：只重绘发生变化的区域
- **缓存友好**：重用计算结果减少重复工作

### 内存管理
- **高效存储**：使用HashSet存储跳过单元格，提供O(1)查找
- **资源复用**：共享悬垂瓦片资源，避免重复加载  
- **按需创建**：延迟初始化非必要组件

### 渲染效率
- **批处理友 好**：与底层Tilemap渲染器完全兼容
- **视锥剔除**：自动跳过屏幕外的悬垂渲染
- **纹理优化**：利用GPU纹理缓存提高渲染效率

## 使用模式

### 自动集成
```java
// 在游戏场景中创建墙壁悬垂图层
DungeonWallsTilemap walls = new DungeonWallsTilemap();
walls.map(Dungeon.level.map, Dungeon.level.width());
scene.add(walls);
```

### 动态更新
```java
// 当门状态发生变化时
Dungeon.level.set(doorPos, Terrain.OPEN_DOOR);
walls.updateMapCell(doorPos); // 自动更新悬垂效果

// 当墙壁被破坏时
Dungeon.level.destroyWall(wallPos);
walls.updateMapCell(wallPos); // 自动移除悬垂效果
```

## 扩展性设计

### 新悬垂类型
添加新的悬垂效果非常简单：
1. 在 `DungeonTileSheet` 中定义新的悬垂瓦片常量
2. 在 `getTileVisual()` 方法中添加相应的处理逻辑  
3. 确保新悬垂类型与现有系统的兼容性

### 自定义悬垂逻辑
- **条件悬垂**：基于游戏状态或玩家属性的条件性悬垂
- **动画悬垂**：支持动态变化的悬垂效果（如闪烁、脉动）
- **交互悬垂**：玩家接近时触发的特殊悬垂效果

### 多层悬垂
- **垂直层次**：支持不同高度的多层悬垂效果
- **复杂结构**：处理拱门、塔楼等复杂建筑结构
- **动态变化**：支持可移动或可变形结构的悬垂更新
