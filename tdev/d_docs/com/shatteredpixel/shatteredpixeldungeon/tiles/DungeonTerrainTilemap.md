# DungeonTerrainTilemap.java - 基础地形渲染系统

## 概述
`DungeonTerrainTilemap` 是《破碎像素地牢》的主要地形渲染器，负责处理所有基础地牢地形的视觉表现，包括地板、墙壁、水域、深渊、门等核心元素。作为渲染管线的底层，它为整个地牢环境提供基础的视觉框架。

## 核心功能

### 地形类型支持
系统全面支持游戏中的所有基础地形类型：

#### 基础地板类型
- **EMPTY** - 标准地板
- **GRASS** - 普通草地  
- **HIGH_GRASS** - 高草（抬升效果）
- **FURROWED_GRASS** - 犁沟草地
- **EMBERS** - 余烬地板
- **WATER** - 水域

#### 墙壁和结构
- **WALL** - 标准墙壁
- **DOOR** - 普通门
- **OPEN_DOOR** - 开启的门  
- **LOCKED_DOOR** - 锁定的门
- **SECRET_DOOR** - 隐藏门

#### 特殊功能区域
- **ENTRANCE/EXIT** - 楼层入口/出口
- **WELL** - 生命之泉
- **EMPTY_WELL** - 空井
- **PEDESTAL** - 祭坛基座
- **BOOKSHELF** - 书架
- **BARRICADE** - 路障
- **ALCHEMY** - 炼金锅

#### 危险地形
- **CHASM** - 深渊
- **TRAP** - 陷阱底座
- **SECRET_TRAP** - 隐藏陷阱

### 视觉效果处理
`getTileVisual()` 方法实现复杂的地形视觉选择逻辑：

1. **直接地形映射**：通过 `directVisuals` 映射表直接转换常见地形
2. **水域拼接**：使用 `DungeonTileSheet.stitchWaterTile()` 处理水域邻接效果  
3. **深渊拼接**：使用 `DungeonTileSheet.stitchChasmTile()` 处理深渊边缘效果
4. **抬升地形**：为墙壁、门、雕像、草地等提供抬升视觉效果
5. **平坦变体**：处理特殊平坦地形的视觉变体

## 技术实现

### 直接视觉映射
```java
private static final SparseArray<Integer> directVisuals = new SparseArray<>();
static {
    directVisuals.put(Terrain.EMPTY,     DungeonTileSheet.FLOOR);
    directVisuals.put(Terrain.GRASS,     DungeonTileSheet.GRASS);
    directVisuals.put(Terrain.WALL,      DungeonTileSheet.WALL);
    // ... 其他地形映射
}
```

- **高效查找**：O(1) 时间复杂度的地形到视觉映射
- **内存优化**：使用SparseArray避免空值存储开销
- **易于维护**：清晰的映射关系便于添加新地形

### 拼接系统集成
- **水域拼接**：自动检测邻接地板并生成适当的水域边缘
- **深渊拼接**：根据上方地形生成正确的深渊衔接效果
- **内部墙壁拼接**：处理复杂墙壁连接的视觉一致性

### 抬升效果处理
系统为多种地形提供抬升视觉效果：
- **墙壁和门**：使用 `RAISED_WALL` 和 `RAISED_DOOR` 瓦片
- **雕像**：使用 `RAISED_STATUE` 创建立体效果  
- **高草和犁沟草**：使用抬升版本增强深度感
- **炼金锅和路障**：特殊结构的抬升视觉效果

## 渲染性能

### 增量更新优化
- **智能缓存**：只在地形发生变化时更新相关单元格
- **批量处理**：支持区域更新减少重复计算
- **早期退出**：简单地形快速返回，复杂地形深度处理

### 内存效率
- **资源复用**：共享瓦片视觉资源，避免重复加载
- **按需计算**：拼接和抬升效果仅在需要时计算
- **对象池**：重用临时计算对象减少GC压力

### 渲染管线集成
- **批处理友好**：与底层Tilemap渲染器完全兼容
- **视锥剔除**：自动跳过屏幕外的地形渲染  
- **纹理优化**：利用GPU纹理缓存提高渲染效率

## 使用模式

### 基本集成
```java
// 在游戏场景中创建基础地形图层
DungeonTerrainTilemap terrain = new DungeonTerrainTilemap();
terrain.map(Dungeon.level.map, Dungeon.level.width());
scene.add(terrain);
```

### 动态地形更新
```java
// 当地形发生变化时
Dungeon.level.set(pos, Terrain.WATER);
terrain.updateMapCell(pos); // 自动应用水域拼接效果

// 批量地形更新
Dungeon.level.fillArea(rect, Terrain.WALL);
terrain.updateArea(rect); // 批量更新墙面拼接
```

### 独立瓦片创建
```java
// 创建单个地形瓦片用于UI或其他用途
Image floorTile = DungeonTerrainTilemap.tile(pos, Terrain.EMPTY);
uiElement.setImage(floorTile);
```

## 扩展性设计

### 新地形支持
添加新的地形类型非常简单：
1. 在 `DungeonTileSheet` 中定义新的瓦片常量
2. 在 `directVisuals` 映射表中添加新的地形映射
3. 如需要特殊处理，在 `getTileVisual()` 中添加相应逻辑

### 自定义视觉效果
- **条件视觉**：基于游戏状态或玩家属性的条件性地形视觉
- **动态效果**：支持随时间变化的地形视觉（如闪烁、脉动）
- **交互反馈**：玩家接近或互动时的特殊地形视觉效果

### 高级地形系统
- **多层地形**：支持垂直方向的多层地形系统
- **可变形地形**：支持动态变化的地形形态
- **生态地形**：地形之间的相互影响和生态系统

## 设计原则

### 基础性
- **单一职责**：专注于基础地形渲染，不处理高级特效
- **稳定可靠**：作为渲染管线的基础，必须高度稳定
- **向后兼容**：保持API稳定性，支持平滑升级

### 性能优先  
- **高效实现**：最小化的计算开销和内存使用
- **按需渲染**：只在必要时进行复杂计算
- **跨平台优化**：在所有目标平台上保持良好性能

### 视觉一致性
- **统一风格**：确保所有地形遵循相同的设计语言
- **无缝衔接**：不同地形之间平滑过渡，无视觉断裂
- **细节丰富**：在不影响性能的前提下提供丰富的视觉细节

## 与其他系统的协同

- **动态更新**：地形变化时同步更新迷雾渲染
