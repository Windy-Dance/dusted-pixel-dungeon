# GridTileMap.java - 网格叠加层系统

## 概述
`GridTileMap` 提供可选的视觉网格叠加层，主要用于玩家辅助和导航。该系统在地牢地形上显示半透明的网格线，帮助玩家更好地理解瓦片边界和进行精确的移动规划。

## 核心功能

### 网格显示控制
- **用户设置驱动**：通过 `SPDSettings.visualGrid()` 控制网格显示
- **动态切换**：支持运行时启用/禁用网格显示
- **性能友好**：仅在启用时进行渲染计算

### 网格图案设计
系统为不同类型的地形提供差异化的网格显示：

#### 棋盘图案 (Checkerboard Pattern)
- **地板瓦片**：标准棋盘格图案
- **草地瓦片**：与地板相同的棋盘格图案
- **视觉对比**：交替的明暗格子增强边界识别

#### 门瓦片特殊处理
- **墙壁邻接检测**：根据相邻墙壁位置调整网格显示
- **动态适应**：网格图案根据门的周围环境自动调整
- **一致性维护**：确保门区域的网格与周围地形协调

## 技术实现

### 设置集成
```java
@Override
public void updateMap() {
    enabled = SPDSettings.visualGrid();
}
```

- **实时同步**：每次地图更新时检查设置状态
- **零开销**：禁用时完全跳过渲染逻辑
- **用户友好**：设置变更立即生效

### 瓦片选择逻辑
`getTileVisual()` 方法根据地形类型返回适当的网格瓦片：

```java
// 基础地板和草地使用棋盘格
if (tile == Terrain.EMPTY || tile == Terrain.GRASS /* ... */) {
    return ((x + y) % 2 == 0) ? GRID : GRID_ALT;
}

// 门瓦片的特殊处理
if (tile == Terrain.DOOR || tile == Terrain.OPEN_DOOR /* ... */) {
    // 根据邻接墙壁调整网格显示
    return getDoorGridVisual(x, y, neighbors);
}
```

### 性能优化
- **条件渲染**：仅在启用时执行计算
- **简单算法**：棋盘格使用简单的坐标奇偶性判断
- **缓存友好**：重用网格瓦片常量，避免重复计算

## 使用模式

### 自动集成
```java
// 在游戏场景中创建网格图层
GridTileMap grid = new GridTileMap();
grid.map(Dungeon.level.map, Dungeon.level.width());
scene.add(grid);
```

### 用户控制
```java
// 用户在设置中切换网格显示
SPDSettings.visualGrid(true);  // 启用
SPDSettings.visualGrid(false); // 禁用

// 网格图层自动响应设置变更
```

## 性能特性

- **轻量级实现**：仅使用两个网格瓦片常量
- **无额外资源**：复用现有的纹理资源
- **最小内存占用**：不存储额外的状态信息

### 渲染性能

- **视锥剔除**：自动跳过屏幕外的网格渲染

### CPU开销
- **条件执行**：禁用时零CPU开销
- **高效算法**：启用时最小化计算负担
- **帧率友好**：不影响游戏流畅度
