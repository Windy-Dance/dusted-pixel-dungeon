# SpellSprite 类

## 概述
`SpellSprite` 类是 Shattered Pixel Dungeon 游戏中的法术图标精灵类。它继承自 `Image`，专门用于在角色头顶显示各种法术、状态或技能的图标效果。

## 功能描述
该类实现了法术图标的核心功能，具有以下特点：
- **三阶段动画**: 完整的淡入、静态显示、淡出三阶段动画循环
- **角色绑定**: 自动跟随目标角色的位置和可见性
- **颜色调整**: 支持自定义RGB颜色调整以适应不同效果
- **单例管理**: 确保每个角色同时只显示一个法术图标
- **纹理优化**: 使用纹理胶片技术高效管理多个图标帧

## 核心字段

### 图标类型常量
- **FOOD (0)**: 饱食图标
- **MAP (1)**: 地图图标  
- **CHARGE (2)**: 充能图标
- **BERSERK (3)**: 狂暴图标
- **ANKH (4)**: 复活图标
- **HASTE (5)**: 加速图标
- **VISION (6)**: 视野图标
- **PURITY (7)**: 净化图标

### 动画配置
- **SIZE**: 图标尺寸固定为16像素
- **FADE_IN_TIME**: 淡入时间0.2秒
- **STATIC_TIME**: 静态显示时间0.8秒
- **FADE_OUT_TIME**: 淡出时间0.4秒

### 内部状态
- **Phase**: 动画阶段枚举 (FADE_IN, STATIC, FADE_OUT)
- **target**: 目标角色对象，用于位置跟随
- **duration/passed**: 动画持续时间和已过时间
- **all**: 所有活跃图标的HashMap映射表

## 构造函数

### SpellSprite()
**功能**: 创建基础法术图标实例

**实现细节**:
- 加载 `Assets.Effects.SPELL_ICONS` 纹理资源
- **纹理胶片**: 初始化16x16像素的纹理胶片 (`new TextureFilm(texture, SIZE)`)
- 仅在首次创建时初始化纹理胶片，避免重复操作

## 核心方法

### reset(int index)
**功能**: 重置并配置法术图标

**实现逻辑**:
1. **帧设置**: 从纹理胶片获取指定索引的图标帧 (`film.get(index)`)
2. **原点设置**: 设置原点为中心点，便于后续定位
3. **动画重置**: 
   - 设置初始阶段为 `FADE_IN`
   - 设置持续时间为 `FADE_IN_TIME` (0.2秒)
   - 重置已过时间为0

### update()
**功能**: 每帧更新法术图标状态

**实现逻辑**:
1. **位置同步**: 
   - 如果目标角色存在且精灵可见，同步到角色头顶位置
   - X坐标: `sprite.center().x - SIZE/2`
   - Y坐标: `sprite.y - SIZE` (位于角色头顶上方)
2. **动画阶段处理**:
   - **FADE_IN**: 透明度和缩放从0线性增加到1
   - **STATIC**: 保持完全不透明和完整尺寸
   - **FADE_OUT**: 透明度从1线性减少到0
3. **阶段转换**: 
   - 时间到达后自动切换到下一阶段
   - FADE_IN → STATIC → FADE_OUT → 销毁

### kill()
**功能**: 销毁法术图标

**实现逻辑**:
- 调用父类 `kill()` 方法
- 从全局映射表 `all` 中移除对应的角色条目

### show(Char ch, int index)
### show(Char ch, int index, float r, float g, float b)
**功能**: 静态工厂方法，在指定角色头顶显示法术图标

**实现逻辑**:
1. **可见性检查**: 如果角色不可见则直接返回
2. **冲突处理**: 如果角色已有活跃图标，先销毁旧图标
3. **新图标创建**: 
   - 从 `GameScene` 获取新的法术图标实例
   - 绑定目标角色
   - 重置图标类型
   - 应用颜色调整 (`hardlight(r, g, b)`)
   - 激活图标 (`revive()`)
4. **注册管理**: 将新图标添加到全局映射表中

## 技术特点

### 动画系统
- **三阶段设计**: 提供完整的入场-停留-离场动画体验
- **精确时长**: 各阶段时间经过精心调校确保视觉舒适
- **平滑过渡**: 线性插值确保动画流畅自然

### 资源管理
- **纹理胶片**: 高效管理多个小图标帧，减少纹理切换开销
- **单例约束**: 确保每个角色不会同时显示多个图标，避免视觉混乱
- **自动清理**: 图标销毁时自动清理相关资源和映射

### 性能优化
- **延迟初始化**: 纹理胶片仅在需要时初始化
- **可见性检查**: 只在角色可见时进行位置同步
- **对象复用**: 通过 `GameScene.spellSprite()` 获取复用的实例

## 使用场景

### Buff显示
- **增益状态**: 各种正面buff的持续视觉提醒
- **减益状态**: debuff效果的视觉指示
- **特殊状态**: 狂暴、隐身等特殊状态的图标显示

### 技能反馈
- **技能触发**: 技能激活时的视觉确认
- **充能完成**: 技能充能完成的提示效果
- **冷却提示**: 技能冷却相关的视觉反馈

### UI系统
- **状态栏补充**: 作为状态栏图标的补充或替代
- **战斗日志**: 配合战斗日志提供视觉上下文
- **教学引导**: 新手教程中的技能演示效果

### 特殊事件
- **剧情演出**: 关键剧情中的状态变化效果
- **成就解锁**: 特殊成就达成时的庆祝效果
- **Boss战**: Boss特殊状态的视觉表现

## 相关类
- **Image**: 图像基类，提供基础渲染功能
- **Assets.Effects.SPELL_ICONS**: 法术图标纹理资源
- **TextureFilm**: 纹理胶片类，管理多帧纹理
- **Char**: 角色基类，作为图标的目标对象
- **GameScene**: 游戏场景类，提供图标实例池
- **Game**: 游戏主类，提供时间流逝信息

## 设计哲学
`SpellSprite` 类体现了游戏UI设计的几个重要原则：

### 用户友好
通过清晰的图标和流畅的动画提供直观的状态反馈，降低玩家认知负担。

### 视觉层次
位于角色头顶的位置确保图标既明显又不会遮挡重要游戏元素。

### 性能意识
通过资源复用和智能管理确保大量角色同时存在时的性能稳定。

### 可扩展性
虽然当前只使用了8种图标，但设计支持轻松添加更多图标类型。

## 扩展建议

### 视觉增强
- **动态效果**: 支持图标旋转、脉冲等动态效果

- **主题适配**: 根据游戏主题自动调整图标风格

### 功能扩展
- **交互支持**: 点击图标显示详细信息
- **优先级系统**: 不同重要性的图标有不同的显示优先级
- **持久化**: 支持图标状态的保存和加载

### 用户定制
- **大小调节**: 允许玩家调整图标尺寸
- **位置偏移**: 支持自定义图标显示位置
- **动画速度**: 允许调节动画播放速度
