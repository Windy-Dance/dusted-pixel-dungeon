# SpectralWallParticle 类

## 概述
`SpectralWallParticle` 类是 Shattered Pixel Dungeon 游戏中的幽灵墙粒子效果类。它继承自 `PixelParticle`，专门用于表现不同地牢层级中魔法墙壁、幽灵屏障或神秘结界的粒子视觉效果。

## 功能描述
该类实现了幽灵墙粒子的核心特性，具有以下特点：
- **关卡适配**: 根据地牢深度自动调整外观，适配5种不同的地牢主题
- **类型系统**: 支持5种不同的视觉类型（下水道、监狱、洞穴、矮人城、恶魔殿）
- **长生命周期**: 固定4秒的超长生命周期确保持续的视觉效果
- **半透明效果**: 固定60%的透明度创造幽灵般的半透明效果
- **智能频率**: 根据地牢类型自动调整粒子发射频率

## 核心字段

### 工厂实例
- **FACTORY**: 静态工厂实例，负责幽灵墙粒子的创建和回收
  - **智能频率控制**: 
    - 类型1(下水道): 50%发射概率 (Random.Int(2) != 0)
    - 类型2(监狱): 33%发射概率 (Random.Int(3) != 0)  
    - 类型3-4(洞穴/矮都): 100%发射概率
    - 类型5(恶魔殿): 25%发射概率 (Random.Int(4) != 0)
  - `lightMode()`: 返回 false，使用正常混合模式

### 视觉属性  
- **type**: 当前粒子类型 (1-5)，对应不同地牢主题
- **lifespan**: 生命周期固定为4秒（超长持续时间）
- **am**: 固定透明度为0.6f (60%不透明)

## 构造函数

### SpectralWallParticle()
**功能**: 创建基础幽灵墙粒子实例

**实现细节**:
- 调用父类构造函数初始化基础 particle
- 配置超长生命周期为4秒
- 设置固定透明度为60%

## 核心方法

### reset(float x, float y)
**功能**: 重置并配置幽灵墙粒子的初始状态

**实现逻辑**:
1. **激活粒子**: 调用 `revive()` 激活粒子使其可见
2. **类型确定**: 
   - 根据 `Dungeon.depth` 计算类型: `1 + depth/5`
   - 最大类型限制为5
3. **位置设置**: 设置基础坐标
4. **生命周期**: 设置完整生命周期为4秒
5. **类型特定配置**:
   - **类型1(下水道)**: 灰色系 (0xD4D4D4-0xABABAB)，网格对齐7x4
   - **类型2(监狱)**: 黄褐色系 (0xc4be9c-0x9c927d)，网格对齐7x6  
   - **类型3(洞穴)**: 灰白色系 (0xb7b0a5-0x6a6662)，基于坐标的颜色插值
   - **类型4(矮都)**: 棕黄色系 (0xd0bca3-0xa38d81)，网格对齐4x4
   - **类型5(恶魔殿)**: 深褐色系 (0xa2947d-0x594847)，水平对齐4，垂直对齐16

### update()
**功能**: 每帧更新幽灵墙粒子状态

**实现逻辑**:
- 调用父类的 `update()` 处理基础的运动逻辑
- **缩放控制**: 根据类型和生命周期进度动态调整尺寸:
  - **类型1**: X=6*sizeFactor, Y=3*sizeFactor (宽扁矩形)
  - **类型2**: X=6*sizeFactor, Y=5*sizeFactor (接近正方形)
  - **类型3**: X=4*sizeFactor, Y=4*sizeFactor (正方形)
  - **类型4**: X=3*sizeFactor, Y=3*sizeFactor (小正方形)
  - **类型5**: X=4*sizeFactor, Y=20*sizeFactor (高瘦矩形)

## 技术特点

### 关卡适配
- **深度映射**: `1 + depth/5` 将地牢深度映射到5种类型
- **主题一致性**: 每种类型的颜色和形状都匹配对应的地牢主题
- **网格对齐**: 大部分类型使用网格对齐创造砖墙或石墙效果

### 视觉设计
- **颜色语义**: 每种地牢类型使用符合主题的颜色方案
- **形状差异**: 不同类型的缩放比例创造独特的视觉形态
- **半透明效果**: 固定60%透明度创造幽灵般的视觉效果

### 性能优化
- **频率控制**: 根据地牢类型调整发射频率，避免过度密集
- **对象池**: 通过 `emitter.recycle()` 复用粒子对象
- **简单计算**: 使用高效的数学运算避免复杂开销

## 使用场景

### 环境效果
- **魔法屏障**: 魔法墙或结界的粒子效果
- **幽灵墙壁**: 幽灵或亡灵相关的墙壁效果
- **神秘结界**: 神秘区域的特殊墙壁表现
- **主题适配**: 自动适配不同地牢主题的视觉效果

### 战斗系统
- **Boss技能**: Boss召唤的魔法墙或屏障效果
- **环境互动**: 角色与魔法墙壁的交互效果
- **范围技能**: 墙壁相关的范围技能视觉反馈

### UI系统
- **区域提示**: 特殊区域或危险区域的视觉提醒
- **状态指示**: 与墙壁相关的状态效果
- **成就解锁**: 相关成就达成时的庆祝效果

### 特殊事件
- **剧情演出**: 关键魔法时刻的视觉强化
- **隐藏要素**: 隐藏墙壁或秘密通道的提示效果
- **机关触发**: 魔法机关激活时的效果

## 相关类
- **PixelParticle**: 基础粒子类，提供基础的渲染和运动功能
- **Emitter**: 粒子发射器，管理粒子池和发射逻辑  
- **ColorMath**: 颜色数学工具，提供颜色随机和插值功能
- **Random**: 随机数生成器，用于频率控制和颜色随机化
- **Dungeon**: 地牢系统，提供深度信息用于类型确定

## 设计哲学
`SpectralWallParticle` 类体现了游戏特效设计的几个重要原则：

### 环境整合
通过关卡深度自动适配，确保特效与游戏环境保持完美的视觉一致性。

### 主题表达
每种地牢类型都有独特的视觉特征，强化了不同地牢区域的主题感。

### 性能平衡
通过智能的频率控制和简单的计算，在保证视觉质量的同时确保性能稳定。

### 用户沉浸
半透明的幽灵效果和主题一致的外观增强了游戏的沉浸感和氛围营造。
