# Pushing 类

## 概述
`Pushing` 类是 Shattered Pixel Dungeon 游戏中的推挤/击退效果系统。它继承自 `Actor`，专门用于处理角色被强制移动（如击退、传送、交换位置）时的平滑动画效果。

## 功能描述
该类实现了完整的推挤效果系统，具有以下核心特性：
- **平滑移动**: 使用缓动函数实现角色的平滑位移动画
- **视觉同步**: 确保角色精灵与实际游戏逻辑位置保持同步
- **镜头跟随**: 当英雄被推挤时自动调整相机跟随
- **回调机制**: 支持推挤完成后执行回调函数
- **优先级管理**: 设置较高的动作优先级确保及时执行
- **并发控制**: 处理多个同时发生的推挤效果

## 核心字段

### 配置参数
- **sprite**: 目标角色的精灵对象
- **from**: 起始位置（一维格子索引）
- **to**: 目标位置（一维格子索引）
- **ch**: 被推挤的角色对象
- **callback**: 完成回调函数
- **effect**: 内部视觉效果实例

### 优先级设置
- **actPriority**: 设置为 `VFX_PRIO+10`，确保视觉效果优先执行

## 构造函数

### Pushing(Char ch, int from, int to)
**功能**: 创建基础推挤效果实例

**实现细节**:
- 保存角色、起始和目标位置信息
- 如果推挤的是英雄，启动相机跟随 (`Camera.main.panFollow`)
- 初始化回调为 null

### Pushing(Char ch, int from, int to, Callback callback)
**功能**: 创建带回调的推挤效果实例

**实现细节**:
- 调用基础构造函数
- 保存完成回调函数

## 核心方法

### act()
**功能**: 执行推挤动作的主要逻辑

**实现逻辑**:
1. **自我移除**: 从Actor系统中移除当前实例
2. **可见性检查**: 确保起始或目标位置在英雄视野内时角色可见
3. **效果创建**: 如果内部效果未创建，则实例化Effect内部类
4. **并发协调**: 检查是否有其他同时进行的推挤效果，如果有则延迟执行
5. **调度控制**: 返回false表示需要继续调度，true表示完成

### pushingExistsForChar(Char ch)
**功能**: 静态工具方法，检查指定角色是否正在进行推挤效果

**使用场景**: 在触发新的推挤效果前检查是否已有推挤正在进行，避免冲突

## 内部类 Effect

### Effect()
**功能**: 内部视觉效果构造函数

**实现细节**:
- 继承自 `Visual` 提供基础视觉功能
- 将世界坐标转换为相机坐标 (`worldToCamera`)
- 计算缓动运动参数：
  - 初始速度: `2 * distance / DELAY`
  - 加速度: `-speed / DELAY`
- 将效果添加到精灵父容器中

### update()
**功能**: 每帧更新推挤效果状态

**核心逻辑**:
1. **时间跟踪**: 累计经过的时间
2. **动画阶段判断**:
   - **进行中阶段** (`delay < DELAY`): 将角色精灵位置同步到当前动画位置
   - **完成阶段** (`delay >= DELAY`): 
     - 将角色精灵定位到最终位置
     - 销毁视觉效果 (`killAndErase`)
     - 从Actor系统移除主推挤对象
     - 执行回调函数（如果存在）
     - 重新排序怪物精灵层级 (`GameScene.sortMobSprites`)
     - 调度下一个动作 (`next()`)

## 技术特点

### 缓动动画
- **二次缓动**: 使用匀加速运动创造自然的推挤效果
- **精确时长**: 固定0.15秒的动画时长确保一致性
- **坐标转换**: 正确处理世界坐标到屏幕坐标的转换

### 系统集成
- **Actor系统**: 完全集成到游戏的Actor调度系统
- **视觉层级**: 正确管理精灵的渲染层级
- **相机控制**: 与相机系统协同工作提供最佳视觉体验

### 并发安全
- **效果协调**: 确保多个推挤效果不会互相干扰
- **状态检查**: 提供工具方法检查推挤状态避免冲突
- **资源管理**: 正确的创建和销毁生命周期管理

## 使用场景

### 战斗系统
- **击退技能**: 角色或敌人的击退攻击效果
- **爆炸效果**: 爆炸产生的冲击波推挤效果
- **魔法推挤**: 各种推挤类魔法技能的视觉表现
- **环境伤害**: 陷阱或环境危险造成的强制移动

### 移动系统
- **位置交换**: 两个角色位置互换的动画
- **传送效果**: 短距离传送的平滑过渡
- **强制移动**: 任何强制改变角色位置的游戏机制

### 特殊事件
- **Boss技能**: 强大Boss的范围推挤技能
- **剧情演出**: 关键剧情中的强制移动效果
- **机关触发**: 游戏机关激活时的推挤效果

## 相关类
- **Actor**: 游戏动作系统基类，提供调度和优先级管理
- **Char**: 角色基类，作为推挤的目标对象
- **CharSprite**: 角色精灵类，提供视觉表现和坐标转换
- **Camera**: 相机系统，处理英雄推挤时的镜头跟随
- **GameScene**: 游戏场景类，提供精灵排序功能
- **Visual**: 视觉对象基类，为内部Effect提供基础功能
- **Callback**: 回调接口，处理推挤完成事件

## 设计哲学
`Pushing` 类体现了游戏开发中的几个重要设计原则：

### 分离关注点
将视觉动画（Effect内部类）与游戏逻辑（Pushing主类）分离，提高代码可维护性。

### 系统集成
深度集成到现有的Actor、Camera、Visual等系统中，而不是创建孤立的功能。

### 用户体验
通过相机跟随、平滑动画等细节优化玩家的游戏体验。

### 性能意识
使用对象复用、精确调度和资源管理确保性能稳定。

## 扩展建议

### 动画增强
- **不同推挤类型**: 支持不同类型的推挤动画（直线、弧线、瞬移等）
- **强度可视化**: 根据推挤力度调整动画速度和距离
- **粒子效果**: 配合粒子系统增强推挤的视觉冲击力

### 功能扩展
- **碰撞检测**: 推挤过程中与障碍物的交互处理
- **连锁推挤**: 一个角色推挤另一个角色的连锁反应
- **方向指示**: 显示推挤方向的视觉提示

### 用户定制
- **动画速度**: 允许玩家调节推挤动画的速度
- **效果强度**: 根据玩家偏好调整推挤效果的明显程度
