# BloodParticle 类

## 概述
`BloodParticle` 类是 Shattered Pixel Dungeon 游戏中的血液粒子效果类。它继承自 `PixelParticle.Shrinking`，专门用于表现角色或敌人受伤时产生的血液飞溅和滴落效果。

## 功能描述
该类实现了血液粒子的两种不同行为模式，具有以下特点：
- **收缩动画**: 粒子在生命周期内逐渐缩小
- **鲜红色调**: 使用固定的深红色 (0xCC0000) 表现血液颜色
- **双重模式**: 支持静态滴落和爆发飞溅两种效果
- **重力效果**: 应用向下的加速度模拟血液下落
- **工厂模式**: 提供两种不同的工厂实例适应不同使用场景

## 核心字段

### 工厂实例
- **FACTORY**: 基础工厂实例，用于普通的血液滴落效果
  - `emit()`: 调用 `reset()` 方法创建静态滴落粒子
  
- **BURST**: 爆发工厂实例，用于血液飞溅效果  
  - `emit()`: 调用 `resetBurst()` 方法创建爆发飞溅粒子
  - `lightMode()`: 返回 true，使用光照混合模式增强视觉效果

### 视觉属性
- **color**: 固定为深红色 (0xCC0000)，表现血液的真实颜色
- **lifespan**: 基础生命周期为0.8秒
- **acc**: 加速度设置为 (0, +40)，模拟向下的重力效果

## 构造函数

### BloodParticle()
**功能**: 创建基础血液粒子实例

**实现细节**:
- 调用父类构造函数初始化收缩粒子
- 设置固定的颜色为深红色 (0xCC0000)
- 配置基础生命周期为0.8秒
- 设置向下的加速度 (+40) 模拟重力

## 核心方法

### reset(float x, float y)
**功能**: 重置并配置静态滴落粒子的初始状态

**实现逻辑**:
1. **激活粒子**: 调用 `revive()` 激活粒子使其可见
2. **位置设置**: 设置粒子的初始坐标
3. **生命周期**: 设置完整生命周期 (0.8秒)
4. **尺寸初始化**: 设置初始大小为4像素
5. **速度配置**: 速度设置为零，粒子仅受重力影响垂直下落

### resetBurst(float x, float y)
**功能**: 重置并配置爆发飞溅粒子的初始状态

**实现逻辑**:
1. **激活粒子**: 调用 `revive()` 激活粒子
2. **位置设置**: 设置粒子的初始坐标
3. **速度配置**: 
   - 随机角度: 在完整圆周范围内随机选择
   - 随机速度: 在16-32像素/秒范围内随机选择
   - 使用极坐标方式设置速度向量
4. **尺寸初始化**: 设置初始大小为5像素（略大于滴落模式）
5. **生命周期**: 设置较短的生命周期为0.5秒

### update()
**功能**: 每帧更新粒子状态

**实现逻辑**:
- 调用父类的 `update()` 处理基础的收缩和运动逻辑
- **透明度控制**: 
  - 在生命周期后40% (p > 0.6f) 内，透明度从1线性减少到0
  - 公式: `am = (1 - p) * 2.5f`
  - 在前60%时间内保持完全不透明 (am = 1)

## 技术特点

### 双重行为模式
- **滴落模式**: 适用于普通伤害的血液滴落，表现更加真实
- **爆发模式**: 适用于暴击或重击的血液飞溅，增加视觉冲击力

### 物理模拟
- **重力影响**: 向下的加速度使粒子轨迹呈现自然的抛物线形状
- **速度差异**: 爆发模式中的随机速度创造更真实的飞溅效果
- **尺寸调整**: 不同模式使用不同的粒子大小以匹配视觉需求

### 视觉优化
- **淡出效果**: 生命周期结束前的淡出避免粒子突然消失
- **光照混合**: 爆发模式使用特殊的混合模式增强视觉冲击力
- **颜色一致性**: 固定的深红色确保血液效果的视觉统一性

### 性能考虑
- **对象池**: 通过 `emitter.recycle()` 复用粒子对象
- **参数共享**: 两种模式共享大部分配置，减少内存开销
- **自动销毁**: 粒子生命周期结束后自动失效并可被回收

## 使用场景

### 战斗系统
- **普通伤害**: FACTORY 用于普通攻击造成的血液滴落
- **暴击伤害**: BURST 用于暴击或重击造成的血液飞溅
- **持续伤害**: 毒素、燃烧等持续伤害的血液效果

### UI系统  
- **生命值警告**: 低血量时的血液效果提示
- **伤害数字**: 血液粒子配合伤害数字显示

### 特殊事件
- **Boss战**: 强大敌人的特殊攻击造成的大量血液效果
- **剧情演出**: 关键战斗场面的血液特效
- **死亡动画**: 角色或重要NPC死亡时的血液效果

## 相关类
- **PixelParticle.Shrinking**: 收缩粒子基类，提供基础的缩小动画
- **Emitter**: 粒子发射器，管理粒子池和发射逻辑  
- **Factory**: 粒子工厂接口，标准化粒子创建流程
- **Random**: 随机数生成器，用于爆发模式的行为随机化
- **PointF**: 2D点坐标类，提供数学常数 PI2 和向量操作

## 设计哲学
`BloodParticle` 类体现了游戏特效设计的几个重要原则：

### 场景适配
通过两种不同的工厂和重置方法，精确适配不同的游戏场景需求。

### 参数调优
精心调整的颜色、生命周期、加速度等参数，创造出符合游戏暴力美学的视觉效果。

### 扩展性设计
虽然是专用类，但通过工厂模式和多重重置方法提供了良好的扩展基础。

### 性能平衡
在保证视觉质量的同时，通过简单的计算和对象复用确保性能稳定。
