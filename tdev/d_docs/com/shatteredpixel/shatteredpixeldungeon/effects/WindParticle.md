# WindParticle 类

## 概述
`WindParticle` 类是 Shattered Pixel Dungeon 游戏中的风粒子效果类。它继承自 `PixelParticle`，专门用于表现气流、风效果或风暴相关的视觉效果。

## 功能描述
该类实现了风粒子的核心特性，具有以下特点：
- **全局方向**: 所有风粒子共享相同的基础方向，创造统一的风向效果
- **随机参数**: 每个粒子的生命周期、尺寸和微调角度都随机变化
- **淡入淡出**: 透明度先增加后减少，创造自然的出现和消失效果
- **内部类Wind**: 提供持续的风效果发射器，支持视野优化
- **工厂模式**: 提供标准的粒子工厂用于高效创建和复用

## 核心字段

### 静态配置
- **angle**: 全局风向角度，使用随机初始化但所有粒子共享
- **speed**: 全局风速向量，基于角度和固定速度(5)计算

### 工厂实例
- **FACTORY**: 静态工厂实例，负责风粒子的创建和回收
  - `emit()`: 从发射器回收并重置粒子

### 内部状态
- **lifespan**: 生命周期在1-2秒之间随机
- **size**: 尺寸在0-3像素之间随机
- **am**: 透明度使用特殊的淡入淡出曲线

## 构造函数

### WindParticle()
**功能**: 创建基础风粒子实例

**实现细节**:
- 调用父类构造函数初始化基础 particle
- **生命周期随机**: 设置1-2秒之间的随机生命周期
- **尺寸随机**: 设置0-3像素之间的随机初始尺寸
- **缩放设置**: 应用随机尺寸到缩放属性

## 核心方法

### reset(float x, float y)
**功能**: 重置并配置风粒子的初始状态

**实现逻辑**:
1. **激活粒子**: 调用 `revive()` 激活粒子使其可见
2. **生命周期**: 设置随机的完整生命周期 (1-2秒)
3. **速度配置**: 
   - 使用全局风速向量 (`WindParticle.speed`)
   - 根据粒子尺寸缩放速度 (`speed.scale(size)`)
   - 创造大小不同的粒子有不同的移动速度
4. **位置计算**: 
   - 计算起始位置确保粒子在生命周期中点经过目标点
   - 公式: `x = targetX - speed.x * lifespan / 2`
5. **方向微调**: 
   - 为全局角度添加小的随机偏移 (-0.1到+0.1弧度)
   - 更新全局速度向量以保持一致性

### update()
**功能**: 每帧更新风粒子状态

**实现逻辑**:
- 调用父类的 `update()` 处理基础的运动逻辑
- **透明度控制**: 
  - 前半段 (p < 0.5f): `am = p * size * 0.2f` (线性增加)
  - 后半段: `am = (1 - p) * size * 0.2f` (线性减少)
  - 透明度与粒子尺寸成正比，大粒子更明显

## 内部类 Wind

### Wind(int pos)
**功能**: 创建持续风效果发射器

**实现逻辑**:
1. **位置计算**: 使用 `DungeonTilemap.tileToWorld()` 转换格子坐标到世界坐标
2. **发射配置**: 
   - 设置发射区域为整个格子大小
   - 以2.5粒子/秒的频率持续发射风粒子
3. **视野优化**: 只在英雄视野内时才更新和渲染

### update()
**功能**: 更新持续风效果

**实现逻辑**:
- **视野检查**: 只有当格子在英雄视野内时才更新 (`Dungeon.level.heroFOV[pos]`)
- **性能优化**: 视野外自动暂停更新，节省性能开销
- **调用父类**: 在视野内时调用父类更新逻辑

## 技术特点

### 全局一致性
- **统一风向**: 所有粒子共享相同的基础方向，创造真实的风场效果
- **动态微调**: 每次创建粒子时微调全局方向，避免完全机械重复
- **尺寸速度关联**: 大粒子移动更快，符合真实物理直觉

### 视觉设计
- **淡入淡出**: 特殊的透明度曲线模拟风粒子的自然出现和消散
- **尺寸差异**: 随机尺寸增加视觉层次感
- **生命周期变化**: 不同持续时间创造动态的粒子密度变化

### 性能优化
- **对象池**: 通过 `emitter.recycle()` 复用粒子对象
- **视野裁剪**: 只在必要时更新，大幅减少性能开销
- **简单计算**: 使用高效的线性插值和基本运算

- **风系魔法**: 风系魔法技能的视觉表现
- **范围技能**: 风系范围攻击的视觉反馈
- **环境互动**: 角色与风环境的交互效果

### UI系统
- **环境提示**: 风力环境的视觉提醒
- **状态指示**: 风系状态的持续视觉反馈
- **技能预览**: 风系技能的预览效果

### 特殊事件
- **Boss战**: 风系Boss的特殊技能效果
- **剧情演出**: 关键风场景的视觉强化
- **机关触发**: 风力机关激活时的效果

- **DungeonTilemap**: 地牢瓦片地图，提供坐标转换功能

## 设计哲学
`WindParticle` 类体现了游戏特效设计的几个重要原则：

### 环境模拟
通过全局一致的风向和随机参数的结合，创造出逼真的风场效果。

### 性能意识
通过视野裁剪和对象复用，在保证视觉质量的同时确保性能稳定。

### 实用导向
内部Wind类提供了开箱即用的持续风效果，简化了上层代码的使用。

### 视觉和谐
淡入淡出的透明度和随机尺寸确保风效果与游戏整体美术风格保持一致。
