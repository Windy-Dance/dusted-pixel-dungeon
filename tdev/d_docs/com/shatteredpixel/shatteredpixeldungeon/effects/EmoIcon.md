# EmoIcon 类

## 概述
`EmoIcon` 类是 Shattered Pixel Dungeon 游戏中用于在角色头顶显示情绪图标效果的基础类。它继承自 `Image`，提供了一个可扩展的框架，支持多种情绪状态的动态图标显示，如睡眠、警觉、调查和迷路等。

## 功能描述
该类实现了情绪图标的自动定位、缩放动画和生命周期管理。它会自动跟随目标角色移动，并提供呼吸式的缩放动画效果，使图标更加生动和引人注目。

## 核心特性

### 自动定位
- 图标始终显示在角色精灵的右上角位置
- 支持相机对齐确保像素完美渲染
- 可通过重写 `centerPoint()` 方法自定义定位偏移

### 呼吸动画  
- 在1.0到`maxSize`之间进行循环缩放
- 使用线性插值实现平滑的放大缩小效果
- 通过`timeScale`参数控制动画速度

### 生命周期管理
- 自动添加到游戏场景 (`GameScene.add()`)
- 根据可见性状态智能更新
- 无需手动清理，随角色销毁而自动移除

## 核心字段

### 动画控制
- **maxSize**: 最大缩放比例，默认值为2.0
- **timeScale**: 动画速度比例，默认值为1.0  
- **growing**: 当前是否处于放大阶段的标志

### 关联对象
- **owner**: 目标角色精灵 (`CharSprite`)

## 构造函数

### EmoIcon(CharSprite owner)
**功能**: 创建基础情绪图标实例

**实现细节**:
- 调用父类构造函数
- 保存所有者引用
- 自动添加到当前游戏场景

## 核心方法

### update()
**功能**: 每帧更新图标状态和位置

**实现逻辑**:
1. **动画更新**: 
   - 如果正在放大：`scale = min(scale + elapsed * timeScale, maxSize)`
   - 如果正在缩小：`scale = max(scale - elapsed * timeScale, 1.0)`
   - 在达到极值时切换方向
2. **位置更新**:
   - 计算中心点偏移
   - 使用 `PixelScene.align()` 确保像素对齐
   - 定位在角色右上角：`x = owner.x + owner.width - center.x`, `y = owner.y - center.y`

### centerPoint()
**功能**: 获取图标的中心参考点

**默认实现**: 返回图像的几何中心 `(width/2, height/2)`

**可重写**: 子类可以重写此方法实现自定义定位

## 内部子类

### Sleep (睡眠)
- **图标**: 使用 `Icons.SLEEP` 图标
- **动画参数**: `maxSize=1.2f`, `timeScale=0.5f`
- **定位偏移**: 显著向上偏移，模拟漂浮的Z字形睡眠符号
- **随机初始化**: 随机起始缩放比例增加变化性

### Alert (警觉)
- **图标**: 使用 `Icons.ALERT` 图标  
- **动画参数**: `maxSize=1.3f`, `timeScale=2.0f`
- **定位偏移**: 向上并向左偏移，强调紧急感
- **底部对齐**: 以图标底部左侧为对齐点

### Investigate (调查)
- **图标**: 使用 `Icons.INVESTIGATE` 图标
- **动画参数**: `maxSize=1.3f`, `timeScale=1.5f`  
- **定位偏移**: 与Alert相同，保持一致性
- **使用场景**: 角色正在探索或检查环境时

### Lost (迷路)
- **图标**: 使用 `Icons.LOST` 图标
- **动画参数**: `maxSize=1.25f`, `timeScale=1.0f`
- **定位偏移**: 与Alert/Investigate相同
- **使用场景**: AI角色失去目标或困惑状态

## 技术特点
- **面向对象设计**: 通过继承实现不同情绪类型的差异化
- **配置驱动**: 动画参数集中配置，便于调整
- **视觉层次**: 不同情绪类型有适当的动画强度区分
- **性能优化**: 只在可见时更新，减少不必要的计算

## 使用场景
该类系统主要用于以下情况：
- **AI状态指示**: 显示敌人的当前行为状态（睡眠、警觉等）
- **玩家反馈**: 在特定情况下向玩家提供视觉提示
- **游戏机制可视化**: 如隐身失效、探索模式等
- **叙事元素**: 增强角色的表现力和游戏沉浸感

## 相关类
- **CharSprite**: 角色精灵基类，作为图标的宿主
- **Icons**: 图标资源管理类，提供各种情绪图标
- **GameScene**: 游戏场景管理，处理对象生命周期
- **PixelScene**: 像素对齐工具，确保清晰渲染
- **PointF**: 二维坐标类，处理位置计算

## 扩展性
该设计具有良好的扩展性：
- **新情绪类型**: 只需继承 `EmoIcon` 并重写构造函数和 `centerPoint()`
- **自定义动画**: 可以重写 `update()` 方法实现不同的动画效果
- **复合效果**: 可以组合多个情绪图标或与其他特效结合
- **动态参数**: 支持运行时修改动画参数以适应不同情境

## 设计模式
- **模板方法模式**: `update()` 提供基础框架，`centerPoint()` 允许定制
- **工厂模式**: 静态内部类作为具体产品的工厂
- **组合模式**: 多个图标可以同时存在于场景中
