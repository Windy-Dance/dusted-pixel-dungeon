# TargetedCell 类

## 概述
`TargetedCell` 类是 Shattered Pixel Dungeon 游戏中的目标单元格效果类。它继承自 `Image`，专门用于在地牢格子上显示目标选择、技能范围或攻击目标的高亮效果。

## 功能描述
该类实现了目标标记的核心行为，具有以下特点：
- **目标图标**: 使用标准的目标图标 (`Icons.TARGET`) 表现选中效果
- **颜色定制**: 支持自定义颜色以适配不同技能或效果类型
- **淡出动画**: 简洁的淡出和缩放动画，2秒内完全消失
- **格子对齐**: 精确对齐到地牢格子系统
- **自动销毁**: 效果结束后自动清理资源

## 核心字段

### 配置参数
- **alpha**: 当前透明度值，控制淡出进度
- **origin**: 设置为水平中心点，确保图标居中显示

### 视觉属性  
- **图标**: 使用 `Icons.TARGET` 目标图标纹理
- **颜色**: 通过 `hardlight()` 方法应用自定义颜色
- **淡出速度**: 固定2秒淡出时间 (`Game.elapsed/2f`)

## 构造函数

### TargetedCell(int pos, int color)
**功能**: 创建目标单元格效果实例

**实现细节**:
- 加载 `Icons.TARGET` 目标图标纹理
- **颜色设置**: 使用 `hardlight(color)` 应用指定颜色
- **原点设置**: 设置水平原点为中心点，确保图标居中
- **位置计算**: 使用 `DungeonTilemap.tileToWorld(pos)` 将格子坐标转换为世界坐标
- **初始状态**: 设置初始透明度为1（完全不透明）

## 核心方法

### update()
**功能**: 每帧更新目标效果状态

**实现逻辑**:
- **淡出计算**: 
  - 透明度递减: `alpha -= Game.elapsed/2f`
  - 对应2秒的完整淡出时间
- **视觉更新**: 
  - 设置透明度: `alpha(alpha)`
  - 设置缩放: `scale.set(alpha)` (同步缩小效果)
- **生命周期管理**: 
  - 透明度降至0时调用 `killAndErase()` 销毁对象
  - 自动清理资源并从渲染树中移除

## 技术特点

### 坐标系统
- **精确对齐**: 使用 `tileToWorld()` 确保图标精确对齐到格子中心
- **系统集成**: 完美集成到地牢瓦片地图系统
- **坐标转换**: 正确处理一维格子索引到二维世界坐标的转换

### 视觉设计
- **颜色语义**: 自定义颜色可以表示不同类型的目标（红色攻击、蓝色技能等）
- **同步缩放**: 透明度和缩放同步变化，创造统一的淡出效果
- **简洁高效**: 最小化的代码实现确保高性能

### 性能优化
- **轻量实现**: 仅包含必要的功能，避免不必要的开销
- **自动清理**: 效果结束后自动释放所有资源
- **简单计算**: 使用高效的数学运算避免复杂开销

## 使用场景

### 技能系统
- **技能范围**: 显示技能影响范围的格子高亮
- **目标选择**: 玩家选择技能目标时的视觉反馈
- **攻击预览**: 近战或远程攻击范围的预览效果

### 战斗系统
- **攻击目标**: 显示具体攻击目标的格子标记
- **AOE效果**: 范围伤害技能的影响区域高亮
- **状态效果**: 特殊状态影响的格子标记

### UI系统
- **交互反馈**: 重要交互操作的目标确认
- **教学引导**: 新手教程中的目标选择演示
- **战术提示**: AI建议或战术提示的目标标记

### 特殊事件
- **剧情演出**: 关键剧情中的目标指示效果
- **机关触发**: 机关激活时的目标格子高亮
- **成就解锁**: 相关成就达成时的目标效果

## 相关类
- **Image**: 图像基类，提供基础渲染功能
- **Icons.TARGET**: 目标图标资源
- **DungeonTilemap**: 地牢瓦片地图，提供坐标转换功能
- **Game**: 游戏主类，提供时间流逝信息

专注于单一目的（目标标记），避免不必要的复杂性。

### 视觉清晰
目标图标在各种背景下都能清晰可见，提供明确的选择反馈。

### 性能意识
通过最小化的实现确保在高频率使用时的性能稳定。

### 系统集成
