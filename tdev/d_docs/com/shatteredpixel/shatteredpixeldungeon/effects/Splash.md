# Splash 类

## 概述
`Splash` 类是 Shattered Pixel Dungeon 游戏中用于创建粒子飞溅效果的静态工具类。它提供了一系列静态方法，用于在指定位置生成指定颜色和数量的粒子效果，常用于表现伤害、治疗、魔法释放等各种游戏事件的视觉反馈。

## 功能描述
该类实现了高效的粒子飞溅系统，具有以下特点：
- **颜色缓存**: 为每种颜色维护独立的工厂实例，支持同时显示多种颜色的飞溅效果  
- **灵活定向**: 支持全向飞溅和定向锥形飞溅
- **多种模式**: 支持瞬间爆发和持续发射两种模式
- **智能管理**: 自动处理粒子池和生命周期管理

## 核心设计

### 工厂缓存系统
- **FACTORIES**: `HashMap<Integer, SplashFactory>` 存储每种颜色对应的工厂
- **按需创建**: 当请求的颜色没有对应工厂时，自动创建新的工厂实例  
- **并发支持**: 多个不同颜色的飞溅效果可以同时存在而互不影响

### SplashFactory 内部类
继承自 `Emitter.Factory`，负责实际的粒子创建：
- **颜色定制**: 每个工厂实例绑定特定的颜色值
- **方向控制**: 支持指定发射方向 (`dir`) 和锥形角度 (`cone`)
- **粒子配置**: 创建 `PixelParticle.Shrinking` 粒子，设置颜色、大小、速度和加速度

## 核心方法

### at(int cell, int color, int n)
**功能**: 在指定地图格子位置创建全向飞溅效果

**实现细节**:
- 转换格子坐标到世界坐标中心点
- 调用重载方法进行实际创建

### at(PointF p, int color, int n)
**功能**: 在指定世界坐标创建全向飞溅效果

**参数说明**:
- `p`: 世界坐标位置
- `color`: ARGB颜色值  
- `n`: 粒子数量

**实现细节**:
- 方向设为 -π/2 (向上)
- 锥形角度设为 π (180度，半圆形向上)

### at(PointF p, float dir, float cone, int color, int n)
**功能**: 在指定位置创建定向锥形飞溅效果

**参数说明**:
- `dir`: 发射方向（弧度，0 = 向右）
- `cone`: 锥形角度（弧度，π = 180度）

### around(Visual v, int color, int n)
**功能**: 围绕指定视觉对象创建飞溅效果

**使用场景**: 表现角色受到攻击或获得增益时的环绕效果

### at(PointF p, float dir, float cone, int color, int n, float interval)
**功能**: 在指定位置创建持续发射的飞溅效果

**关键区别**: 使用 `emitter.start()` 而非 `emitter.burst()`
- **burst**: 瞬间发射所有粒子
- **start**: 按指定间隔持续发射粒子

## 粒子特性
- **类型**: `PixelParticle.Shrinking` (会随时间缩小的像素粒子)
- **大小**: 4像素基础尺寸
- **透明度**: 随机 0.5f - 1.0f
- **速度**: 随机 40-80 像素/秒，在指定锥形范围内
- **加速度**: 固定向下加速度 +100 (模拟重力效果)
- **生命周期**: 自动管理，无需手动清理

## 技术特点
- **内存效率**: 复用粒子池，避免频繁对象创建销毁
- **性能优化**: 颜色工厂缓存减少重复初始化开销  
- **灵活性**: 支持任意颜色、方向、数量和发射模式
- **易用性**: 静态方法设计，调用简单直观

## 使用场景
该类广泛应用于各种游戏效果：
- **战斗反馈**: 伤害数字周围的血花效果（红色）
- **治疗效果**: 治疗时的绿色粒子飞溅
- **魔法效果**: 各种法术释放时的元素粒子（蓝色、紫色等）  
- **环境互动**: 物品破坏、机关触发时的碎片效果
- **UI反馈**: 重要事件的视觉强调效果

## 相关类
- **Emitter**: 粒子发射器基类，提供粒子池管理
- **PixelParticle**: 像素粒子基类，提供基础粒子行为
- **GameScene.emitter()**: 获取全局粒子发射器实例
- **DungeonTilemap.tileCenterToWorld()**: 坐标转换工具
- **Random**: 随机数生成器，用于粒子参数随机化

## 扩展性
虽然该类设计简洁，但具有良好的扩展性：
- **自定义粒子**: 可以创建不同的粒子工厂实现特殊效果
- **复杂轨迹**: 修改加速度和速度计算实现曲线运动  
- **声音配合**: 结合音效系统增强反馈体验
- **多层效果**: 组合多个飞溅效果创建复合视觉体验

## 性能考量
- **批量操作**: 单次调用可创建大量粒子，减少方法调用开销
- **资源复用**: 粒子对象池避免GC压力
- **条件检查**: 对无效参数（n <= 0）进行早期返回
- **延迟初始化**: 工厂实例按需创建，减少启动开销
