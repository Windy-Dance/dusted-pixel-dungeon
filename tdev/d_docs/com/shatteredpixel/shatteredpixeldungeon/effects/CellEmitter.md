# CellEmitter 类

## 概述
`CellEmitter` 类是 Shattered Pixel Dungeon 游戏中用于在指定地图格子位置创建粒子发射器的工厂类。它提供了一系列静态方法，用于在不同位置和配置下快速创建粒子效果。

## 功能描述
该类封装了地图格子坐标到世界坐标的转换逻辑，并与游戏场景中的粒子发射器系统集成，使得开发者可以轻松地在任意地图格子上创建各种粒子效果。

## 核心方法

### floor(int cell)
**功能**: 创建地板粒子发射器

**参数**: 
- `cell`: 地图格子索引

**实现细节**:
- 使用 `DungeonTilemap.tileToWorld()` 将格子索引转换为世界坐标
- 从 `GameScene.floorEmitter()` 获取专门用于地板效果的发射器
- 设置发射器位置和尺寸为整个格子区域 (`DungeonTilemap.SIZE x DungeonTilemap.SIZE`)
- **用途**: 主要用于地面相关的粒子效果，如灰尘、水渍、血迹等

### get(int cell)
**功能**: 创建通用粒子发射器（覆盖整个格子）

**参数**: 
- `cell`: 地图格子索引

**实现细节**:
- 转换格子坐标到世界坐标
- 从 `GameScene.emitter()` 获取通用发射器
- 设置发射器覆盖整个格子区域
- **用途**: 适用于需要在整个格子范围内随机发射粒子的效果

### center(int cell)
**功能**: 创建中心点粒子发射器

**参数**: 
- `cell`: 地图格子索引

**实现细节**:
- 转换格子坐标到世界坐标
- 在格子中心位置创建发射器 (`x + SIZE/2, y + SIZE/2`)
- **用途**: 适用于从格子中心向外扩散的效果，如爆炸、魔法光环等

### bottom(int cell)
**功能**: 创建底部粒子发射器

**参数**: 
- `cell`: 地图格子索引

**实现细节**:
- 转换格子坐标到世界坐标  
- 在格子底部边缘创建发射器 (`y + SIZE`)
- 宽度设为 `DungeonTilemap.SIZE`，高度设为 0（线性发射）
- **用途**: 适用于从底部向上发射的效果，如火焰、蒸汽等

## 技术特点
- **坐标转换封装**: 隐藏了格子索引到世界坐标的复杂转换逻辑
- **多种发射模式**: 提供四种不同的发射器配置，满足不同视觉需求
- **性能优化**: 复用游戏场景中的发射器池，避免频繁创建销毁对象
- **类型安全**: 所有方法都返回 `Emitter` 对象，便于链式调用和配置

## 坐标系统说明
- **格子索引 (cell)**: 一维数组索引，表示地图中的位置
- **世界坐标 (world coordinates)**: 二维像素坐标，用于实际渲染
- **格子尺寸**: `DungeonTilemap.SIZE` 定义了每个格子的像素大小（通常是 16x16 或 32x32）

## 使用示例
```java
// 在指定格子显示火花效果
CellEmitter.get(pos).burst(SparkParticle.FACTORY, 10);

// 在格子中心显示魔法效果  
CellEmitter.center(pos).pour(MagicParticle.FACTORY, 0.5f);

// 在地板上显示血迹效果
CellEmitter.floor(pos).start(BloodParticle.FACTORY, 0.1f, 5);
```

## 相关类
- **DungeonTilemap**: 地图瓦片管理类，提供坐标转换功能
- **GameScene**: 游戏场景类，管理粒子发射器池
- **Emitter**: 粒子发射器基类，提供实际的粒子管理功能
- **PointF**: 二维浮点坐标类，用于存储世界坐标

## 设计模式
该类采用了**工厂模式**和**静态工厂方法模式**：
- **工厂模式**: 封装了对象创建的复杂性
- **静态工厂方法**: 提供多个具有不同语义的创建方法
- **单一职责**: 专门负责粒子发射器的创建，不涉及其他逻辑
