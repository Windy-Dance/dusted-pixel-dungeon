# Lightning 类

## 概述
`Lightning` 类是 Shattered Pixel Dungeon 游戏中用于创建闪电效果的高级视觉组件类。它继承自 `Group`，通过组合多个动态更新的弧线段来实现逼真的闪电放电效果，常用于雷电魔法、电系攻击或其他需要强烈视觉冲击的场合。

## 功能描述
该类实现了完整的闪电效果系统，包括：
- **多弧线支持**: 可以同时显示多条闪电弧线，创建复杂的放电网络  
- **动态几何**: 弧线形状每帧随机微调，模拟真实的闪电分支效果
- **淡出动画**: 闪电效果在指定时间内平滑淡出
- **回调机制**: 效果结束后可触发自定义回调函数

## 技术原理
- **分段渲染**: 每条闪电由两个图像段组成（起始段和结束段）
- **动态更新**: 每帧重新计算弧线控制点，创造闪烁效果
- **光照混合**: 使用OpenGL光照混合模式增强视觉冲击力  
- **坐标转换**: 支持格子索引和世界坐标的灵活输入

## 核心常量和字段

### 时间控制
- **DURATION**: 闪电效果持续时间 (0.3秒)
- **life**: 剩余生命周期时间

### 几何参数
- **A**: 弧度到角度的转换系数 (`180 / Math.PI`)
- **arcs**: 存储所有闪电弧线的列表
- **callback**: 效果结束后的回调函数

## 构造函数

### 单弧线构造函数
提供四种重载版本，支持不同的坐标输入方式：
- `Lightning(int from, int to, Callback callback)`: 格子到格子
- `Lightning(PointF from, int to, Callback callback)`: 世界坐标到格子  
- `Lightning(int from, PointF to, Callback callback)`: 格子到世界坐标
- `Lightning(PointF from, PointF to, Callback callback)`: 世界坐标到世界坐标

**实现细节**: 将单个弧线包装为列表，调用主构造函数

### 主构造函数 Lightning(List<Arc> arcs, Callback callback)
**功能**: 创建包含多条弧线的闪电效果

**实现逻辑**:
- 调用父类构造函数
- 保存弧线列表引用
- 将所有弧线添加到组容器中
- 保存回调函数引用
- 初始化生命周期为 DURATION

## 核心方法

### update()
**功能**: 每帧更新闪电状态

**实现逻辑**:
1. **生命周期管理**: 
   - 如果 `life <= 0`: 销毁对象并执行回调
   - 否则: 计算当前透明度比例 (`alpha = life / DURATION`)
2. **透明度更新**: 遍历所有弧线，设置统一的透明度值
3. **父类更新**: 调用 `super.update()` 处理基础更新逻辑

### draw()
**功能**: 渲染闪电效果

**实现逻辑**:
- 设置光照混合模式 (`Blending.setLightMode()`)
- 调用父类绘制方法渲染所有弧线
- 恢复正常混合模式 (`Blending.setNormalMode()`)

## Arc 内部类

### 设计原理
`Arc` 类代表单条闪电弧线，继承自 `Group` 并包含两个图像段：

- **arc1**: 起始段，从起点延伸到中间控制点
- **arc2**: 结束段，从中间控制点延伸到终点  
- **动态控制点**: 每帧在起点和终点中点附近随机偏移

### 构造函数
提供四种重载版本，与主类保持一致的坐标输入方式。

**核心实现**:
1. **坐标转换**: 将格子索引转换为世界坐标中心点
2. **图像创建**: 使用 `Effects.get(Effects.Type.LIGHTNING)` 获取闪电纹理
3. **原点设置**: 设置原点为左边缘中心点 (`origin.set(0, height/2)`)
4. **初始布局**: 调用 `update()` 进行首次几何计算

### update()
**功能**: 每帧重新计算弧线几何

**算法细节**:
1. **控制点生成**: 
   - 计算起点和终点的中点
   - 在中点周围随机偏移 (-4, +4) 像素
2. **起始段计算**:
   - 方向: `atan2(dy, dx)` 从起点指向控制点  
   - 缩放: 距离除以纹理宽度
3. **结束段计算**:
   - 方向: `atan2(dy, dx)` 从控制点指向终点
   - 位置和缩放: 同起始段计算逻辑

### alpha(float alpha)
**功能**: 设置弧线透明度

**实现**: 同时设置两个图像段的透明度值

## 视觉特性
- **逼真效果**: 随机控制点创造自然的闪电分支外观
- **强烈对比**: 光照混合模式使闪电更加醒目
- **短暂冲击**: 0.3秒的短持续时间强调瞬间爆发感
- **灵活布局**: 支持任意两点间的闪电连接

## 使用场景
该类主要用于以下游戏效果：
- **雷电魔法**: 法师的闪电链、雷击等技能
- **电系攻击**: 电属性敌人的远程攻击  
- **环境效果**: 雷暴天气、电力机关的视觉表现
- **BOSS技能**: 强力敌人释放的连锁闪电攻击
- **特殊事件**: 某些剧情或任务触发的闪电效果

## 性能考虑
- **轻量级**: 对象结构简单，内存占用小
- **高效渲染**: 复用现有纹理资源，避免额外加载
- **自动清理**: 生命周期结束后自动销毁，防止内存泄漏
- **批量处理**: 支持多条弧线的同时渲染

## 相关类
- **Effects**: 提供闪电纹理资源
- **Group**: 容器基类，提供批量管理功能  
- **Image**: 图像基类，处理纹理和变换
- **DungeonTilemap**: 坐标转换工具类
- **Blending**: OpenGL混合模式管理
- **Callback**: 回调接口，处理效果完成事件

## 扩展性
虽然专注于闪电效果，但设计支持多种扩展：
- **颜色定制**: 可以修改纹理或添加着色效果
- **复杂分支**: 添加更多弧线段创建更复杂的闪电网络
- **声音同步**: 结合音效系统增强沉浸感  
- **物理交互**: 与游戏物理系统集成实现真实碰撞效果

- **简洁API**: 多种构造函数覆盖常见使用场景
- **高性能**: 最小化计算开销，适合频繁使用
- **视觉冲击**: 光照混合模式最大化视觉效果
