# BannerSprites 类

## 概述
`BannerSprites` 类是 Shattered Pixel Dungeon 游戏中横幅/标语图像的工厂类，负责创建和管理各种游戏界面横幅。

## 功能描述
该类提供了一个静态方法 `get()`，可以根据指定的横幅类型返回对应的图像对象。所有横幅都存储在单一的纹理图集（texture atlas）中，通过 UV 坐标来裁剪出特定的横幅区域。

## 枚举类型 Type
`BannerSprites.Type` 枚举定义了游戏中可用的所有横幅类型：

- **TITLE_PORT**: 竖屏标题横幅，用于竖屏模式下的游戏标题显示
- **TITLE_GLOW_PORT**: 竖屏发光标题横幅，带有发光效果的竖屏标题
- **TITLE_LAND**: 横屏标题横幅，用于横屏模式下的游戏标题显示
- **TITLE_GLOW_LAND**: 横屏发光标题横幅，带有发光效果的横屏标题
- **BOSS_SLAIN**: BOSS 击败横幅，显示 "BOSS SLAIN" 字样的胜利提示
- **GAME_OVER**: 游戏结束横幅，显示 "GAME OVER" 字样的失败提示

## 核心方法

### get(Type type)
**功能**: 根据指定的类型创建并返回对应的横幅图像

**参数**: 
- `type`: `BannerSprites.Type` 枚举值，指定要创建的横幅类型

**返回值**: 
- `Image`: 返回从纹理图集中裁剪出的横幅图像对象

**实现细节**:
- 使用 `Assets.Interfaces.BANNERS` 作为纹理资源（注意这里是 Interfaces 而不是 Effects）
- 通过 `uvRect()` 方法指定每个横幅在纹理图集中的具体位置和尺寸
- 区分了横屏和竖屏布局，适配不同设备方向
- 发光版本的横幅通常是普通版本的增强视觉效果

## 技术特点
- **多方向支持**: 同时支持横屏和竖屏两种布局模式
- **视觉层次**: 提供普通和发光两种视觉效果，增加界面丰富度
- **纹理图集优化**: 所有横幅打包在同一个纹理文件中，减少内存占用和加载时间
- **精确尺寸控制**: 每个横幅都有精确的像素尺寸定义，确保显示效果一致

## 使用场景
该类主要被游戏的界面系统调用，例如：
- 游戏启动时的标题屏幕
- BOSS 战斗胜利后的奖励界面
- 角色死亡后的游戏结束界面
- 各种需要显示大型文本横幅的游戏状态切换场景
