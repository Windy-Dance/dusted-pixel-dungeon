# Swap 类

## 概述
`Swap` 类是 Shattered Pixel Dungeon 游戏中的位置交换效果系统。它继承自 `Actor`，专门用于处理两个角色位置互换时的平滑动画效果和游戏逻辑同步。

## 功能描述
该类实现了完整的位置交换系统，具有以下核心特性：
- **双向移动**: 同时处理两个角色的相向移动动画
- **距离适配**: 交换延迟时间基于角色间距离自动计算
- **音频反馈**: 播放传送音效提供听觉反馈
- **逻辑同步**: 确保视觉动画与游戏逻辑完全同步
- **环境更新**: 自动处理英雄相关的视野和迷雾更新

## 核心字段

### 配置参数
- **ch1/ch2**: 交换的两个角色对象
- **eff1/eff2**: 对应的两个视觉效果实例
- **delay**: 交换延迟时间，基于距离计算 (`distance * 0.1f`)

### 内部状态
- **passed**: 已经过的时间（在Effect内部类中）

## 构造函数

### Swap(Char ch1, Char ch2)
**功能**: 创建位置交换效果实例

**实现细节**:
- 保存两个角色引用
- **延迟计算**: `delay = distance(ch1.pos, ch2.pos) * 0.1f`
  - 距离越远，动画时间越长
  - 最小0.1秒，每格距离增加0.1秒
- **效果创建**: 
  - 创建两个Effect实例，分别处理两个角色的移动
  - 每个Effect从起始位置移动到目标位置
- **音效播放**: 播放 `Assets.Sounds.TELEPORT` 传送音效

## 核心方法

### act()
**功能**: Actor系统的主执行方法

**实现逻辑**:
- 返回 false，表示不立即完成，而是由内部Effect控制完成时机
- 实际的逻辑在 `finish()` 方法中处理

### finish(Effect eff)
**功能**: 完成交换逻辑的主要方法

**实现逻辑**:
1. **效果跟踪**: 标记完成的Effect实例为null
2. **完成检测**: 只有当两个Effect都完成后才执行最终逻辑
3. **资源清理**: 从Actor系统中移除当前Swap实例
4. **位置交换**: 交换两个角色的实际游戏位置
5. **占用更新**: 调用 `Dungeon.level.occupyCell()` 更新地图占用状态
6. **环境同步**: 如果涉及英雄，更新视野和迷雾 (`Dungeon.observe()`, `GameScene.updateFog()`)

## 内部类 Effect

### Effect(CharSprite sprite, int from, int to)
**功能**: 内部视觉效果构造函数

**实现细节**:
- 继承自 `Visual` 提供基础视觉功能
- **坐标转换**: 使用 `worldToCamera()` 将世界坐标转换为相机坐标
- **缓动计算**: 
  - 初始速度: `2 * distance / delay`
  - 加速度: `-speed / delay`
  - 实现匀加速运动的缓动效果
- **层级管理**: 将效果添加到精灵父容器中

### update()
**功能**: 每帧更新交换效果状态

**核心逻辑**:
1. **时间跟踪**: 累计经过的时间
2. **动画阶段判断**:
   - **进行中阶段** (`passed < delay`): 将角色精灵位置同步到当前动画位置
   - **完成阶段** (`passed >= delay`): 
     - 将角色精灵定位到最终位置
     - 销毁视觉效果 (`killAndErase`)
     - 调用 `finish()` 完成交换逻辑

## 技术特点

### 缓动动画
- **二次缓动**: 使用匀加速运动创造自然的交换效果
- **精确时长**: 基于距离的动态时间确保不同距离的一致体验
- **坐标转换**: 正确处理世界坐标到屏幕坐标的转换

### 系统集成
- **Actor系统**: 完全集成到游戏的Actor调度系统
- **音频系统**: 与音效系统协同提供完整的感官反馈
- **环境系统**: 自动处理视野、迷雾等环境状态更新

### 并发安全
- **双效果协调**: 确保两个角色的效果完全同步
- **状态检查**: 只有当两个效果都完成才执行最终逻辑
- **资源管理**: 正确的创建和销毁生命周期管理

## 使用场景

### 战斗系统
- **位置交换技能**: 角色或敌人的位置交换技能效果
- **战术位移**: 战术性的角色位置调整
- **Boss技能**: 强大Boss的位置操控技能

### 移动系统
- **传送交换**: 传送类技能的位置交换效果
- **强制移动**: 任何强制改变两个角色位置的游戏机制
- **机关触发**: 位置交换机关的视觉表现

### 特殊事件
- **剧情演出**: 关键剧情中的角色位置交换
- **谜题解决**: 位置相关的谜题完成效果
- **成就解锁**: 位置交换相关成就的庆祝效果

## 相关类
- **Actor**: 游戏动作系统基类，提供调度功能
- **Char**: 角色基类，作为交换的目标对象
- **CharSprite**: 角色精灵类，提供视觉表现和坐标转换
- **Dungeon.level**: 地牢关卡信息，提供距离计算和占用更新
- **GameScene**: 游戏场景类，提供迷雾更新功能
- **Sample**: 音频系统，播放传送音效
- **Visual**: 视觉对象基类，为内部Effect提供基础功能

## 设计哲学
`Swap` 类体现了游戏开发中的几个重要设计原则：

### 分离关注点
将视觉动画（Effect内部类）与游戏逻辑（Swap主类）分离，提高代码可维护性。

### 系统集成
深度集成到现有的Actor、音频、环境等系统中，而不是创建孤立的功能。

### 用户体验
通过缓动动画、音效反馈和环境同步提供完整的用户体验。

### 性能意识
使用对象复用、精确调度和资源管理确保性能稳定。

## 扩展建议

### 动画增强
- **不同交换类型**: 支持不同类型的交换动画（直线、弧线、瞬移等）
- **强度可视化**: 根据交换距离调整动画速度和轨迹
- **粒子效果**: 配合粒子系统增强交换的视觉冲击力

### 功能扩展
- **碰撞检测**: 交换过程中与障碍物的交互处理
- **连锁交换**: 多个角色的连锁位置交换
- **方向指示**: 显示交换方向的视觉提示

### 用户定制
- **动画速度**: 允许玩家调节交换动画的速度
- **效果强度**: 根据玩家偏好调整交换效果的明显程度
- **音效选项**: 支持不同的交换音效选择
