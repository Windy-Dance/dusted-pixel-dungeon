# BlobEmitter 类

## 概述
`BlobEmitter` 类是 Shattered Pixel Dungeon 游戏中用于渲染和显示"blob"（区域效果）的粒子发射器类。它继承自 `Emitter`，专门用于处理游戏中的各种区域效果，如毒雾、火焰、水等。

## 功能描述
该类负责根据 blob 对象的数据，在游戏地图上相应位置发射粒子效果。它与 `Blob` 系统紧密集成，确保只有在玩家视野内或标记为始终可见的区域才会显示粒子效果。

## 主要字段

### 核心字段
- **blob**: 关联的 `Blob` 对象，包含实际的效果数据
- **bound**: `RectF` 类型的边界矩形，默认值为 (0, 0, 1, 1)

### bound 字段说明
`bound` 字段定义了粒子在单个地图格子内的发射范围：
- **left/top**: 粒子发射区域的左上角偏移（相对于格子左上角）
- **right/bottom**: 粒子发射区域的右下角偏移（相对于格子左上角）
- 默认值 (0, 0, 1, 1) 表示整个格子都可以发射粒子
- 通过调整这个值可以控制粒子在格子内的分布密度和位置

## 构造函数

### BlobEmitter(Blob blob)
**功能**: 创建新的 blob 发射器实例

**参数**: 
- `blob`: 要关联的 `Blob` 对象

**实现细节**:
- 调用父类构造函数
- 保存 blob 引用
- 调用 `blob.use(this)` 将发射器注册到 blob 对象

## 核心方法

### emit(int index)
**功能**: 重写父类的 emit 方法，实现基于 blob 数据的粒子发射逻辑

**实现细节**:
1. **体积检查**: 如果 blob 的 volume ≤ 0，直接返回（无效果可显示）
2. **区域初始化**: 如果 blob 的 area 为空，调用 `setupArea()` 初始化区域
3. **遍历区域**: 遍历 blob 区域内的每个地图格子
4. **可见性检查**: 对每个格子进行以下检查：
   - 格子索引是否在有效范围内
   - 格子在玩家视野内 **或** blob 标记为始终可见
   - 格子的 blob 强度值 > 0
5. **粒子发射**: 对于满足条件的格子，在随机位置发射粒子

**坐标计算**:
- 使用 `Random.Float(bound.left, bound.right)` 和 `Random.Float(bound.top, bound.bottom)` 在格子内生成随机偏移
- 乘以 `DungeonTilemap.SIZE` 转换为实际像素坐标
- 调用 `factory.emit()` 创建并发射粒子

## 技术特点
- **性能优化**: 只在玩家视野内渲染粒子效果，减少不必要的计算
- **灵活配置**: 通过 `bound` 字段可以精细控制粒子发射区域
- **自动管理**: 与 blob 系统自动同步，无需手动管理生命周期
- **内存效率**: 复用现有的 blob 数据结构，避免额外内存开销

## 使用场景
该类主要用于以下类型的区域效果：
- **毒雾效果**: 显示绿色毒气粒子
- **火焰效果**: 显示燃烧的火焰粒子  
- **水效果**: 显示水面波动或水花粒子
- **魔法效果**: 各种持续性的魔法区域效果
- **环境效果**: 如蒸汽、烟雾等环境粒子

## 相关类
- **Blob**: 区域效果数据类，存储每个格子的效果强度
- **Emitter**: 粒子发射器基类，提供基础的粒子管理功能
- **DungeonTilemap**: 地图瓦片管理类，提供格子尺寸信息
- **Dungeon.level.heroFOV**: 英雄视野数组，用于可见性判断

## 扩展性
通过继承 `BlobEmitter` 或修改 `bound` 字段，可以轻松创建不同视觉效果的区域效果发射器：
- 减小 bound 范围可以使粒子集中在格子中心
- 调整 bound 形状可以创建特定的粒子分布模式
- 结合不同的粒子工厂可以改变粒子的外观和行为
