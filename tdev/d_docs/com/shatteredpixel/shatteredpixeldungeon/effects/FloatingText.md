# FloatingText 类

## 概述
`FloatingText` 类是 Shattered Pixel Dungeon 游戏中用于显示漂浮文本效果的核心UI组件类。它继承自 `RenderedTextBlock`，专门用于在战斗、交互或游戏事件中显示短暂的数值反馈、状态提示和图标信息。

## 功能描述
该类实现了完整的漂浮文本系统，包括：
- **数值显示**: 伤害值、治疗量、经验值等数字反馈
- **图标系统**: 内置40+种游戏图标，涵盖伤害类型、状态效果、货币等
- **堆叠管理**: 智能处理多个文本的垂直堆叠，避免重叠
- **动画效果**: 平滑的上升淡出动画，提供专业的视觉体验

## 核心特性

### 图标系统
类中定义了大量静态常量，组织成逻辑分类：

**战斗伤害图标 (0-4)**
- `PHYS_DMG`: 物理伤害（可格挡）
- `PHYS_DMG_NO_BLOCK`: 穿透物理伤害  
- `MAGIC_DMG`: 魔法伤害
- `PICK_DMG`: 破甲伤害

**持续伤害图标 (5-17)**
- `HUNGER`: 饥饿伤害
- `BURNING`: 燃烧伤害
- `SHOCKING`: 电击伤害  
- `FROST`: 冰冻伤害
- `WATER`: 水属性伤害
- `BLEEDING`: 流血伤害
- `TOXIC`: 毒性伤害
- `CORROSION`: 腐蚀伤害
- `POISON`: 中毒伤害
- `OOZE`: 黏液伤害
- `DEFERRED`: 延迟伤害
- `CORRUPTION`: 腐化伤害
- `AMULET`: 护符相关伤害

**正面效果图标 (18-22)**
- `HEALING`: 治疗效果
- `SHIELDING`: 护盾效果  
- `EXPERIENCE`: 经验值获得
- `STRENGTH`: 力量增益

**货币图标 (23-24)**
- `GOLD`: 金币
- `ENERGY`: 能量点数

**命中原因图标 (36-47)**
显示攻击命中的具体原因，如武器加成、祝福效果、准确度戒指等

**未命中原因图标 (72-82)**
显示攻击未命中的具体原因，如护甲加成、闪避戒指、敏捷姿态等

### 堆叠系统
- **智能排列**: 自动检测文本重叠并垂直调整位置
- **时间管理**: 被推挤的文本会减少剩余时间，防止刷屏
- **键值系统**: 通过 `key` 参数实现相关文本的智能分组

## 核心常量

### 动画参数
- **LIFESPAN**: 文本显示总时间 (1.0秒)
- **DISTANCE**: 文本上升总距离 (等于 `DungeonTilemap.SIZE`)
- **ICON_WIDTH/HEIGHT**: 图标尺寸 (7x8像素)

### 系统常量  
- **NO_ICON**: 表示无图标 (-1)

## 主要字段
- **icon**: 图标图像对象（如果存在）
- **iconLeft**: 图标是否显示在文本左侧
- **timeLeft**: 剩余显示时间
- **key**: 用于堆叠管理的键值
- **stacks**: 静态堆叠管理器 (`SparseArray<ArrayList<FloatingText>>`)

## 构造函数

### FloatingText()
**功能**: 创建基础漂浮文本实例

**实现细节**:
- 调用父类构造函数，设置字体大小为 `9*PixelScene.defaultZoom`
- 禁用文本高亮功能

## 核心方法

### update()
**功能**: 每帧更新文本状态

**实现逻辑**:
- **透明度控制**: 前50%时间完全不透明，后50%线性淡出
- **位置更新**: 每帧向上移动 `DISTANCE/LIFESPAN * elapsed` 像素
- **生命周期**: 时间结束后调用 `kill()` 销毁对象

### layout()
**功能**: 布局计算，处理图标位置

**实现逻辑**:
- 如果有图标，根据 `iconLeft` 决定放置在左侧或右侧
- 使用 `PixelScene.align()` 确保像素对齐
- 调用父类布局方法确保文本正确排版

### width()
**功能**: 计算总宽度（包括图标）

**实现逻辑**: 返回父类宽度 + 图标宽度 - 0.5像素（微调）

### kill()/destroy()
**功能**: 对象销毁，清理堆叠引用

**实现逻辑**: 从对应的堆叠列表中移除自身，避免内存泄漏

### reset(float x, float y, String text, int color, int iconIdx, boolean left)
**功能**: 重置并配置文本显示

**参数说明**:
- `x/y`: 显示位置（世界坐标）
- `text`: 要显示的文本
- `color`: 文本颜色（ARGB格式）
- `iconIdx`: 图标索引，`NO_ICON` 表示无图标
- `left`: 图标是否在左侧

**实现逻辑**:
- 复活对象（如果已销毁）
- 设置文本、颜色和缩放
- 创建并配置图标（如果需要）
- 设置精确位置（像素对齐）
- 重置生命周期计时器

## 静态工厂方法

### show(float x, float y, String text, int color)
**功能**: 显示无图标的漂浮文本

### show(float x, float y, int key, String text, int color)
**功能**: 显示带键值的漂浮文本（支持堆叠）

### show(float x, float y, int key, String text, int color, int iconIdx, boolean left)
**功能**: 完整版本，支持所有参数

**实现细节**:
- 使用 `Game.runOnRenderThread()` 确保线程安全
- 从 `GameScene.status()` 获取可用的文本对象池
- 调用 `reset()` 配置显示参数
- 如果有键值，调用 `push()` 进行堆叠管理

### push(FloatingText txt, int key)
**功能**: 将文本添加到堆叠系统

**堆叠算法**:
1. 获取对应键值的堆叠列表
2. 如果列表非空，从上到下检查重叠
3. 对重叠的文本：
   - 向上调整位置（间隔4像素）
   - 减少剩余时间（防止刷屏滥用）
4. 将新文本添加到列表末尾

## 命中/未命中原因分析系统

### getHitReasonIcon(Char attacker, float accRoll, Char defender, float defRoll)
**功能**: 分析攻击命中的具体原因

**分析流程**:
1. **保证命中检查**: 先检查必然命中的特殊情况
2. **装备检测**: 获取攻击者武器和防御者护甲
3. **增益分析**: 收集所有提高命中率的因素
4. **减益分析**: 收集所有降低闪避率的因素  
5. **逆向计算**: 从最终命中结果"逆推"出主要原因
6. **排序返回**: 按影响程度排序，返回最主要的原因

### getMissReasonIcon(Char attacker, float accRoll, Char defender, float defRoll)
**功能**: 分析攻击未命中的具体原因

**分析流程**: 与命中原因类似，但关注提高闪避和降低命中的因素

## 技术特点
- **线程安全**: 通过 `Game.runOnRenderThread()` 确保渲染线程安全
- **对象池**: 复用文本对象，避免频繁创建销毁
- **智能堆叠**: 防止文本重叠和刷屏问题
- **丰富图标**: 内置完整的视觉反馈系统
- **精确分析**: 复杂的命中/未命中原因推断系统

## 使用场景
该类系统主要用于：
- **战斗反馈**: 显示每次攻击的伤害值和命中情况
- **状态变化**: 显示buff/debuff的效果数值
- **资源获得**: 显示金币、经验等资源获取
- **技能效果**: 显示各种技能的具体数值反馈
- **游戏事件**: 任何需要短暂数值显示的场合

## 相关类
- **RenderedTextBlock**: 文本块基类，提供基础文本渲染
- **GameScene**: 游戏场景管理，提供对象池
- **Assets.Effects.TEXT_ICONS**: 图标纹理资源
- **PixelScene**: 像素对齐工具
- **SparseArray**: 高效的稀疏数组实现

## 设计复杂性
命中/未命中原因分析系统被作者注释为"messy"（混乱），因为：
- **逆向工程**: 需要从最终结果反推原因，而非正向跟踪
- **重复逻辑**: 多处代码重复相同的数值计算
- **硬编码值**: 大量魔法数字和比例值
- **状态依赖**: 依赖多个buff和装备系统的内部状态

尽管如此，该系统在实际游戏中工作良好，提供了丰富的战斗反馈信息。
