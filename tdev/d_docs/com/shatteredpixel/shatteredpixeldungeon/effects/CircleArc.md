# CircleArc 类

## 概述
`CircleArc` 类是 Shattered Pixel Dungeon 游戏中用于创建圆形弧线效果的高级视觉组件类。它继承自 `Visual`，通过 OpenGL 直接绘制三角形扇形来实现动态的圆形扫描或填充效果。

## 功能描述
该类实现了基于三角剖分的圆形弧线渲染系统，支持动态调整弧线的扫掠角度、颜色模式和动画效果。常用于技能范围指示、充能效果、雷达扫描等圆形视觉表现。

## 技术原理
- **三角剖分**: 将圆形弧线分解为多个三角形进行渲染
- **OpenGL 直接绘制**: 使用底层 OpenGL API 进行高效渲染  
- **动态更新**: 根据扫掠角度实时重新计算顶点数据
- **混合模式**: 支持光照模式和正常混合模式

## 核心字段

### 时间控制
- **duration**: 动画总持续时间
- **lifespan**: 剩余生命周期时间

### 几何参数  
- **sweep**: 扫掠角度比例 (0.0 = 无弧线, 1.0 = 完整圆形)
- **dirty**: 顶点数据是否需要更新的标记
- **lightMode**: 是否使用光照混合模式
- **nTris**: 三角形数量（决定弧线精度）
- **rad**: 圆形半径

### 图形数据
- **texture**: 纯白色纹理（用于着色）
- **vertices**: 临时顶点数据数组
- **verticesBuffer**: 顶点缓冲区（FloatBuffer）
- **indices**: 索引缓冲区（ShortBuffer）

## 构造函数

### CircleArc(int triangles, float radius)
**功能**: 创建圆形弧线实例

**参数说明**:
- `triangles`: 三角形数量，越多精度越高但性能开销越大
- `radius`: 圆形半径（像素）

**实现细节**:
1. **纹理创建**: 使用 `TextureCache.createSolid(0xFFFFFFFF)` 创建纯白纹理
2. **缓冲区分配**: 
   - 顶点缓冲区: `(nTris * 2 + 1) * 4 * sizeof(float)` 字节
   - 索引缓冲区: `nTris * 3 * sizeof(short)` 字节
3. **初始设置**: 默认扫掠角度为 1.0（完整圆形）
4. **顶点初始化**: 调用 `updateTriangles()` 生成初始几何数据

## 核心方法

### color(int color, boolean lightMode)
**功能**: 设置弧线颜色和混合模式

**参数说明**:
- `color`: ARGB 颜色值
- `lightMode`: true 使用光照模式，false 使用正常混合

**实现细节**:
- 调用 `hardlight(color)` 应用颜色
- 设置 `lightMode` 标志位
- 支持链式调用

### show(Visual visual, float duration)
**功能**: 在指定视觉对象中心显示弧线动画

**实现细节**:
- 设置位置为中心点
- 添加到父容器的后部（背景层）
- 设置动画持续时间

### show(Group parent, PointF pos, float duration)  
**功能**: 在指定位置和容器中显示弧线动画

**实现细节**:
- 设置绝对位置
- 添加到指定父容器
- 设置动画持续时间

### setSweep(float sweep)
**功能**: 设置扫掠角度比例

**实现细节**:
- 范围检查避免不必要的更新
- 设置 `dirty = true` 标记需要重新计算顶点

### updateTriangles()
**功能**: 重新计算三角形顶点数据

**算法细节**:
1. **中心顶点**: 第一个顶点设为圆心 (0, 0)
2. **起始角度**: `2 * (π - π * sweep) - π/2.0` （从顶部开始）
3. **顶点生成**: 对每个三角形：
   - 计算两个边缘顶点的角度和坐标
   - 存储到顶点缓冲区
4. **索引生成**: 为每个三角形生成索引 (0, edge1, edge2)

### update()
**功能**: 每帧更新动画状态

**实现逻辑**:
- 如果有持续时间且生命周期未结束：
  - 计算当前扫掠比例：`sweep = lifespan/duration`
  - 标记需要更新顶点数据
- 否则销毁对象

### draw()
**功能**: 渲染圆形弧线

**渲染流程**:
1. **数据更新**: 如果 `dirty` 为真，调用 `updateTriangles()`
2. **混合模式**: 根据 `lightMode` 设置 OpenGL 混合模式
3. **纹理绑定**: 绑定纯白纹理
4. **着色器设置**: 设置模型矩阵和光照参数
5. **绘制调用**: 使用 `drawElements()` 绘制三角形
6. **模式恢复**: 如果使用了光照模式，恢复为正常混合

## 视觉特性
- **高精度**: 通过增加三角形数量可获得平滑的圆形边缘
- **动态范围**: 扫掠角度可以实时调整，实现扫描或填充动画
- **灵活着色**: 支持任意颜色和两种混合模式
- **性能优化**: 只在必要时重新计算几何数据

## 使用场景
该类主要用于以下游戏效果：
- **技能范围**: 显示圆形技能的影响范围
- **充能效果**: 表现技能或物品的充能进度  
- **雷达扫描**: 敌人探测或区域扫描的视觉效果
- **魔法光环**: 角色周围的保护或增益光环
- **爆炸预览**: 爆炸技能的范围预览效果

## 性能考虑
- **三角形数量**: 平衡精度和性能，通常 16-32 个三角形足够
- **缓冲区管理**: 使用直接字节缓冲区提高 OpenGL 性能
- **懒更新**: 只在几何参数改变时重新计算顶点
- **自动清理**: 动画完成后自动销毁，避免内存泄漏

## 相关类和技术
- **Visual**: 视觉组件基类，提供基础变换和渲染功能
- **SmartTexture/TextureCache**: 纹理管理系统
- **OpenGL 缓冲区**: FloatBuffer 和 ShortBuffer 用于 GPU 数据传输
- **NoosaScript**: 游戏引擎的 OpenGL 封装层
- **Blending**: 混合模式管理类

## 扩展应用
通过组合多个 `CircleArc` 实例或与其他特效结合，可以创建更复杂的视觉效果：
- **同心圆**: 多个不同半径的弧线创建层次感
- **渐变效果**: 结合透明度变化创建淡入淡出
- **复合形状**: 与其他几何图形组合创建复杂UI元素
